1. Bearer Tokenмқҳ л¬ём ң

OAuth 2.0мқҙ к·ңм •н•ҳлҠ” кё°ліё нҶ нҒ° нғҖмһ…мқҖ BearerлӢӨ. "Bearer"лҠ” "мҶҢм§Җмһҗ"лқјлҠ” лң» к·ёлҢҖлЎң, нҶ нҒ°мқ„ к°Җм§„ мһҗ лҲ„кө¬м—җкІҢлӮҳ к¶Ңн•ңмқ„ мқём •н•ңлӢӨ.

Authorization: Bearer eyJhbGciOi...

Bearerмқҳ кө¬мЎ°м Ғ н•ңкі„

  • нҶ нҒ° мһҗмІҙк°Җ к¶Ңн•ңмқҳ мң мқјн•ң мҰқкұ°лӢӨ. л°ңмӢ мһҗк°Җ лҲ„кө¬мқём§Җ, м •лӢ№н•ң мҶҢмң мһҗмқём§Җ кІҖмҰқн•ҳм§Җ м•ҠлҠ”лӢӨ.
  • мӨ‘к°„мһҗ(MITM)В·н”„лЎқмӢңВ·лЎңк·ёВ·лёҢлқјмҡ°м Җ нҷ•мһҘВ·м•…м„ұ м•ұ л“ұ м–ҙл–Ө кІҪлЎңлЎңл“  нҶ нҒ°мқҙ мң м¶ңлҗҳл©ҙ, кіөкІ©мһҗлҠ” л§ҢлЈҢ мӢңк°Ғк№Ңм§Җ мһҗмң лЎӯкІҢ к·ё нҶ нҒ°мқ„ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.
  • м§§мқҖ Access Token мҲҳлӘ…(5~15л¶„)мқҖ мҷ„нҷ” мҲҳлӢЁмқҙм§Җл§Ң, Refresh TokenмңјлЎң мһ¬л°ңкёүл°ӣлҠ” кө¬мЎ°м—җм„ң RTк°Җ мң м¶ңлҗҳл©ҙ мһҘкё°к°„ нҶ нҒ° мһҗлҸҷ мһ¬мғқмӮ°мқҙ к°ҖлҠҘн•ҳлӢӨ.

Bearerмқҳ н•ңкі„лҘј к·јліём ҒмңјлЎң н•ҙкІ°н•ҳл Өл©ҙ "нҶ нҒ°мқ„ к°Җм§„ кІғл§ҢмңјлЎңлҠ” л¶ҖмЎұн•ҳкі , л°ңмӢ мһҗк°Җ м •лӢ№н•ң мҶҢмң мһҗмһ„мқ„ м¶”к°ҖлЎң мҰқлӘ…"н•ҙм•ј н•ңлӢӨ. мқҙкІғмқҙ Sender-Constrained Tokenмқҳ к°ңл…җмқҙлӢӨ.

мқҙ мұ•н„°лҠ” м„ё л°©н–ҘмңјлЎң лҢҖмұ…мқ„ мӮҙнҺҙліёлӢӨ.

  • Refresh Token Rotation: RT мң м¶ңмқ„ к°җм§Җн•ҳкі  мЎ°кё° м°ЁлӢЁ.
  • DPoP: мҡ”мІӯл§ҲлӢӨ к°ңмқёнӮӨ м„ңлӘ…мңјлЎң мҶҢмң  мҰқлӘ….
  • mTLS Bound Token: нҒҙлқјмқҙм–ёнҠё TLS мқёмҰқм„ңм—җ нҶ нҒ°мқ„ л°”мқёл”©.

2. Refresh Token Rotation (RTR)

Refresh Token RotationмқҖ "л§Ө к°ұмӢ л§ҲлӢӨ мғҲ RTлҘј л°ңкёүн•ҳкі , мқҙм „ RTлҘј мҰүмӢң л¬ҙнҡЁнҷ”"н•ҳлҠ” м „лһөмқҙлӢӨ. кө¬ RTк°Җ мһ¬мӮ¬мҡ©лҗҳл©ҙ нғҲм·Ё мқҳмӢ¬мңјлЎң к°„мЈјн•ҙ м „мІҙ м„ём…ҳмқ„ нҸҗкё°н•ңлӢӨ.

мғҒм„ё кө¬нҳ„ мӮ¬лЎҖлҠ” лё”лЎңк·ё нҸ¬мҠӨнҠё Refresh нҶ нҒ°кіј RTRм—җ Spring Security кё°л°ҳ мҳҲм ңлЎң м •лҰ¬лҗҳм–ҙ мһҲлӢӨ.

  • н•ңкі„: AT мһҗмІҙмқҳ нғҲм·ЁлҠ” л§үм§Җ лӘ»н•Ё(м§§мқҖ мҲҳлӘ…м—җ мқҳмЎҙ). мһ¬мӮ¬мҡ© к°җм§Җ false positive к°ҖлҠҘм„ұ.

3. Sender-Constrained Token к°ңмҡ”

Sender-Constrained TokenмқҖ "мқҙ нҶ нҒ°мқҖ нҠ№м • л°ңмӢ мһҗл§Ң мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ"лҠ” м ңм•Ҫмқ„ нҶ нҒ°м—җ лӮҙмһҘн•ң кІғмқҙлӢӨ. н•өмӢ¬мқҖ "нҶ нҒ° л°ңкёү мӢң л°ңмӢ мһҗмқҳ нӮӨ мһ¬лЈҢ(public key, certificate)лҘј нҶ нҒ°м—җ л°”мқёл”©"н•ҳкі , "нҶ нҒ° мӮ¬мҡ© мӢң л°ңмӢ мһҗк°Җ к·ё нӮӨлҘј мӢӨм ңлЎң мҶҢмң н•Ёмқ„ мҰқлӘ…"н•ҳкІҢ н•ҳлҠ” кІғмқҙлӢӨ.

Token Binding(RFC 8471)мқҖ лёҢлқјмҡ°м Җ м§Җмӣҗ л¶Җмһ¬лЎң 2018л…„м—җ мӮ¬мӢӨмғҒ нҸҗкё°лҗҳм—ҲлӢӨ. нҳ„мһ¬ мӢӨм „м—җм„ң мқҳлҜё мһҲлҠ” кІғмқҖ DPoPкіј mTLSлӢӨ.

4. DPoP (Demonstrating Proof-of-Possession)

DPoPлҠ” RFC 9449(2023)лЎң н‘ңмӨҖнҷ”лҗң кІҪлҹү Proof-of-Possession л©”м»ӨлӢҲмҰҳмқҙлӢӨ. нҒҙлқјмқҙм–ёнҠёк°Җ нӮӨнҺҳм–ҙлҘј мғқм„ұн•ҙ кіөк°ңнӮӨлҘј нҶ нҒ°м—җ л°”мқёл”©н•ҳкі , мҡ”мІӯл§ҲлӢӨ к°ңмқёнӮӨлЎң DPoP Proof JWTлҘј м„ңлӘ…н•ҙ ліҙлӮёлӢӨ.

нҒҙлқјмқҙм–ёнҠё мёЎ мӨҖл№„

  • нҒҙлқјмқҙм–ёнҠёк°Җ кё°лҸҷ мӢң нӮӨнҺҳм–ҙ мғқм„ұ(ES256 к¶ҢмһҘ). к°ңмқёнӮӨлҠ” л””мҠӨнҒ¬м—җ м“°м§Җ м•ҠлҠ”лӢӨ. SPAлҠ” лёҢлқјмҡ°м Җ л©”лӘЁлҰ¬лӮҳ л№„лӮҙліҙлӮё(non-extractable) Web Crypto нӮӨ.
  • кіөк°ңнӮӨлҠ” JWK нҳ•нғңлЎң мӨҖл№„н•ҙ нҶ нҒ° мҡ”мІӯм—җ нҸ¬н•Ё.

нҶ нҒ° л°ңкёү нқҗлҰ„

Header:
  typ: "dpop+jwt"

  ath: "b64(SHA256(AT))"        вҶҗ AT мҡ”мІӯ мӢң AT л°”мқёл”©

GET /resource HTTP/1.1
Host: rs.example.com
Authorization: DPoP eyJhbGciOi...AT
DPoP: eyJ0eXAiOiJkcG9wK2p3dC...proof

мЈјлӘ©н•  м җмқҖ л‘җ к°Җм§ҖлӢӨ.

  • Authorizationмқҳ мҠӨнӮҙмқҙ Bearerк°Җ м•„лӢҲлқј DPoPлӢӨ.
  • л§Ө мҡ”мІӯл§ҲлӢӨ мғҲ DPoP Proof JWTк°Җ н•„мҡ”н•ҳлӢӨ. htmВ·htuм—җ н•ҙлӢ№ мҡ”мІӯмқҳ methodмҷҖ URLмқ„ лӢҙлҠ”лӢӨ.

RS кІҖмҰқ лЎңм§Ғ

1. Authorization н—ӨлҚ”м—җм„ң AT м¶”м¶ң вҶ’ м„ңлӘ…/л§ҢлЈҢ кІҖмҰқ
2. ATмқҳ cnf.jkt м¶”м¶ң (л°”мқёл”©лҗң кіөк°ңнӮӨ м§Җл¬ё)
3. DPoP н—ӨлҚ”м—җм„ң proof JWT м¶”м¶ң
4. proof headerмқҳ jwkлЎң м„ңлӘ… кІҖмҰқ
5. jwkмқҳ SHA-256 м§Җл¬ёмқҙ cnf.jktмҷҖ мқјм№ҳн•ҳлҠ”к°Җ?
6. proofмқҳ htm, htuк°Җ нҳ„мһ¬ мҡ”мІӯкіј мқјм№ҳн•ҳлҠ”к°Җ?
7. proofмқҳ iatмқҙ мөңк·ј(Вұ1л¶„)мқҙкі  jtiк°Җ мһ¬мӮ¬мҡ© м•„лӢҢк°Җ?

DPoPмқҳ нҡЁкіј

кіөкІ©мһҗк°Җ ATлҘј нғҲм·Ён•ҙлҸ„ нҒҙлқјмқҙм–ёнҠёмқҳ к°ңмқёнӮӨк°Җ м—Ҷмңјл©ҙ мң нҡЁн•ң DPoP ProofлҘј л§Ңл“Ө мҲҳ м—ҶлӢӨ. AT мһҗмІҙл§ҢмңјлЎңлҠ” м“ёлӘЁк°Җ м—Ҷм–ҙм§„лӢӨ.

н•ңкі„

  • л§Ө мҡ”мІӯл§ҲлӢӨ JWT м„ңлӘ…мқҙ н•„мҡ”н•ҙ м—°мӮ° л№„мҡ© мҰқк°Җ. ES256мқҖ л№„көҗм Ғ л№ лҘҙм§Җл§Ң л¬ҙмӢңн•  мҲҳмӨҖмқҖ м•„лӢҲлӢӨ.
  • RSм—җм„ң jtiлҘј лӢЁкё°к°„ м ҖмһҘн•ҙ мһ¬мӮ¬мҡ© л°©м§Җн•ҙм•ј н•Ё. л¶„мӮ° RS нҷҳкІҪм—җм„ңлҠ” replay cache кіөмң к°Җ н•„мҡ”.
  • Proofмқҳ htuк°Җ м •нҷ•н•ҙм•ј н•ҳлҜҖлЎң н”„лЎқмӢңк°Җ URLмқ„ мһ¬мһ‘м„ұн•ҳл©ҙ мӢӨнҢЁ.

5. mTLS Certificate-Bound Token

RFC 8705(2020)лҠ” TLS нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ңм—җ Access Tokenмқ„ л°”мқёл”©н•ҳлҠ” л°©мӢқмқ„ к·ңм •н•ңлӢӨ. м—”н„°н”„лқјмқҙмҰҲ B2B мӢңлӮҳлҰ¬мҳӨм—җм„ң л„җлҰ¬ м“°мқёлӢӨ.

к°ңмҡ”

  • нҒҙлқјмқҙм–ёнҠёлҠ” нҶ нҒ° мҡ”мІӯ мӢң mTLSлЎң ASм—җ м—°кІ°. ASлҠ” TLS мёөм—җм„ң нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ңлҘј нҷ•мқё.
  • л°ңкёүлҗң Access Tokenмқҳ cnf нҒҙл Ҳмһ„м—җ нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ңмқҳ SHA-256 м§Җл¬ёмқ„ лӢҙлҠ”лӢӨ.
  • лҰ¬мҶҢмҠӨ м„ңлІ„лҠ” TLS н•ёл“ңм…°мқҙнҒ¬м—җм„ң м–»мқҖ нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ң м§Җл¬ёмқҙ ATмқҳ cnf.x5t#S256кіј мқјм№ҳн•ҳлҠ”м§Җ кІҖмҰқ.

ATмқҳ cnf нҒҙл Ҳмһ„ мҳҲмӢң

{
  "iss": "https://as.example.com",
  "sub": "service-A",
  "aud": "https://rs.example.com",
  "exp": 1715003600,
  "cnf": {
    "x5t#S256": "bwcK0esc3ACC3DB2Y5_lESsXE8o9ltc05O89jdN-dg2"
  }
}

x5t#S256мқҖ X.509 мқёмҰқм„ңмқҳ SHA-256 thumbprintлӢӨ.

RS кІҖмҰқ нқҗлҰ„

1. TLS н•ёл“ңм…°мқҙнҒ¬ мҷ„лЈҢ вҶ’ нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ң нҷ•ліҙ
2. мқёмҰқм„ң SHA-256 м§Җл¬ё кі„мӮ° вҶ’ cert_thumbprint
3. AT м„ңлӘ… кІҖмҰқ, cnf.x5t#S256 м¶”м¶ң вҶ’ bound_thumbprint
4. cert_thumbprint == bound_thumbprint мқёк°Җ?
   - мҳҲ: мІҳлҰ¬
   - м•„лӢҲмҳӨ: 401 invalid_token

мһҘм җ

  • лі„лҸ„ м• н”ҢлҰ¬мјҖмқҙм…ҳ л ҲлІЁ м„ңлӘ…мқҙ н•„мҡ” м—ҶлӢӨ. TLS мҠӨнғқмқҙ лӘЁл“  л¬ҙкұ°мҡҙ мқјмқ„ н•ңлӢӨ.
  • м„ұлҠҘ мҳҒн–Ҙмқҙ DPoPліҙлӢӨ мһ‘лӢӨ(н•ёл“ңм…°мқҙнҒ¬лҠ” м„ём…ҳ мһ¬мӮ¬мҡ©).
  • м—”н„°н”„лқјмқҙмҰҲ PKIк°Җ мқҙлҜё мһҲлӢӨл©ҙ мҰүмӢң лҸ„мһ… к°ҖлҠҘ.

н•ңкі„

  • лёҢлқјмҡ°м Җ/SPAлҠ” нҳ„мӢӨм ҒмңјлЎң лӘ» м”Җ. нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ң кҙҖлҰ¬ UXк°Җ мӮ¬мҡ©мһҗм—җкІҢ л§һм§Җ м•ҠмқҢ.
  • м„ңл№„мҠӨ л©”мӢңВ·API кІҢмқҙнҠёмӣЁмқҙм—җм„ң TLSк°Җ мў…лЈҢлҗҳл©ҙ нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ңк°Җ мҶҗмӢӨлҗЁ. кІҢмқҙнҠёмӣЁмқҙк°Җ м§Җл¬ёмқ„ н—ӨлҚ”(x-ssl-client-sha256)лЎң м „лӢ¬н•ҳлҸ„лЎқ кө¬м„ұ н•„мҡ”.
  • мқёмҰқм„ң лЎңн…Ңмқҙм…ҳ мӢң нҶ нҒ°лҸ„ мһ¬л°ңкёү н•„мҡ”.

мЈјлҗң мӮ¬мҡ©мІҳлҠ” м„ңл№„мҠӨ к°„(B2BВ·M2M) нҶөмӢ мқҙлӢӨ. лӘЁл°”мқјВ·SPAм—җлҠ” DPoPк°Җ лҚ” м Ғн•©н•ҳлӢӨ.

6. м „лһө м„ нғқ к°Җмқҙл“ң

мӢӨл¬ҙм—җм„ң м–ҙл–Ө мЎ°н•©мқ„ м“ём§Җ кІ°м •н•  л•Ңмқҳ к°Җмқҙл“ңлқјмқёмқҙлӢӨ.

м• н”ҢлҰ¬мјҖмқҙм…ҳ мң нҳ•лі„ к¶ҢмһҘ

нҒҙлқјмқҙм–ёнҠёAT ліҙнҳёRT ліҙнҳё
м „нҶө мӣ№ л°ұм—”л“ң(Confidential)Bearer + м§§мқҖ expRotation
лӘЁл°”мқј л„ӨмқҙнӢ°лёҢ м•ұDPoP к¶ҢмһҘRotation + л””л°”мқҙмҠӨ л°”мқёл”©
SPABFF нҢЁн„ҙ(лӢӨмқҢ мұ•н„°) мҡ°м„ , л¶Ҳк°Җн”ј мӢң DPoPBFF лӮҙл¶Җ м ҖмһҘ
м„ңл№„мҠӨ к°„(M2M)mTLS BoundRT л¶Ҳн•„мҡ”н•ң кІҪмҡ°к°Җ л§ҺмқҢ (Client Credentials)

мЎ°н•© нҢЁн„ҙ

  • RTR + DPoP: ATлҠ” DPoPлЎң мҶҢмң  мҰқлӘ…, RTлҠ” RotationмңјлЎң нғҲм·Ё к°җм§Җ. к°ҖмһҘ к°•л Ҙн•ң мЎ°н•©.
  • RTR + mTLS: м—”н„°н”„лқјмқҙмҰҲ B2Bм—җ м Ғн•©.
  • Short AT + RTR: к°ҖмһҘ л„җлҰ¬ м“°мқҙлҠ” мӨ‘к°„ к°•лҸ„ н”„лЎңнҢҢмқј.

нҳ„мӢӨм Ғ нҢҗлӢЁ

мҷ„лІҪн•ң ліҙм•ҲмқҖ м—ҶлӢӨ. нҠёл Ҳмқҙл“ңмҳӨн”„лҘј л°ӣм•„л“Өмқҙкі  "мқҙ м„ңл№„мҠӨк°Җ л…ёлҰ¬лҠ” мң„нҳ‘ мҲҳмӨҖ"м—җ л§һкІҢ м„ нғқн•ңлӢӨ.

  • мҶҢк·ңлӘЁ B2C м„ңл№„мҠӨ: Short AT + RTRлЎң м¶©л¶„.
  • кёҲмңөВ·н—¬мҠӨмјҖм–ҙ л“ұ к·ңм ң мӮ°м—…: DPoP лҳҗлҠ” mTLS н•„мҲҳ.
  • лӘЁл°”мқј м „мҡ© м•ұ: OS Keystoreм—җ м ҖмһҘлҗң нӮӨ + DPoPк°Җ мң л Ҙ.

кіөнҶө мІҙнҒ¬лҰ¬мҠӨнҠё

  • AT мҲҳлӘ…мқҖ 5~15л¶„ лІ”мң„.
  • RTлҠ” Rotationмқ„ кё°ліёк°’мңјлЎң.
  • нҶ нҒ° revocation м—”л“ңнҸ¬мқёнҠёлҘј м ңкіөн•ҳкі  нҒҙлқјмқҙм–ёнҠёк°Җ лЎңк·ём•„мӣғ мӢң нҳём¶ң.
  • л°ңкёүВ·мӮ¬мҡ©В·revocation мқҙлІӨнҠёлҘј к°җмӮ¬ лЎңк·ёлЎң лӮЁкёҙлӢӨ.
  • ASВ·RS к°„ мӢңкі„ лҸҷкё°нҷ”(NTP)лҠ” н•„мҲҳ. м„ңлӘ… нҶ нҒ°мқҳ exp/iat кІҖмҰқмқҙ к№Ём§ҖлҠ” мЈјлҗң мӣҗмқёмқҙлӢӨ.

н•өмӢ¬ м •лҰ¬

  • Bearer TokenмқҖ "мҶҢм§Җмһҗк°Җ кі§ м •лӢ№н•ң мӮ¬мҡ©мһҗ"лЎң к°„мЈјн•ҳлҠ” кө¬мЎ°лқј, мң м¶ң мӢң л§ҢлЈҢк№Ңм§Җ л¬ҙл Ҙн•ҳлӢӨ. мқҙлҘј ліҙмҷ„н•ҳл Өл©ҙ м§§мқҖ мҲҳлӘ…В·RotationВ·Sender-Constrained м „лһөмқ„ мЎ°н•©н•ңлӢӨ.
  • Refresh Token RotationмқҖ л§Ө к°ұмӢ л§ҲлӢӨ RTлҘј көҗмІҙн•ҳкі , кіјкұ° RT мһ¬мӮ¬мҡ©мқ„ нғҲм·Ё мӢ нҳёлЎң к°„мЈјн•ҙ нҢЁл°ҖлҰ¬ м „мІҙлҘј нҸҗкё°н•ңлӢӨ. м •мғҒ мһ¬мӢңлҸ„лҘј кө¬л¶„н•ҳл Ө м§§мқҖ grace periodлҘј л‘”лӢӨ.
  • DPoPлҠ” нҒҙлқјмқҙм–ёнҠёк°Җ нӮӨнҺҳм–ҙлҘј мғқм„ұн•ҙ кіөк°ңнӮӨлҘј ATм—җ л°”мқёл”©н•ҳкі  мҡ”мІӯл§ҲлӢӨ к°ңмқёнӮӨлЎң Proof JWTлҘј м„ңлӘ…н•ңлӢӨ. AT нғҲм·ЁмһҗлҠ” к°ңмқёнӮӨк°Җ м—Ҷм–ҙ мһ¬мӮ¬мҡ© л¶Ҳк°Җ.
  • mTLS Bound TokenмқҖ TLS нҒҙлқјмқҙм–ёнҠё мқёмҰқм„ңмқҳ SHA-256 м§Җл¬ёмқ„ ATмқҳ cnf.x5t#S256м—җ мӢ¬м–ҙ, RSк°Җ TLS мёө кІҖмҰқмңјлЎң мҶҢмң лҘј нҷ•мқён•ңлӢӨ. B2BВ·M2M нҷҳкІҪм—җ м Ғн•©.
  • мӢ к·ң кө¬нҳ„мқҳ к¶ҢмһҘ мЎ°н•©мқҖ "Short AT + RTR + (н•„мҡ” мӢң DPoP)"мқҙл©°, м• н”ҢлҰ¬мјҖмқҙм…ҳ мң нҳ•кіј к·ңм ң мҲҳмӨҖм—җ л”°лқј mTLSлӮҳ BFF нҢЁн„ҙмқ„ м¶”к°Җн•ңлӢӨ.
