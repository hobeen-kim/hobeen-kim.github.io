Zanzibar 이해하기

구글의 Zanzibar는 YouTube·Drive·Photos·Cloud 등 수많은 서비스의 권한 검사를 초당 수백만 건 단위로 처리하는 글로벌 인가 시스템이다. 2019년 발표된 논문 *"Zanzibar: Google's Consistent, Global Authorization System"*은 관계 기반 접근 제어(ReBAC) 모델의 사실상 표준이 됐다. SpiceDB·OpenFGA 같은 오픈소스 구현은 모두 이 논문을 재료로 삼는다.

이 스터디는 Zanzibar 논문을 읽어내는 데 목적이 있다. 개념과 알고리즘에 집중하고, 실제 API 호출이나 배포는 후속 스터디인 SpiceDB에서 다룬다.

선행 지식

  • OAuth 스터디 CH3. 인증과 인가에서 인증(AuthN)과 인가(AuthZ)의 구분을 선행한다.
  • Keycloak 스터디 CH8. Authorization Services와 UMA가 대조군이다. Keycloak은 정책 엔진 기반, Zanzibar는 관계 그래프 기반.

목차

Zanzibar 기초

  1. 인가 모델과 ReBAC — RBAC/ABAC/ReBAC 비교와 SpiceDB/OpenFGA/Cedar 선택 기준
  2. Zanzibar 핵심 개념 — 구글이 풀려던 문제, relation tuple, 설계 원칙
  3. 관계 기반 데이터 모델 — object, relation, userset

아키텍처

  1. Userset Rewrite Rules — union/intersection/exclusion, tuple-to-userset
  2. Consistency와 Zookie — New Enemy 문제, Spanner snapshot read
  3. Check/Expand 평가 알고리즘 — 분산 질의 평가, 재귀 pointer chasing
  4. Watch와 Leopard 인덱스 — 변경 감지, 간접 멤버십 가속

다음 스터디

Zanzibar의 개념을 실제 오픈소스 구현에 적용하려면 SpiceDB 스터디로 이어진다. Schema 언어 작성, API 호출, 배포 토폴로지, 운영 튜닝까지 구현자와 운영자 시각에서 다룬다.
