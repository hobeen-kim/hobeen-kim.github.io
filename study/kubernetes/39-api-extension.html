1. API Aggregation Layer

37В·38мһҘмқҳ CRDлҠ” "apiserverк°Җ лҢҖмӢ  м ҖмһҘВ·кІҖмҰқн•ҙмЈјлҠ”" нҷ•мһҘмқҙм—ҲлӢӨ. лҚ°мқҙн„°лҠ” etcdм—җ л“Өм–ҙк°Җкі , мҠӨнӮӨл§Ҳ кІҖмҰқл§Ң мҡ°лҰ¬к°Җ м •мқҳн–ҲлӢӨ. к·ёлҹ°лҚ° м–ҙл–Ө APIлҠ” etcdм—җ м ҖмһҘн•ҳл©ҙ м•Ҳ лҗңлӢӨ. metrics-serverмқҳ metrics.k8s.io APIк°Җ к·ёл ҮлӢӨ. нҢҢл“ң CPUВ·л©”лӘЁлҰ¬лҠ” л§ӨмҙҲ ліҖн•ҳлҠ” мӢӨмӢңк°„ к°’мқҙлқј м ҖмһҘн•  н•„мҡ”лҸ„ м—Ҷкі  м ҖмһҘн•ҙм„ңлҸ„ м•Ҳ лҗңлӢӨ. мҡ”мІӯмқҙ мҳ¬ л•Ң мҰүм„қм—җм„ң кі„мӮ°н•ҙ лҸҢл ӨмӨҳм•ј н•ңлӢӨ.

мқҙлҹҙ л•Ң м“°лҠ” кІғмқҙ API Aggregation LayerлӢӨ. мҡ°лҰ¬к°Җ м§Ғм ‘ л§Ңл“  extension apiserverлҘј лқ„мҡ°кі , л©”мқё apiserverм—җкІҢ "metrics.k8s.io к·ёлЈ№ мҡ”мІӯмқҖ лӮҙ м„ңлІ„лЎң н”„лЎқмӢңн•ҙлқј"лқјкі  л“ұлЎқн•ңлӢӨ. л“ұлЎқ мҲҳлӢЁмқҙ APIService мҳӨлёҢм қнҠёлӢӨ.

2. CRD vs Aggregation Layer

л‘җ нҷ•мһҘ л°©мӢқмқҖ к°ҷмқҖ лӘ©н‘ң(мғҲ API м¶”к°Җ)лҘј лӢӨлҘё л№„мҡ©В·мһҗмң лҸ„лЎң лӢ¬м„ұн•ңлӢӨ. кіөмӢқ л№„көҗ л¬ём„ңлҘј л°”нғ•мңјлЎң м •лҰ¬н•ңлӢӨ.

мёЎл©ҙCRDAggregation Layer
кө¬нҳ„ лӮңмқҙлҸ„YAMLл§ҢмңјлЎң л“ұлЎқ, мҪ”л“ң л¶Ҳн•„мҡ”мһҗмІҙ apiserverлҘј GoлЎң кө¬нҳ„В·мҡҙмҳҒ
м ҖмһҘмҶҢetcd(мҝ лІ„л„ӨнӢ°мҠӨк°Җ кҙҖлҰ¬)мһҗмң (etcd лі„лҸ„ мқёмҠӨн„ҙмҠӨ, DB, л¬ҙм ҖмһҘ л“ұ)
кІҖмҰқВ·кё°ліёк°’OpenAPI мҠӨнӮӨл§Ҳ + CELмһ„мқҳ лЎңм§Ғ(мҪ”л“ңлЎң л¬ҙм—Үмқҙл“ )

мӢӨл¬ҙ мӣҗм№ҷмқҖ лӢЁмҲңн•ҳлӢӨ. к°ҖлҠҘн•ҳл©ҙ CRD, м–ҙм©” мҲҳ м—Ҷмқ„ л•Ңл§Ң AggregationмқҙлӢӨ. CRDлҠ” мҡҙмҳҒ л¶ҖлӢҙмқҙ кұ°мқҳ м—Ҷкі  мғқнғңкі„ лҸ„кө¬к°Җ лӢӨ нҶөн•ңлӢӨ. AggregationмқҖ metrics-server, apiservicesлҘј м“°лҠ” custom/external metrics API, мқјл¶Җ ліҙм•Ҳ м ңн’ҲмІҳлҹј "etcdм—җ лӘ» лӢҙкұ°лӮҳ н‘ңмӨҖ CRUDлҘј лІ—м–ҙлӮҳлҠ”" кІҪмҡ°м—җл§Ң м •лӢ№нҷ”лҗңлӢӨ. лҢҖлӢӨмҲҳ OperatorлҠ” CRDлЎң м¶©л¶„н•ҳлӢӨ.

3. admission webhook вҖ” мҡ”мІӯмқ„ к°ҖлЎңмұ„лӢӨ

API нҷ•мһҘмқҳ лҳҗ лӢӨлҘё м¶•мқҖ мғҲ APIлҘј м¶”к°Җн•ҳлҠ” кІҢ м•„лӢҲлқј кё°мЎҙ мҡ”мІӯмқ„ к°ҖлЎңмұ„лҠ” кІғмқҙлӢӨ. 8мһҘм—җм„ң ліё admission controlмқ„ л– мҳ¬лҰ¬мһҗ. мҡ”мІӯмқҙ мқёмҰқВ·мқёк°ҖлҘј нҶөкіјн•ң л’Ө, etcdм—җ м ҖмһҘлҗҳкё° м§Ғм „м—җ admission лӢЁкі„лҘј кұ°м№ңлӢӨ. м—¬кё°м—җ мҡ°лҰ¬ лЎңм§Ғмқ„ кҪӮлҠ” кІғмқҙ admission webhookмқҙлӢӨ.

л‘җ мў…лҘҳк°Җ мһҲлӢӨ.

  • Mutating webhook: к°қмІҙлҘј ліҖнҳ•н•ңлӢӨ. мӮ¬мқҙл“ңм№ҙ мЈјмһ…, кё°ліёк°’ м„Өм •, л Ҳмқҙлё” м¶”к°Җ л“ұ. JSON PatchлҘј лҸҢл ӨмӨҖлӢӨ.
  • Validating webhook: к°қмІҙлҘј кІҖмҰқл§Ң н•ңлӢӨ. м •мұ… мң„л°ҳмқҙл©ҙ кұ°л¶Җ. ліҖнҳ•мқҖ лӘ» н•ңлӢӨ.

мҲңм„ңк°Җ мӨ‘мҡ”н•ҳлӢӨ. mutatingмқҙ лЁјм Җ лҸҢм•„ к°қмІҙлҘј л°”кҫёкі , к·ёлӢӨмқҢ validatingмқҙ мөңмў… кІ°кіјлҘј кІҖмҰқн•ңлӢӨ.

Istio мӮ¬мқҙл“ңм№ҙ мһҗлҸҷ мЈјмһ…мқҙ mutating webhookмқҳ лҢҖн‘ң мӮ¬лЎҖлӢӨ. мӮ¬мҡ©мһҗк°Җ нҸүлІ”н•ң DeploymentлҘј applyн•ҳл©ҙ, mutating webhookмқҙ Envoy мӮ¬мқҙл“ңм№ҙ м»Ён…Ңмқҙл„ҲлҘј мҠ¬м©Қ м¶”к°Җн•ҙ м ҖмһҘн•ңлӢӨ. мӮ¬мҡ©мһҗлҠ” мӮ¬мқҙл“ңм№ҙлҘј мқҳмӢқн•ҳм§Җ лӘ»н•ңлӢӨ. OPA GatekeeperВ·Kyvernoмқҳ м •мұ… к°•м ңлҠ” validating webhook мӮ¬лЎҖлӢӨ.

4. admission webhook кө¬нҳ„ вҖ” м„ңлІ„В·мқёмҰқм„ңВ·кө¬м„ұ

webhookмқҖ м„ё мЎ°к°ҒмңјлЎң кө¬м„ұлҗңлӢӨ. (1) HTTPS м„ңлІ„, (2) TLS мқёмҰқм„ң, (3) Configuration мҳӨлёҢм қнҠёлӢӨ.

(1) webhook м„ңлІ„

webhook м„ңлІ„лҠ” apiserverк°Җ ліҙлӮҙлҠ” AdmissionReview JSONмқ„ л°ӣм•„, allowed м—¬л¶Җ(мҷҖ mutatingмқҙл©ҙ patch)лҘј лӢҙмқҖ AdmissionReviewлҘј мқ‘лӢөн•ҳлҠ” HTTPS н•ёл“Өлҹ¬лӢӨ.

func handleMutate(w http.ResponseWriter, r *http.Request) {
    var review admissionv1.AdmissionReview
    json.NewDecoder(r.Body).Decode(&review)

    // мҳҲ: лӘЁл“  нҢҢл“ңм—җ л Ҳмқҙлё” мЈјмһ…н•ҳлҠ” JSON Patch
    patch := `[{"op":"add","path":"/metadata/labels/injected","value":"true"}]`
    patchType := admissionv1.PatchTypeJSONPatch

    review.Response = &admissionv1.AdmissionResponse{
        UID:       review.Request.UID,   // мҡ”мІӯ UIDлҘј л°ҳл“ңмӢң echo
        Allowed:   true,
        Patch:     []byte(patch),
        PatchType: &patchType,
    }
    json.NewEncoder(w).Encode(review)
}

(2) TLS мқёмҰқм„ң

apiserverлҠ” webhookмқ„ л°ҳл“ңмӢң HTTPSлЎң нҳём¶ңн•ҳкі , webhook м„ңлІ„ мқёмҰқм„ңлҘј кІҖмҰқн•ңлӢӨ. к·ёлһҳм„ң м„ңлІ„ мқёмҰқм„ңмҷҖ, к·ёкІғмқ„ кІҖмҰқн•  CA лІҲл“Өмқ„ мӨҖл№„н•ҙ webhook configurationмқҳ caBundleм—җ л„Јм–ҙм•ј н•ңлӢӨ. мҡҙмҳҒм—җм„ңлҠ” cert-managerлЎң мқёмҰқм„ңлҘј л°ңкёүВ·к°ұмӢ н•ҳкі  CA InjectorлЎң caBundleмқ„ мһҗлҸҷ мЈјмһ…н•ҳлҠ” нҢЁн„ҙмқҙ н‘ңмӨҖмқҙлӢӨ.

(3) Configuration мҳӨлёҢм қнҠё

м–ҙл–Ө мҡ”мІӯмқ„ м–ҙл–Ө webhookмңјлЎң ліҙлӮјм§ҖлҠ” ValidatingWebhookConfiguration / MutatingWebhookConfigurationмңјлЎң м„ м–ён•ңлӢӨ.

apiVersion: admissionregistration.k8s.io/v1
kind: MutatingWebhookConfiguration
metadata:
  name: pod-label-injector
webhooks:
  - name: inject.example.com
    clientConfig:
      service:
        namespace: webhook-system
        name: webhook-svc
        path: /mutate
      caBundle: <base64 CA>
    rules:                       # м–ҙл–Ө мҡ”мІӯмқ„ к°ҖлЎңмұҢм§Җ
      - apiGroups: [""]
        apiVersions: ["v1"]
        operations: ["CREATE"]
        resources: ["pods"]
        scope: "Namespaced"
    # к°Җмҡ©м„ұВ·лІ”мң„ м ңм–ҙ
    namespaceSelector:           # нҠ№м • л„Өмһ„мҠӨнҺҳмқҙмҠӨл§Ң
      matchLabels:
        webhook: enabled
    failurePolicy: Fail          # webhook мӢӨнҢЁ мӢң мҡ”мІӯ кұ°л¶Җ(Fail) vs нҶөкіј(Ignore)
    sideEffects: None
    timeoutSeconds: 5
    admissionReviewVersions: ["v1"]

rulesлЎң лҢҖмғҒ мҡ”мІӯмқ„ мўҒнһҲкі , namespaceSelector/objectSelectorлЎң лІ”мң„лҘј м ңн•ңн•ҳкі , failurePolicy/timeoutSecondsлЎң к°Җмҡ©м„ұ лҸҷмһ‘мқ„ м •н•ңлӢӨ. мқҙ н•„л“ңл“Өмқҳ м„Өкі„к°Җ кі§ webhook мҡҙмҳҒ м•Ҳм •м„ұмқ„ мўҢмҡ°н•ңлӢӨ.

5. webhook мҡҙмҳҒ мЈјмқҳм җ

webhookмқҖ к°•л Ҙн•ҳм§Җл§Ң, лӘЁл“  кҙҖл Ё API мҡ”мІӯ кІҪлЎңм—җ лҒјм–ҙл“ңлҠ” лӢЁмқј мһҘм• м җмқҙ лҗ  мҲҳ мһҲлӢӨ. мӢӨл¬ҙм—җм„ң л°ҳл“ңмӢң мұҷкІЁм•ј н•  н•Ём •л“ӨмқҙлӢӨ.

failurePolicyмқҳ л”ңл Ҳл§Ҳ

webhook м„ңлІ„к°Җ мЈҪкұ°лӮҳ лҠҗлҰ¬л©ҙ м–ҙл–»кІҢ н•  кІғмқёк°Җ.

  • failurePolicy: Fail: webhook нҳём¶ң мӢӨнҢЁ мӢң мҡ”мІӯмқ„ кұ°л¶Җн•ңлӢӨ. м •мұ…мқҙ нҷ•мӢӨнһҲ к°•м ңлҗҳм§Җл§Ң, webhookмқҙ мЈҪмңјл©ҙ н•ҙлӢ№ лҰ¬мҶҢмҠӨ мғқм„ұмқҙ м „л¶Җ л§үнһҢлӢӨ. ліҙм•Ҳ м •мұ…мІҳлҹј мҡ°нҡҢлҘј м ҲлҢҖ н—Ҳмҡ©н•ҳл©ҙ м•Ҳ лҗ  л•Ң м“ҙлӢӨ.
  • failurePolicy: Ignore: мӢӨнҢЁ мӢң мҡ”мІӯмқ„ нҶөкіјмӢңнӮЁлӢӨ. к°Җмҡ©м„ұмқҖ м§ҖнӮӨм§Җл§Ң м •мұ…мқҙ мғҲм–ҙ лӮҳк°„лӢӨ.

мҲңнҷҳ мқҳмЎҙ(circular dependency)

к°ҖмһҘ мң„н—ҳн•ң н•Ём •мқҙлӢӨ. webhook м„ңлІ„ мһҗмӢ мқҙ PodлЎң л–  мһҲкі , к·ё webhookмқҙ pods CREATEлҘј failurePolicy: FailлЎң к°ҖлЎңмұҲлӢӨкі  н•ҳмһҗ. м–ҙлҠҗ лӮ  webhook нҢҢл“ңк°Җ м „л¶Җ мЈҪмңјл©ҙ, мғҲ webhook нҢҢл“ңлҘј лқ„мҡ°л ӨлҠ” Pod CREATE мҡ”мІӯмқҙ мЈҪмқҖ webhookмқ„ нҳём¶ңн•ҳл ӨлӢӨ кұ°л¶ҖлҗңлӢӨ. мҳҒмҳҒ лӘ» мӮҙм•„лӮңлӢӨ. мқҙлҘј н”јн•ҳл Өл©ҙ namespaceSelectorлЎң мһҗкё° л„Өмһ„мҠӨнҺҳмқҙмҠӨ(kube-system л“ұ н•өмӢ¬ л„Өмһ„мҠӨнҺҳмқҙмҠӨ нҸ¬н•Ё)лҘј webhook лҢҖмғҒм—җм„ң м ңмҷён•ҙм•ј н•ңлӢӨ.

к·ё л°–мқҳ мҡҙмҳҒ мҲҳм№ҷ

мЈјмқҳм җлҢҖмқ‘
м§Җм—°(latency)webhookмқҖ лҸҷкё° нҳём¶ңмқҙлқј лӘЁл“  мҡ”мІӯмқ„ лҠҗлҰ¬кІҢ л§Ңл“ лӢӨ. л№ лҘҙкІҢ мқ‘лӢөн•ҳкі  timeoutSecondsлҘј м§§кІҢ
к°Җмҡ©м„ұwebhook м„ңлІ„лҘј multi-replica + PodDisruptionBudgetмңјлЎң кі к°Җмҡ© кө¬м„ұ
лІ”мң„ нҸӯмЈјobjectSelector/namespaceSelectorлЎң кјӯ н•„мҡ”н•ң лҢҖмғҒл§Ң
л©ұл“ұм„ұmutatingмқҖ мһ¬нҳём¶ңлҸјлҸ„ к°ҷмқҖ кІ°кіјк°Җ лҗҳкІҢ(мқҙлҜё мЈјмһ…лҗң мӮ¬мқҙл“ңм№ҙлҘј лҳҗ мЈјмһ…н•ҳм§Җ м•ҠкІҢ)
sideEffectsмҷёл¶Җ мғҒнғңлҘј л°”кҫёл©ҙ sideEffectsлҘј м •нҷ•нһҲ м„ м–ё(dry-run м•Ҳм „м„ұ)

failurePolicy: FailмқҖ нҒҙлҹ¬мҠӨн„°лҘј л©Ҳм¶ң мҲҳ мһҲлӢӨ

л„“мқҖ rules(мҳҲ: лӘЁл“  лҰ¬мҶҢмҠӨ)м—җ Failмқ„ кұём–ҙл‘җкі  webhookмқҙ мЈҪмңјл©ҙ, нҒҙлҹ¬мҠӨн„°мқҳ кұ°мқҳ лӘЁл“  м“°кё°к°Җ л§үнһҢлӢӨ. н•өмӢ¬ л„Өмһ„мҠӨнҺҳмқҙмҠӨ м ңмҷё, мўҒмқҖ rules, м§§мқҖ timeout, кі к°Җмҡ© webhookмқ„ н•Ёк»ҳ к°–м¶”м§Җ м•ҠмқҖ мұ„ Failмқ„ м“°лҠ” кІғмқҖ нҒҙлҹ¬мҠӨн„° м „мІҙлҘј мқём§ҲлЎң мһЎлҠ” н–үмң„лӢӨ.

н•өмӢ¬ м •лҰ¬

  • API Aggregation LayerлҠ” мһҗмІҙ extension apiserverлҘј л©”мқё apiserver л’Өм—җ н”„лЎқмӢңлЎң лҒјмҡ°л©°, APIServiceлЎң л“ұлЎқн•ңлӢӨ. м ҖмһҘВ·кі„мӮ°В·мқёмҰқмқ„ мҷ„м „нһҲ м ңм–ҙн•ңлӢӨ(мҳҲ: metrics-server).
  • CRD vs Aggregation: к°ҖлҠҘн•ҳл©ҙ CRD(мҡҙмҳҒ л¶ҖлӢҙ лӮ®мқҢ), etcdм—җ лӘ» лӢҙкұ°лӮҳ н‘ңмӨҖ CRUDлҘј лІ—м–ҙлӮ  л•Ңл§Ң Aggregation.
  • admission webhookмқҖ мҡ”мІӯмқ„ к°ҖлЎңмұ„ mutating(ліҖнҳ•)В·validating(кІҖмҰқ)н•ңлӢӨ. mutatingмқҙ лЁјм Җ, validatingмқҙ лӮҳмӨ‘м—җ лҸҲлӢӨ.
  • webhookмқҖ HTTPS м„ңлІ„ + TLS мқёмҰқм„ң(caBundle) + Configuration мҳӨлёҢм қнҠё м„ё мЎ°к°ҒмңјлЎң кө¬м„ұлҗҳл©°, rulesВ·selectorВ·failurePolicyлЎң лІ”мң„мҷҖ к°Җмҡ©м„ұ лҸҷмһ‘мқ„ м •н•ңлӢӨ.
  • мҡҙмҳҒмқҳ н•өмӢ¬мқҖ failurePolicy м„ нғқ, мҲңнҷҳ мқҳмЎҙ нҡҢн”ј(н•өмӢ¬ л„Өмһ„мҠӨнҺҳмқҙмҠӨ м ңмҷё), м§§мқҖ timeout, кі к°Җмҡ© кө¬м„ұмқҙлӢӨ. мһҳлӘ» м“°л©ҙ webhookмқҙ нҒҙлҹ¬мҠӨн„° м „мІҙлҘј л©Ҳм¶ҳлӢӨ.

лӢӨмқҢ мұ•н„°

м—¬кё°к№Ңм§Җ мҝ лІ„л„ӨнӢ°мҠӨ APIлҘј нҷ•мһҘн•ҳлҠ” м„ё кёё вҖ” CRD, Operator, Aggregation/Webhook вҖ” мқ„ лӘЁл‘җ лӢӨлӨҳлӢӨ. нҷ•мһҘн•ң мӢңмҠӨн…ңмқҙ мһҳ лҸ„лҠ”м§Җ м•Ңл Өл©ҙ кҙҖмёЎмқҙ н•„мҡ”н•ҳлӢӨ. 40мһҘ л©”нҠёлҰӯкіј лӘЁлӢҲн„°л§Ғм—җм„ң metrics-server, Prometheus, к·ёлҰ¬кі  мқҙлІҲ мһҘм—җм„ң ліё custom/external metrics APIк°Җ м–ҙл–»кІҢ мҳӨнҶ мҠӨмјҖмқјл§Ғкіј кҙҖмёЎм„ұмңјлЎң мқҙм–ҙм§ҖлҠ”м§Җ мӮҙнҺҙліёлӢӨ.
