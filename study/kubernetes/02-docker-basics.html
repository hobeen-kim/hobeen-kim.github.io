Docker кё°мҙҲ

н•ҷмҠө лӘ©н‘ң

  • Dockerк°Җ clientВ·dockerdВ·containerdВ·runcлЎң мқҙм–ҙм§ҖлҠ” кі„мёө кө¬мЎ°лЎң лҸҷмһ‘н•Ёмқ„ мқҙн•ҙн•ңлӢӨ.
  • мқҙлҜём§Җ(image)мҷҖ м»Ён…Ңмқҙл„Ҳ(container)мқҳ кҙҖкі„лҘј нҒҙлһҳмҠӨмҷҖ мқёмҠӨн„ҙмҠӨм—җ л№—лҢҖ м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.
  • л Ҳм§ҖмҠӨнҠёлҰ¬(Docker Hub, private registry)мқҳ м—ӯн• кіј мқҙлҜём§Җ нғңк·ёВ·лӢӨмқҙм ңмҠӨнҠёмқҳ мқҳлҜёлҘј м•ҲлӢӨ.
  • runВ·psВ·execВ·logsВ·inspectВ·rmВ·pullВ·push л“ұ н•өмӢ¬ лӘ…л №м–ҙлЎң м»Ён…Ңмқҙл„ҲлҘј лӢӨлЈ° мҲҳ мһҲлӢӨ.
  • м»Ён…Ңмқҙл„Ҳмқҳ лқјмқҙн”„мӮ¬мқҙнҒҙ(created вҶ’ running вҶ’ stopped вҶ’ removed)мқ„ мғҒнғң м „мқҙлЎң м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.

1. Docker м•„нӮӨн…ҚмІҳ вҖ” client, dockerd, containerd

docker runмқ„ н•ң лІҲ м№  л•Ң, лӮҙл¶Җм—җм„ңлҠ” м—¬лҹ¬ м»ҙнҸ¬л„ҢнҠёк°Җ нҳ‘л Ҙн•ңлӢӨ. лҸ„м»ӨлҠ” лӢЁмқј н”„лЎңк·ёлһЁмқҙ м•„лӢҲлқј нҒҙлқјмқҙм–ёнҠё-м„ңлІ„(лҚ°лӘ¬) кө¬мЎ°лӢӨ. м „мІҙ к·ёлҰјмқҖ Docker overviewм—җ м •лҰ¬лҸј мһҲлӢӨ.

к°Ғ м»ҙнҸ¬л„ҢнҠёмқҳ м—ӯн• мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

  • docker CLI (client): мӮ¬мҡ©мһҗк°Җ мһ…л Ҙн•ҳлҠ” лӘ…л №н–ү лҸ„кө¬. dockerdмқҳ REST API(ліҙнҶө /var/run/docker.sock мң лӢүмҠӨ мҶҢмј“)лЎң мҡ”мІӯмқ„ ліҙлӮј лҝҗ, м§Ғм ‘ м»Ён…Ңмқҙл„ҲлҘј л§Ңл“Өм§Җ м•ҠлҠ”лӢӨ.
  • dockerd (Docker лҚ°лӘ¬): мқҙлҜём§ҖВ·л„ӨнҠёмӣҢнҒ¬В·ліјлҘЁВ·л№Ңл“ң л“ұ кі мҲҳмӨҖ кё°лҠҘмқ„ кҙҖлҰ¬н•ҳлҠ” м„ңлІ„. нҒҙлқјмқҙм–ёнҠё мҡ”мІӯмқ„ л°ӣм•„ м»Ён…Ңмқҙл„Ҳ мӢӨн–үмқҖ containerdм—җ мң„мһ„н•ңлӢӨ.
  • containerd: мқҙлҜём§Җ н’Җ, мҠӨнҶ лҰ¬м§Җ, м»Ён…Ңмқҙл„Ҳмқҳ мғқлӘ…мЈјкё°лҘј мұ…мһ„м§ҖлҠ” кі мҲҳмӨҖ лҹ°нғҖмһ„. OCI н‘ңмӨҖмқ„ л”°лҘёлӢӨ. мҝ лІ„л„ӨнӢ°мҠӨлҸ„ лҸ„м»Ө м—Ҷмқҙ containerdлҘј м§Ғм ‘ м“ҙлӢӨ.
  • runc: namespaceВ·cgroupмқ„ м„Өм •н•ҙ мӢӨм ңлЎң м»Ён…Ңмқҙл„Ҳ н”„лЎңм„ёмҠӨлҘј лқ„мҡ°лҠ” м ҖмҲҳмӨҖ лҹ°нғҖмһ„(OCI runtime). м»Ён…Ңмқҙл„ҲлҘј "л§Ңл“ңлҠ”" к°ҖмһҘ л§Ҳм§Җл§ү мҶҗ.

мқҙ кі„мёө кө¬мЎ° л•Ңл¬ём—җ, мҝ лІ„л„ӨнӢ°мҠӨк°Җ "лҸ„м»ӨлҘј deprecateн–ҲлӢӨ"лҠ” л§җмқҙ лҸ„м»Ө мқҙлҜём§ҖлҘј лӘ» м“ҙлӢӨлҠ” лң»мқҙ м•„лӢҲлӢӨ. мҝ лІ„л„ӨнӢ°мҠӨлҠ” dockerd лҢҖмӢ  containerdлҘј м§Ғм ‘ нҳём¶ңн•  лҝҗ, мқҙлҜём§Җ нҸ¬л§·мқҖ к·ёлҢҖлЎңлӢӨ. мқҙ лӮҙмҡ©мқҖ м»Ён…Ңмқҙл„Ҳ лҹ°нғҖмһ„ мӢ¬нҷ”м—җм„ң мһҗм„ёнһҲ лӢӨлЈ¬лӢӨ.

docker CLIлҠ” к·ём Җ нҒҙлқјмқҙм–ёнҠёлӢӨ

docker лӘ…л №мқҖ мӣҗкІ© нҳёмҠӨнҠёмқҳ лҚ°лӘ¬м—җлҸ„ л¶ҷмқ„ мҲҳ мһҲлӢӨ(DOCKER_HOST нҷҳкІҪліҖмҲҳ). мҰү лӮҙ л…ёнҠёл¶Ғмқҳ CLIлЎң мӣҗкІ© м„ңлІ„мқҳ dockerdлҘј мЎ°мў…н•  мҲҳ мһҲлӢӨ. CLIмҷҖ лҚ°лӘ¬мқҙ л¶„лҰ¬лҸј мһҲлӢӨлҠ” мӮ¬мӢӨмқҙ мқҙ мң м—°м„ұмқҳ к·јкұ°лӢӨ.

2. мқҙлҜём§ҖмҷҖ м»Ён…Ңмқҙл„Ҳ вҖ” нҒҙлһҳмҠӨмҷҖ мқёмҠӨн„ҙмҠӨ

лҸ„м»ӨлҘј л°°мҡё л•Ң к°ҖмһҘ лЁјм Җ мһЎм•„м•ј н•  к°ңл…җмқҙ мқҙлҜём§ҖмҷҖ м»Ён…Ңмқҙл„Ҳмқҳ кҙҖкі„лӢӨ.

  • мқҙлҜём§Җ(image): м• н”ҢлҰ¬мјҖмқҙм…ҳкіј к·ё мӢӨн–үм—җ н•„мҡ”н•ң нҢҢмқјВ·лқјмқҙлёҢлҹ¬лҰ¬В·л©”нғҖлҚ°мқҙн„°лҘј л¬¶мқҖ мқҪкё° м „мҡ© н…ңн”ҢлҰҝ. м•һ мһҘм—җм„ң ліё л Ҳмқҙм–ҙл“Өмқҳ л¬¶мқҢмқҙлӢӨ.
  • м»Ён…Ңмқҙл„Ҳ(container): мқҙлҜём§ҖлҘј мӢӨн–үн•ҙ л§Ңл“  мӮҙм•„ мһҲлҠ” мқёмҠӨн„ҙмҠӨ. мқҙлҜём§Җ мң„м—җ м“°кё° к°ҖлҠҘн•ң л Ҳмқҙм–ҙк°Җ н•ң мһҘ м–№нһҢ мӢӨн–ү лӢЁмң„лӢӨ.

к°қмІҙм§Җн–Ҙм—җ л№—лҢҖл©ҙ мқҙлҜём§ҖлҠ” нҒҙлһҳмҠӨ, м»Ён…Ңмқҙл„ҲлҠ” мқёмҠӨн„ҙмҠӨлӢӨ. к°ҷмқҖ мқҙлҜём§Җ(нҒҙлһҳмҠӨ)лЎң м»Ён…Ңмқҙл„Ҳ(мқёмҠӨн„ҙмҠӨ)лҘј м—¬лҹ¬ к°ң л§Ңл“Ө мҲҳ мһҲкі , к°Ғ м»Ён…Ңмқҙл„ҲлҠ” лҸ…лҰҪлҗң м“°кё° л Ҳмқҙм–ҙлҘј к°Җм§„лӢӨ.

м—¬кё°м„ң мӨ‘мҡ”н•ң н•Ём •: м»Ён…Ңмқҙл„Ҳмқҳ м“°кё° л Ҳмқҙм–ҙм—җ м ҖмһҘн•ң лҚ°мқҙн„°лҠ” м»Ён…Ңмқҙл„ҲлҘј м§Җмҡ°л©ҙ мӮ¬лқјм§„лӢӨ. лҚ°мқҙн„°лІ мқҙмҠӨ к°ҷмқҖ мҳҒмҶҚ лҚ°мқҙн„°лҠ” ліјлҘЁмңјлЎң л¶„лҰ¬н•ҙм•ј н•ңлӢӨ. м»Ён…Ңмқҙл„ҲлҠ” "м–ём ңл“  лІ„лҰ¬кі  лӢӨмӢң л§Ңл“Ө мҲҳ мһҲлҠ”(ephemeral)" кІғмңјлЎң лӢӨлӨ„м•ј н•ңлӢӨ.

3. л Ҳм§ҖмҠӨнҠёлҰ¬ вҖ” мқҙлҜём§ҖлҘј м ҖмһҘн•ҳкі  л°°нҸ¬н•ҳлҠ” кіі

мқҙлҜём§ҖлҘј л№Ңл“ңн•ң л’Ө лӢӨлҘё мӮ¬лһҢВ·лӢӨлҘё м„ңлІ„мҷҖ кіөмң н•ҳл Өл©ҙ л Ҳм§ҖмҠӨнҠёлҰ¬(registry)к°Җ н•„мҡ”н•ҳлӢӨ. л Ҳм§ҖмҠӨнҠёлҰ¬лҠ” мқҙлҜём§ҖлҘј м ҖмһҘВ·л°°нҸ¬н•ҳлҠ” м„ңлІ„лӢӨ.

  • Docker Hub: к°ҖмһҘ л„җлҰ¬ м“°мқҙлҠ” кіөк°ң л Ҳм§ҖмҠӨнҠёлҰ¬. nginx, redis к°ҷмқҖ кіөмӢқ мқҙлҜём§ҖлҘј нҳёмҠӨнҢ…н•ңлӢӨ.
  • private registry: мӮ¬лӮҙ мқҙлҜём§ҖлҘј ліҙкҙҖн•ҳлҠ” л№„кіөк°ң л Ҳм§ҖмҠӨнҠёлҰ¬. Harbor, AWS ECR, GCP Artifact Registry, GitHub Container Registry л“ұмқҙ мһҲлӢӨ.

мқҙлҜём§ҖлҠ” л Ҳм§ҖмҠӨнҠёлҰ¬/лҰ¬нҸ¬м§Җн„°лҰ¬:нғңк·ё нҳ•мӢқмңјлЎң мӢқлі„н•ңлӢӨ.

нғңк·ё(:v1.4.2, :latest)лҠ” мӮ¬лһҢмқҙ мқҪкё° мўӢмқҖ лі„м№ӯмқҙкі , мӢӨм ң мқҙлҜём§Җмқҳ л¶ҲліҖ мӢқлі„мһҗлҠ” лӢӨмқҙм ңмҠӨнҠё(digest)лӢӨ.

# нғңк·ёлҠ” мҳ®кІЁк°Ҳ мҲҳ мһҲм§Җл§Ң, лӢӨмқҙм ңмҠӨнҠёлҠ” мҪҳн…җмё  н•ҙмӢңлқј л¶ҲліҖмқҙлӢӨ
nginx:1.27
nginx@sha256:5e2b3f1a...  # н•ӯмғҒ к°ҷмқҖ л°”мқҙнҠёлҘј к°ҖлҰ¬нӮЁлӢӨ

:latestлҘј мҡҙмҳҒм—җ м“°м§Җ л§җ кІғ

:latestлҠ” "мөңмӢ "мқҙлқјлҠ” м•ҪмҶҚмқҙ м•„лӢҲлқј к·ёлғҘ кё°ліё нғңк·ё мқҙлҰ„мқј лҝҗмқҙлӢӨ. к°ҷмқҖ :latestлқјлҸ„ мӢңм җм—җ л”°лқј лӢӨлҘё мқҙлҜём§ҖлҘј к°ҖлҰ¬нӮ¬ мҲҳ мһҲм–ҙ мһ¬нҳ„м„ұмқҙ к№Ём§„лӢӨ. мҡҙмҳҒ л°°нҸ¬м—җлҠ” лӘ…мӢңм Ғ лІ„м „ нғңк·ёлӮҳ лӢӨмқҙм ңмҠӨнҠё кі м •мқ„ м“ҙлӢӨ.

4. н•өмӢ¬ лӘ…л №м–ҙ вҖ” м»Ён…Ңмқҙл„Ҳ лӢӨлЈЁкё°

мқҙм ң мӢӨм ң лӘ…л №м–ҙлЎң м»Ён…Ңмқҙл„ҲлҘј лӢӨлӨ„ліёлӢӨ. м „мІҙ л ҲнҚјлҹ°мҠӨлҠ” docker CLI referenceм—җ мһҲлӢӨ.

# мқҙлҜём§Җ к°Җм ёмҳӨкё°
docker pull nginx:1.27

# м»Ён…Ңмқҙл„Ҳ мӢӨн–ү (л°ұк·ёлқјмҡҙл“ң -d, нҸ¬нҠё л§Өн•‘ -p, мқҙлҰ„ м§Җм • --name)
docker run -d -p 8080:80 --name web nginx:1.27

# мӢӨн–ү мӨ‘мқё м»Ён…Ңмқҙл„Ҳ лӘ©лЎқ (-aлҠ” л©Ҳм¶ҳ кІғк№Ңм§Җ)
docker ps
docker ps -a

# мӢӨн–ү мӨ‘мқё м»Ён…Ңмқҙл„Ҳ м•Ҳм—җм„ң лӘ…л № мӢӨн–ү (л””лІ„к№…мқҳ н•өмӢ¬)
docker exec -it web bash

# лЎңк·ё нҷ•мқё (мӢӨмӢңк°„ м¶”м Ғ -f, л§Ҳм§Җл§ү 100мӨ„ --tail)
docker logs -f --tail 100 web

# м»Ён…Ңмқҙл„Ҳмқҳ лӘЁл“  л©”нғҖлҚ°мқҙн„°лҘј JSONмңјлЎң мЎ°нҡҢ
docker inspect web

# м»Ён…Ңмқҙл„Ҳ мӨ‘м§Җ / мӢңмһ‘ / мһ¬мӢңмһ‘
docker stop web
docker start web
docker restart web

# м»Ён…Ңмқҙл„Ҳ м ңкұ° (-fлҠ” мӢӨн–ү мӨ‘мқҙм–ҙлҸ„ к°•м ң)
docker rm -f web

# мқҙлҜём§Җ л Ҳм§ҖмҠӨнҠёлҰ¬м—җ мҳ¬лҰ¬кё°
docker tag myapp:dev registry.example.com/team/myapp:v1.0
docker push registry.example.com/team/myapp:v1.0

нҠ№нһҲ мӢӨл¬ҙ л””лІ„к№…м—җм„ң мһҗмЈј м“°лҠ” м„ё к°Җм§ҖлҘј кё°м–өн•ҙ л‘”лӢӨ.

  • docker exec -it <м»Ён…Ңмқҙл„Ҳ> sh|bash: мӮҙм•„ мһҲлҠ” м»Ён…Ңмқҙл„Ҳ лӮҙл¶Җм—җ л“Өм–ҙк°Җ мғҒнғңлҘј нҷ•мқён•ңлӢӨ.
  • docker logs -f <м»Ён…Ңмқҙл„Ҳ>: н‘ңмӨҖ м¶ңл Ҙ/м—җлҹ¬лҘј мӢӨмӢңк°„мңјлЎң ліёлӢӨ. м»Ён…Ңмқҙл„ҲлҠ” лЎңк·ёлҘј stdoutмңјлЎң лӮҙліҙлӮҙлҠ” кІғмқҙ мӣҗм№ҷмқҙлӢӨ.
  • docker inspect <м»Ён…Ңмқҙл„Ҳ>: IP, л§ҲмҡҙнҠё, нҷҳкІҪліҖмҲҳ, л„ӨнҠёмӣҢнҒ¬ м„Өм • л“ұ лӘЁл“  мӮ¬мӢӨмқ„ н•ң лІҲм—җ ліёлӢӨ.
runм—җ мһҗмЈј л¶ҷлҠ” мҳөм…ҳ м •лҰ¬
  • -d (detached): л°ұк·ёлқјмҡҙл“ң мӢӨн–ү
  • -it: мқён„°лһҷнӢ°лёҢ н„°лҜёл„җ (м…ё м§„мһ… мӢң)
  • -p host:container: нҸ¬нҠё л§Өн•‘
  • -e KEY=VALUE: нҷҳкІҪліҖмҲҳ мЈјмһ…
  • -v host:container: ліјлҘЁ/л°”мқёл“ң л§ҲмҡҙнҠё
  • --rm: мў…лЈҢ мӢң м»Ён…Ңмқҙл„Ҳ мһҗлҸҷ мӮӯм ң
  • --restart=unless-stopped: мһ¬мӢңмһ‘ м •мұ…
  • --network <name>: л„ӨнҠёмӣҢнҒ¬ м§Җм •

5. м»Ён…Ңмқҙл„Ҳ лқјмқҙн”„мӮ¬мқҙнҒҙ

м»Ён…Ңмқҙл„ҲлҠ” лӘ…нҷ•н•ң мғҒнғң(state)лҘј к°Җм§Җл©°, лӘ…л №м—җ л”°лқј мғҒнғңк°Җ м „мқҙлҗңлӢӨ. мқҙ лӘЁлҚёмқ„ мқҙн•ҙн•ҳл©ҙ docker ps -aм—җ м°ҚнһҲлҠ” мғҒнғңк°Җ н•ңлҲҲм—җ мқҪнһҢлӢӨ.

stateDiagram-v2
    [*] --> created: docker create
    created --> running: docker start / run
    running --> paused: docker pause
    paused --> running: docker unpause
    running --> stopped: docker stop<br>(SIGTERMвҶ’SIGKILL)
    stopped --> running: docker start
    stopped --> removed: docker rm
    running --> removed: docker rm -f
    removed --> [*]
  • created: м»Ён…Ңмқҙл„Ҳк°Җ мғқм„ұлҗҗм§Җл§Ң м•„м§Ғ н”„лЎңм„ёмҠӨк°Җ мӢңмһ‘лҗҳм§Җ м•ҠмқҖ мғҒнғң(docker create).
  • running: л©”мқё н”„лЎңм„ёмҠӨк°Җ лҸҢкі  мһҲлҠ” мғҒнғң. docker runмқҖ create + startлҘј н•ң лІҲм—җ н•ңлӢӨ.
  • paused: cgroup freezerлЎң лӘЁл“  н”„лЎңм„ёмҠӨлҘј мқјмӢң м •м§Җн•ң мғҒнғң.
  • stopped(exited): л©”мқё н”„лЎңм„ёмҠӨк°Җ мў…лЈҢлҗң мғҒнғң. мў…лЈҢ мҪ”л“ң(exit code)к°Җ лӮЁлҠ”лӢӨ.
  • removed: м»Ён…Ңмқҙл„ҲмҷҖ к·ё м“°кё° л Ҳмқҙм–ҙк°Җ мӮӯм ңлҗң мғҒнғң.

docker stopмқҖ м»Ён…Ңмқҙл„Ҳмқҳ PID 1м—җ лЁјм Җ SIGTERMмқ„ ліҙлӮҙ м •мғҒ мў…лЈҢ(graceful shutdown) кё°нҡҢлҘј мЈјкі , мң мҳҲ мӢңк°„(кё°ліё 10мҙҲ) м•Ҳм—җ лҒқлӮҳм§Җ м•Ҡмңјл©ҙ SIGKILLлЎң к°•м ң мў…лЈҢн•ңлӢӨ. мқҙ мӢ нҳё мІҳлҰ¬ л°©мӢқмқҖ мҝ лІ„л„ӨнӢ°мҠӨмқҳ Pod мў…лЈҢ м Ҳм°ЁмҷҖлҸ„ лҳ‘к°ҷмқҙ мқҙм–ҙм§ҖлҜҖлЎң м§ҖкёҲ мқөнҳҖл‘җл©ҙ мўӢлӢӨ.

# мў…лЈҢ мҪ”л“ң нҷ•мқё вҖ” 0мқҙл©ҙ м •мғҒ, 137мқҙл©ҙ SIGKILL(128+9), 143мқҙл©ҙ SIGTERM(128+15)
docker inspect web --format '{{.State.ExitCode}}'

н•өмӢ¬ м •лҰ¬

  • лҸ„м»ӨлҠ” client вҶ’ dockerd вҶ’ containerd вҶ’ runcлЎң мқҙм–ҙм§ҖлҠ” кі„мёө кө¬мЎ°лӢӨ. CLIлҠ” нҒҙлқјмқҙм–ёнҠёмқј лҝҗ мӢӨм ң мӢӨн–үмқҖ н•ҳмң„ лҹ°нғҖмһ„мқҙ н•ңлӢӨ.
  • мқҙлҜём§ҖлҠ” нҒҙлһҳмҠӨ, м»Ён…Ңмқҙл„ҲлҠ” мқёмҠӨн„ҙмҠӨлӢӨ. м»Ён…Ңмқҙл„Ҳмқҳ м“°кё° л Ҳмқҙм–ҙ лҚ°мқҙн„°лҠ” м»Ён…Ңмқҙл„ҲлҘј м§Җмҡ°л©ҙ мӮ¬лқјм§ҖлҜҖлЎң мҳҒмҶҚ лҚ°мқҙн„°лҠ” ліјлҘЁмңјлЎң л¶„лҰ¬н•ңлӢӨ.
  • мқҙлҜём§ҖлҠ” л Ҳм§ҖмҠӨнҠёлҰ¬/лҰ¬нҸ¬м§Җн„°лҰ¬:нғңк·ёлЎң мӢқлі„н•ҳлҗҳ, мҡҙмҳҒм—җлҠ” :latest лҢҖмӢ  лІ„м „ нғңк·ёлӮҳ л¶ҲліҖ лӢӨмқҙм ңмҠӨнҠёлҘј м“ҙлӢӨ.
  • runВ·psВ·execВ·logsВ·inspectВ·rmВ·pullВ·pushк°Җ мқјмғҒ лҸ„кө¬мқҙл©°, лқјмқҙн”„мӮ¬мқҙнҒҙмқҖ createdвҶ’runningвҶ’stoppedвҶ’removed мғҒнғң м „мқҙлЎң мқҙн•ҙн•ңлӢӨ.

лӢӨмқҢ мұ•н„°

мқҙм ң мқҙлҜём§ҖлҘј м§Ғм ‘ л§Ңл“Ө м°ЁлЎҖлӢӨ. Dockerfileкіј мқҙлҜём§Җ л№Ңл“ңм—җм„ң Dockerfile лӘ…л №м–ҙмҷҖ л Ҳмқҙм–ҙ мәҗмӢң, л©ҖнӢ°мҠӨн…Ңмқҙм§Җ л№Ңл“ңлЎң к°ҖліҚкі  м•Ҳм „н•ң мқҙлҜём§ҖлҘј л§Ңл“ңлҠ” лІ•мқ„ лӢӨлЈ¬лӢӨ.
