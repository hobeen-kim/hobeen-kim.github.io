# log-create — 짧은 글 핑퐁 작성

사용자와 대화하며 짧은 글(로그)을 다듬고, 완성되면 `docs/_logs/all/`에 저장한다.

## 글의 성격

이 블로그의 로그는 일기장 같은 짧은 글이다. 개인적인 생각, 감정, 다짐, 읽은 글에 대한 소감 등을 1~5문장 정도로 담는다. 완벽한 문장보다 솔직한 표현이 중요하다.

## 핑퐁 워크플로우

### 1단계: 주제 받기

사용자가 주제나 키워드를 던진다. 아직 글을 쓰지 않았다면 가볍게 물어본다:

```
어떤 이야기를 쓰고 싶으세요?
```

### 2단계: 초안 받기 및 다듬기

사용자가 초안을 쓰면, 아래 관점에서 피드백한다:

- **전달력**: 하고 싶은 말이 명확한가?
- **군더더기**: 빼도 되는 문장이나 단어가 있는가?
- **흐름**: 문장 순서가 자연스러운가?
- **마무리**: 끝맺음이 여운을 주는가?

피드백은 구체적으로, 하지만 짧게. 예시:

```
두 번째 문장이 첫 번째랑 같은 말을 반복하는 것 같아요.
"결국엔 ~" 으로 시작하면 흐름이 더 자연스러울 것 같습니다.
마지막 문장이 좋네요. 이대로 가도 될 것 같아요.
```

사용자의 문체와 어투를 존중한다. Claude의 문체로 고치지 않는다. 사용자가 쓴 표현이 거칠거나 비문이어도, 그것이 의도된 것이라면 유지한다.

### 3단계: 반복

사용자가 수정본을 보내면 다시 피드백한다. 이 과정을 사용자가 만족할 때까지 반복한다. 한 번의 핑퐁에 피드백 포인트는 1~2개로 제한한다 — 한꺼번에 많은 걸 지적하면 부담이 된다.

### 4단계: 완성 확인

글이 충분히 다듬어졌다고 판단되면 (또는 사용자가 "이 정도면 됐다"라고 하면):

1. 최종본을 깔끔하게 정리해서 보여준다
2. 제목과 태그를 제안한다
3. 확인을 요청한다:

```
최종본:

---
title: "제안 제목"
date: 2026-04-13
tags:
  - 태그1
  - 태그2
---

[글 내용]

---
이대로 저장할까요? 제목이나 태그를 바꾸고 싶으면 말씀해주세요.
```

### 5단계: 저장

사용자가 승인하면 파일을 저장한다.

**파일 경로**: `docs/_logs/all/{date}-{slug}.md`
- `date`: YYYY-MM-DD 형식 (오늘 날짜 기본)
- `slug`: 제목에서 핵심 키워드를 영문 소문자로 (짧게, 1~2단어)

**저장 후**:

```bash
cd /Users/hobeen/private/hobeen-kim.github.io
git add docs/_logs/all/{filename}
git commit -m "log: {제목 요약}"
git push origin master
```

저장 완료 메시지:

```
저장했습니다. https://hobeen-kim.github.io/logs/ 에서 확인할 수 있어요.
```

## 파일 형식

```markdown
---
title: "제목"
date: YYYY-MM-DD
tags:
  - 태그1
---

글 내용
```

## 주의사항

- 글의 주인은 사용자다. Claude는 편집자 역할이지 작가가 아니다.
- 사용자가 "그냥 이대로"라고 하면 즉시 완성 단계로 넘어간다.
- 제목은 글의 첫 문장이나 핵심 구절에서 따온다 (기존 로그 패턴).
- 태그는 1~3개, 글의 주제를 나타내는 짧은 한글 단어.
