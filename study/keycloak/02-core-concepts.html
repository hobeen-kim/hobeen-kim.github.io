н•өмӢ¬ к°ңл…җкіј мҡ©м–ҙ

н•ҷмҠө лӘ©н‘ң

  • Keycloakмқҳ н•өмӢ¬ к°қмІҙ(Realm, User, Client, Role, Group)лҘј м •мқҳн•ҳкі  м„ңлЎңмқҳ кҙҖкі„лҘј мқҙн•ҙн•ңлӢӨ.
  • нҶ нҒ°В·м„ём…ҳВ·н”ҢлЎңмҡ°В·IdPВ·SPI л“ұ мҡҙмҳҒ мӨ‘ мһҗмЈј л§ҲмЈјм№ҳлҠ” мҡ©м–ҙлҘј кө¬л¶„н•  мҲҳ мһҲлӢӨ.
  • к°Ғ к°ңл…җмқҙ м–ҙлҠҗ мӢӨл¬ҙ мұ•н„°м—җм„ң к№ҠкІҢ лӢӨлӨ„м§ҖлҠ”м§Җ м—°кІ° м§Җмқ„ мҲҳ мһҲлӢӨ.

мқҙ мұ•н„°лҠ” к°ңл…җ мӮ¬м „мқҙлӢӨ. ліёкІ©м Ғмқё м„Өм •В·мӢӨмҠө лӮҙмҡ©мқҖ CH3 мқҙнӣ„м—җм„ң лӢӨлЈЁл©°, м—¬кё°м„ңлҠ” "мҡ©м–ҙк°Җ нҠҖм–ҙлӮҳмҷ”мқ„ л•Ң л°”лЎң м°ҫм•„ліј мҲҳ мһҲлҠ” н•ң мһҘм§ңлҰ¬ л ҲнҚјлҹ°мҠӨ"лҘј лӘ©н‘ңлЎң н•ңлӢӨ.

2. кІ©лҰ¬ лӢЁмң„

Realm

н•ң Keycloak м„ңлІ„ м•Ҳмқҳ л…јлҰ¬м Ғ "лҸ…лҰҪ мҷ•көӯ"мқҙлӢӨ. н•ң Realmмқҳ User/Client/Role/Key/FlowлҠ” лӢӨлҘё Realmм—җм„ң м „нҳҖ ліҙмқҙм§Җ м•ҠлҠ”лӢӨ. л¬јлҰ¬м ҒмңјлЎңлҠ” к°ҷмқҖ DB н…Ңмқҙлё”мқҙм§Җл§Ң realm_idлЎң мҷ„лІҪнһҲ л¶„лҰ¬лҗңлӢӨ.

  • нҶ нҒ°мқҳ iss нҒҙл Ҳмһ„мқҖ https://auth.example.com/realms/{realm} нҳ•нғңлЎң Realm мқҙлҰ„мқ„ нҸ¬н•Ён•ңлӢӨ.
  • Realmл§ҲлӢӨ мһҗмІҙ м„ңлӘ… нӮӨ м…Ӣмқ„ к°Җм§ҖлҜҖлЎң, н•ң Realmмқҳ нҶ нҒ°мқ„ лӢӨлҘё Realmмқҙ кІҖмҰқн•  мҲҳ м—ҶлӢӨ.
  • л©ҖнӢ°н…Ңл„ҢмӢң м„Өкі„мқҳ кё°ліё м„ нғқм§ҖлӢӨ.

мғҒм„ё: CH3. Realmкіј Organizations

Master Realm

м„Өм№ҳ м§Ғнӣ„ мһҗлҸҷ мғқм„ұлҗҳлҠ” нҠ№мҲҳ Realm. Keycloak мһҗмІҙлҘј кҙҖлҰ¬н•ҳлҠ” мҡҙмҳҒмһҗ кі„м •мқҙ мӮ¬лҠ” кіімқҙлӢӨ. мқҙкіімқҳ кҙҖлҰ¬мһҗлҠ” лӢӨлҘё лӘЁл“  Realmмқ„ кұҙл“ңлҰҙ мҲҳ мһҲмңјлҜҖлЎң м„ңл№„мҠӨ мӮ¬мҡ©мһҗлҘј м ҲлҢҖ л‘җм§Җ м•ҠлҠ”лӢӨ.

Organization (v26+)

Realm м•Ҳм—җм„ң B2B кі к°қмӮ¬ лӢЁмң„лҘј н‘ңнҳ„н•ҳлҠ” кё°лҠҘ. мқҙл©”мқј лҸ„л©”мқё(@acme.com)мқ„ нҠ№м • мЎ°м§Ғкіј л°”мқёл”©н•ҳкі , мЎ°м§Ғлі„ IdPлҘј мһҗлҸҷ лқјмҡ°нҢ…н•ңлӢӨ. "Realm per tenantлҠ” кіјн•ҳм§Җл§Ң GroupмңјлЎңлҠ” л¶ҖмЎұн•ң" мӨ‘к°„ мёөмқ„ л©”мҡҙлӢӨ.

мғҒм„ё: CH3. Realmкіј Organizations

3. мЈјмІҙ(Principal)

User

мӮ¬лһҢ мӮ¬мҡ©мһҗ кі„м •мқҙлӢӨ. Realm м•Ҳм—җ м ҖмһҘлҗҳл©°, л№„л°ҖлІҲнҳё н•ҙмӢңВ·OTPВ·WebAuthn мһҗкІ©мҰқлӘ…кіј мҶҚм„ұ(email, name, мӮ¬мҡ©мһҗ м •мқҳ attribute)мқ„ к°Җм§„лӢӨ. к°ҷмқҖ ID(alice)лқјлҸ„ Realmмқҙ лӢӨлҘҙл©ҙ лӢӨлҘё мӮ¬мҡ©мһҗлӢӨ.

  • Credential: л№„л°ҖлІҲнҳёВ·OTPВ·WebAuthnВ·Recovery Code к°ҷмқҖ мһҗкІ©мҰқлӘ… л ҲмҪ”л“ң
  • Attribute: м»ӨмҠӨн…Җ н•„л“ң(м§ҒмӣҗлІҲнҳё, л¶Җм„ң л“ұ)лЎң нҶ нҒ° нҒҙл Ҳмһ„м—җ л§Өн•‘ к°ҖлҠҘ

мғҒм„ё: CH5. мӮ¬мҡ©мһҗмҷҖ мһҗкІ© мҰқлӘ…

Client

нҶ нҒ°мқ„ л°ңкёүл°ӣмңјл ӨлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳ л ҲмҪ”л“ңлӢӨ. мӣ№ SPAВ·лӘЁл°”мқјВ·л°ұм—”л“ң APIВ·м„ңлІ„к°„ нҳём¶ң мЈјмІҙ л“ұмқҙ к°Ғк°Ғ н•ҳлӮҳмқҳ ClientлЎң л“ұлЎқлҗңлӢӨ. OAuthмқҳ ClientмҷҖ к°ҷмқҖ к°ңл…җмқҙлӢӨ.

  • Client ID/Secret: ClientлҘј мӢқлі„В·мқёмҰқн•ҳлҠ” к°’
  • Redirect URI: Authorization CodeлҘј лҸҢл Өл°ӣмқ„ мҪңл°ұ URL нҷ”мқҙнҠёлҰ¬мҠӨнҠё
  • Access Type(Confidential/Public/Bearer-only): л№„л°ҖнӮӨ ліҙмң  м—¬л¶ҖмҷҖ н”ҢлЎңмҡ° м ңм•Ҫ
  • Client Authenticator: Secret, Signed JWT, mTLS л“ұ Client мқёмҰқ л°©мӢқ

мғҒм„ё: CH4. ClientмҷҖ Service Account

Service Account

Client мһҗмІҙк°Җ мЈјмІҙк°Җ лҗҳлҠ” "кё°кі„ кі„м •"мқҙлӢӨ. мӮ¬лһҢ мӮ¬мҡ©мһҗ м—Ҷмқҙ Client Credentials GrantлЎң Client ID/Secretл§Ң к°Җм§Җкі  нҶ нҒ°мқ„ л°ӣлҠ”лӢӨ. м„ңлІ„к°„ нҳём¶ң, нҒ¬лЎ мһЎ, CI нҢҢмқҙн”„лқјмқё л“ұм—җм„ң м“ҙлӢӨ. кё°мҲ м ҒмңјлЎңлҠ” н•ҙлӢ№ Clientм—җ мһҗлҸҷ м—°кІ°лҗң нҠ№мҲҳ User л ҲмҪ”л“ңлӢӨ.

мғҒм„ё: CH4. ClientмҷҖ Service Account

мғҒм„ё: CH6. RoleВ·Groupкіј Composite Role

Group

мӮ¬мҡ©мһҗ л¬¶мқҢ. нҠёлҰ¬ кі„мёө(/engineering/backend) кө¬мЎ°мқҙл©°, Groupм—җ н• лӢ№лҗң RoleмқҖ мҶҢмҶҚ User лӘЁл‘җк°Җ мһҗлҸҷмңјлЎң мғҒмҶҚл°ӣлҠ”лӢӨ. "л¶Җм„ңВ·нҢҖ лӢЁмң„ к¶Ңн•ң мқјкҙ„ л¶Җм—¬"к°Җ м „нҳ•м Ғ мҡ©лҸ„лӢӨ.

Scope (мҡ”мІӯ мҠӨмҪ”н”„)

OAuthмқҳ scope нҢҢлқјлҜён„°м—җ лӢҙкё°лҠ” к°’. Clientк°Җ "мқҙлІҲ мҡ”мІӯм—җм„ң м–ҙл–Ө нҒҙл Ҳмһ„В·к¶Ңн•ңмқ„ мӣҗн•ҳлҠ”к°Җ"лҘј м„ м–ён•ңлӢӨ. Keycloakм—җм„ңлҠ” Client Scope к°қмІҙмҷҖ лҢҖмқ‘н•ңлӢӨ.

5. нҶ нҒ°мқ„ мЎ°лҰҪн•ҳлҠ” мҡ”мҶҢ

Client Scope

"мқҙ Scopeк°Җ мҡ”мІӯлҗҳл©ҙ нҶ нҒ°м—җ мқҙлҹ° м •ліҙлҘј л„Јм–ҙлқј"лҠ” л¬¶мқҢмқҙлӢӨ. Default ScopeлҠ” н•ӯмғҒ нҸ¬н•Ёлҗҳкі , Optional ScopeлҠ” мҡ”мІӯ мӢңм—җл§Ң нҸ¬н•ЁлҗңлӢӨ.

  • н‘ңмӨҖ мҠӨмҪ”н”„: openid, profile, email, offline_access л“ұ
  • м»ӨмҠӨн…Җ мҠӨмҪ”н”„: order:read, admin-panel л“ұ н”„лЎңм қнҠё м „мҡ©

Protocol Mapper

Client Scope м•Ҳм—җ л“Өм–ҙк°ҖлҠ” "ліҖнҷҳ к·ңм№ҷ"мқҙлӢӨ. "Userмқҳ department attributeлҘј нҶ нҒ°мқҳ dept нҒҙл Ҳмһ„мңјлЎң л„ЈлҠ”лӢӨ" к°ҷмқҖ к·ңм№ҷмқҙ Protocol Mapper н•ҳлӮҳлӢӨ. Hardcoded Claim, User Attribute, Group Membership, Audience л“ұ м—¬лҹ¬ мң нҳ•мқҙ мһҲлӢӨ.

мғҒм„ё: CH7. Client ScopeмҷҖ Protocol Mapper

Consent

мӮ¬мҡ©мһҗм—җкІҢ OAuth н‘ңмӨҖ лҸҷмқҳ нҷ”л©ҙ(мҳҲ: "мқҙ м•ұмқҙ лӢ№мӢ мқҳ мқҙл©”мқјВ·н”„лЎңн•„ м •ліҙлҘј л°ӣмңјл Өкі  н•©лӢҲлӢӨ. лҸҷмқҳн•ҳмӢңкІ мҠөлӢҲк№Ң?")мқ„ лқ„мҡ°лҠ” кё°лҠҘ. Clientлі„лЎң лҒ„кі  мјӨ мҲҳ мһҲмңјл©°, OAuth мӣҗм№ҷ(к¶Ңн•ң мң„мһ„ мӢң мӮ¬мҡ©мһҗ лҸҷмқҳ)мқ„ кө¬нҳ„н•ңлӢӨ.
