н•ҷмҠө лӘ©н‘ң

  • OAuthмқҳ 4к°Җм§Җ м—ӯн• (Resource Owner, Client, Authorization Server, Resource Server)кіј к°Ғмһҗмқҳ мұ…мһ„мқ„ м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.
  • мқёк°Җ м„ңлІ„мқҳ л‘җ н•өмӢ¬ м—”л“ңнҸ¬мқёнҠё(/authorize, /token)мқҳ мҡ©лҸ„лҘј кө¬л¶„н•ңлӢӨ.
  • Public ClientмҷҖ Confidential Clientмқҳ м°Ёмқҙ, мҷң NativeВ·SPAк°Җ PublicмңјлЎң л¶„лҘҳлҗҳлҠ”м§Җ м•ҲлӢӨ.
  • Client IDмҷҖ Client Secretмқҳ мқҳлҜёмҷҖ ліҙкҙҖ мң„м№ҳмқҳ мӣҗм№ҷмқ„ мқҙн•ҙн•ңлӢӨ.

1. Resource Owner вҖ” мһҗмӣҗмқҳ мЈјмқё

Resource OwnerлҠ” мһҗмӣҗмқҳ мЈјмқёмқҙлӢӨ. OAuthк°Җ "лҲ„кө¬мқҳ к¶Ңн•ңмқ„ мң„мһ„н•ҳлҠ”к°Җ"лқјкі  л¬јмңјл©ҙ, л°”лЎң мқҙ Resource Ownerмқҳ к¶Ңн•ңмқҙлӢӨ. лҢҖл¶Җл¶„мқҳ кІҪмҡ° мӮ¬мҡ©мһҗ(End User)к°Җ кі§ Resource OwnerлӢӨ.

м •мқҳмҷҖ мұ…мһ„

Resource Owner вҖ” ліҙнҳёлҗң мһҗмӣҗм—җ лҢҖн•ң м ‘к·јмқ„ мҠ№мқён•  мҲҳ мһҲлҠ” м—”нӢ°нӢ°. мӮ¬лһҢмқё кІҪмҡ° End Userлқјкі лҸ„ н•ңлӢӨ. (RFC 6749 В§1.1)

мұ…мһ„мқҖ лӢЁмҲңн•ҳлӢӨ.

  • ASм—җм„ң мһҗмӢ мқҳ мӢ мӣҗмқ„ мқёмҰқн•ңлӢӨ(лЎңк·ёмқё).
  • Consent нҷ”л©ҙм—җм„ң нҒҙлқјмқҙм–ёнҠёмқҳ м ‘к·ј мҡ”мІӯмқ„ мҠ№мқён•ҳкұ°лӮҳ кұ°л¶Җн•ңлӢӨ.
  • м–ём ңл“  л°ңкёүлҗң нҶ нҒ°мқ„ м·ЁмҶҢн•  мҲҳ мһҲлӢӨ.

Resource Ownerк°Җ мӮ¬лһҢмқҙ м•„лӢҢ кІҪмҡ°

л°ҳл“ңмӢң мӮ¬лһҢмқј н•„мҡ”лҠ” м—ҶлӢӨ. M2M(Machine-to-Machine) мӢңлӮҳлҰ¬мҳӨм—җм„ңлҠ” мЎ°м§Ғ(organization) лҳҗлҠ” м„ңл№„мҠӨ кі„м • мһҗмІҙк°Җ Resource Ownerк°Җ лҗңлӢӨ. мқҙ кІҪмҡ° Client Credentials Grant(CH7)к°Җ м“°мқҙкі , мӮ¬мҡ©мһҗ к°ңмһ… м—Ҷмқҙ нҒҙлқјмқҙм–ёнҠёк°Җ ASм—җм„ң м§Ғм ‘ нҶ нҒ°мқ„ л°ӣлҠ”лӢӨ.

мң нҳ•Resource OwnerмҳҲмӢң
мӮ¬мҡ©мһҗ мң„мһ„к°ңмқё мӮ¬мҡ©мһҗлӮҙ Gmailм—җ м ‘к·јн•ҳлҠ” Notion
M2MмЎ°м§ҒВ·м„ңл№„мҠӨ кі„м •л°°м№ҳ м„ңлІ„к°Җ API нҳём¶ң

Resource Owner vs End User

RFCлҠ” л‘җ мҡ©м–ҙлҘј лҜёл¬ҳн•ҳкІҢ кө¬л¶„н•ңлӢӨ. лҢҖл¶Җл¶„ л¬ёл§Ҙм—җм„ңлҠ” нҳјмҡ©н•ҙлҸ„ лҗҳм§Җл§Ң, M2M мӢңлӮҳлҰ¬мҳӨлҘј лӢӨлЈ° л•ҢлҠ” Resource Owner = мЎ°м§Ғ, End User = мӢӨмӮ¬мҡ©мһҗмІҳлҹј лӢ¬лқјм§Ҳ мҲҳ мһҲлӢӨлҠ” м җмқ„ мқҳмӢқн•ҙ л‘”лӢӨ.

2. Client вҖ” м ‘к·јн•ҳл ӨлҠ” м•ұ

ClientлҠ” мһҗмӣҗм—җ м ‘к·јн•ҳл ӨлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙлӢӨ. мӣ№ л°ұм—”л“ң, лӘЁл°”мқј м•ұ, SPA, CLI лҸ„кө¬, м„ңлІ„ лҚ°лӘ¬к№Ңм§Җ лӘЁл‘җ нҒҙлқјмқҙм–ёнҠём—җ н•ҙлӢ№н•ңлӢӨ. мӮ¬мҡ©мһҗлҘј лҢҖмӢ н•ҙм„ң мһҗмӣҗ м„ңлІ„лҘј нҳём¶ңн•ҳлҠ” мЈјмІҙлӢӨ.

м •мқҳмҷҖ мұ…мһ„

Client вҖ” Resource Ownerмқҳ мҠ№мқёмқ„ л°ӣм•„ ліҙнҳёлҗң мһҗмӣҗм—җ м ‘к·ј мҡ”мІӯмқ„ н•ҳлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳ. (RFC 6749 В§1.1)

мұ…мһ„мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

  • ASм—җ мһҗмӢ мқ„ л“ұлЎқн•ҳкі  Client IDлҘј л°ӣлҠ”лӢӨ.
  • мӮ¬мҡ©мһҗлҘј ASлЎң лҰ¬лӢӨмқҙл үнҠён•ҙ мқёмҰқВ·лҸҷмқҳлҘј л°ӣлҠ”лӢӨ.
  • л°ӣм•„лӮё нҶ нҒ°мқ„ м•Ҳм „н•ҳкІҢ ліҙкҙҖн•ңлӢӨ.
  • мһҗмӣҗ м„ңлІ„ нҳём¶ң мӢң Authorization: Bearer <token> н—ӨлҚ”лҘј мІЁл¶Җн•ңлӢӨ.
  • нҶ нҒ°мқҙ л§ҢлЈҢлҗҳл©ҙ Refresh TokenмңјлЎң к°ұмӢ н•ҳкұ°лӮҳ мӮ¬мҡ©мһҗлҘј лӢӨмӢң мқёмҰқмӢңнӮЁлӢӨ.

3. Authorization Server (AS) вҖ” к¶Ңн•ңмқ„ мҠ№мқён•ҳкі  нҶ нҒ°мқ„ л°ңкёү

ASлҠ” OAuth мғқнғңкі„мқҳ мӢ¬мһҘмқҙлӢӨ. мӮ¬мҡ©мһҗлҘј мқёмҰқн•ҳкі , ConsentлҘј л°ӣкі , нҶ нҒ°мқ„ л°ңкёүн•ңлӢӨ. мӢӨл¬ҙм—җм„ңлҠ” GoogleВ·Kakao к°ҷмқҖ IdPмқҳ ASлҘј м“°кұ°лӮҳ KeycloakВ·Auth0В·Okta к°ҷмқҖ мҶ”лЈЁм…ҳмқ„ мһҗмІҙ кө¬м¶•н•ңлӢӨ.

м •мқҳмҷҖ мұ…мһ„

Authorization Server вҖ” мӮ¬мҡ©мһҗ мқёмҰқкіј к¶Ңн•ң мҠ№мқё нӣ„ нҒҙлқјмқҙм–ёнҠём—җкІҢ Access Tokenмқ„ л°ңкёүн•ҳлҠ” м„ңлІ„. (RFC 6749 В§1.1)

мұ…мһ„мқҖ л„“лӢӨ.

  • мӮ¬мҡ©мһҗ мқёмҰқ (л№„л°ҖлІҲнҳёВ·MFAВ·мғқмІҙ л“ұ)
  • нҒҙлқјмқҙм–ёнҠё л“ұлЎқ кҙҖлҰ¬ (Client IDВ·SecretВ·redirect_uri)
  • Consent нҷ”л©ҙ м ңкіө л°Ҹ мҠ№мқё кё°лЎқ ліҙмЎҙ
  • Authorization Code л°ңкёү (Authorization Code Flow)
  • Access TokenВ·Refresh Token л°ңкёү л°Ҹ м„ңлӘ…
  • нҶ нҒ° кІҖмҰқ м—”л“ңнҸ¬мқёнҠё м ңкіө (Introspection)
  • нҶ нҒ° м·ЁмҶҢ м—”л“ңнҸ¬мқёнҠё м ңкіө (Revocation)
  • м„ём…ҳВ·лЎңк·ём•„мӣғ кҙҖлҰ¬

4. Resource Server (RS) вҖ” ліҙнҳё мһҗмӣҗмқ„ к°Җм§„ API м„ңлІ„

Resource ServerлҠ” мӢӨм ң мһҗмӣҗмқ„ к°Җм§Җкі  мһҲкі , нҶ нҒ°мқ„ кІҖмҰқн•ҙм„ң мһҗмӣҗмқ„ лӮҙмЈјлҠ” м„ңлІ„лӢӨ. API кІҢмқҙнҠёмӣЁмқҙ л’Өмқҳ л§ҲмқҙнҒ¬лЎңм„ңл№„мҠӨл“Ө, Google Drive API, Gmail APIмІҳлҹј мҷёл¶Җм—җ л…ём¶ңлҗң ліҙнҳё мһҗмӣҗ м„ңлІ„к°Җ лӘЁл‘җ мқҙм—җ н•ҙлӢ№н•ңлӢӨ.

м •мқҳмҷҖ мұ…мһ„

Resource Server вҖ” ліҙнҳёлҗң мһҗмӣҗмқ„ нҳёмҠӨнҠён•ҳлҠ” м„ңлІ„. Access Tokenмқ„ мӮ¬мҡ©н•ҙ мқёмҰқлҗң мҡ”мІӯмқ„ л°ӣлҠ”лӢӨ. (RFC 6749 В§1.1)

мұ…мһ„мқҖ мғқк°ҒліҙлӢӨ лӢЁмҲңн•ҳлӢӨ.

  • мҡ”мІӯмқҳ Authorization: Bearer <token> н—ӨлҚ”лҘј нҷ•мқён•ңлӢӨ.
  • нҶ нҒ°мқ„ кІҖмҰқн•ңлӢӨ (м„ңлӘ… кІҖмҰқ лҳҗлҠ” Introspection).
  • нҶ нҒ°мқҳ Scopeк°Җ мҡ”мІӯлҗң лҸҷмһ‘м—җ м¶©л¶„н•ңм§Җ нҷ•мқён•ңлӢӨ.
  • н—Ҳмҡ©лҗҳл©ҙ мһҗмӣҗмқ„ л°ҳнҷҳ, кұ°л¶Җлҗҳл©ҙ 401/403мқ„ л°ҳнҷҳн•ңлӢӨ.

RSлҠ” мӮ¬мҡ©мһҗлҘј мқёмҰқн•ҳм§Җ м•ҠлҠ”лӢӨ

RSлҠ” нҶ нҒ°л§Ң ліёлӢӨ. мӮ¬мҡ©мһҗк°Җ лҲ„кө°м§Җ м§Ғм ‘ мқёмҰқн•ҳм§Җ м•Ҡкі , ASк°Җ м„ңлӘ…н•ҙ мӨҖ нҶ нҒ°мқҳ sub н•„л“ңлҘј лҜҝлҠ”лӢӨ. мқҙ мӢ лў° кҙҖкі„к°Җ OAuthмқҳ мһ‘лҸҷ м „м ңлӢӨ.

6. Public vs Confidential Client

OAuthк°Җ нҒҙлқјмқҙм–ёнҠёлҘј л‘җ мң нҳ•мңјлЎң лӮҳлҲ„лҠ” мқҙмң лҠ” "Client Secretмқ„ м•Ҳм „н•ҳкІҢ ліҙкҙҖн•  мҲҳ мһҲлҠ”к°Җ"лқјлҠ” лӢЁмҲңн•ң м§Ҳл¬ё л•Ңл¬ёмқҙлӢӨ.

м •мқҳ

Confidential Client вҖ” Client Secretмқ„ м•Ҳм „н•ҳкІҢ ліҙкҙҖн•  мҲҳ мһҲлҠ” нҒҙлқјмқҙм–ёнҠё. (RFC 6749 В§2.1) Public Client вҖ” Client Secretмқ„ м•Ҳм „н•ҳкІҢ ліҙкҙҖн•  мҲҳ м—ҶлҠ” нҒҙлқјмқҙм–ёнҠё.
