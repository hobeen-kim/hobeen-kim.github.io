ConfigMapкіј Secret

н•ҷмҠө лӘ©н‘ң

  • ConfigMapмқ„ л§Ңл“Өкі  envВ·envFromВ·ліјлҘЁ л§ҲмҡҙнҠёлЎң мЈјмһ…н•ҳлҠ” м„ё к°Җм§Җ л°©мӢқмқ„ мқөнһҢлӢӨ.
  • Secretмқҳ мў…лҘҳмҷҖ base64 мқёмҪ”л”©мқҙ м•”нҳёнҷ”к°Җ м•„лӢҳмқ„, к·ёлҰ¬кі  мӢӨм§Ҳм Ғ ліҙм•Ҳ н•ңкі„лҘј мқҙн•ҙн•ңлӢӨ.
  • нҷҳкІҪліҖмҲҳ мЈјмһ…кіј ліјлҘЁ л§ҲмҡҙнҠёмқҳ м°ЁмқҙмҷҖ м„ нғқ кё°мӨҖмқ„ м•ҲлӢӨ.
  • ConfigMap/Secret ліҖкІҪмқҙ м–ём ң м–ҙл–»кІҢ Podм—җ л°ҳмҳҒлҗҳлҠ”м§Җ, immutable м„Өм •мқҳ нҡЁкіјлҘј лӢӨлЈ¬лӢӨ.

1. ConfigMapмқҙлһҖ

ConfigMapмқҖ л№„л°Җмқҙ м•„лӢҢ м„Өм • лҚ°мқҙн„°лҘј нӮӨ-к°’ мҢҚмңјлЎң м ҖмһҘн•ҳлҠ” мҳӨлёҢм қнҠёлӢӨ. м„Өм •мқ„ м»Ён…Ңмқҙл„Ҳ мқҙлҜём§Җм—җм„ң л¶„лҰ¬н•ҙ, к°ҷмқҖ мқҙлҜём§ҖлҘј нҷҳкІҪ(dev/stage/prod)лі„лЎң лӢӨлҘё м„Өм •кіј н•Ёк»ҳ л°°нҸ¬н•  мҲҳ мһҲкІҢ н•ңлӢӨ. мқҙкІғмқҙ Twelve-Factor Appмқҳ "м„Өм •мқ„ нҷҳкІҪм—җ м ҖмһҘн•ҳлқј" мӣҗм№ҷмқ„ мҝ лІ„л„ӨнӢ°мҠӨм—җм„ң кө¬нҳ„н•ҳлҠ” л°©лІ•мқҙлӢӨ.

ConfigMapмқҖ мһ‘мқҖ м„Өм •к°’(URL, нҸ¬нҠё, н”Ңлһҳк·ё)л¶Җн„° нҶөм§ё м„Өм • нҢҢмқјк№Ңм§Җ лӢҙмқ„ мҲҳ мһҲлӢӨ. лӢЁ, н•ҳлӮҳмқҳ ConfigMapмқҖ 1MiBлҘј л„ҳмқ„ мҲҳ м—ҶлӢӨ.

apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  LOG_LEVEL: "info"
  MAX_CONNECTIONS: "100"
  app.properties: |
    server.port=8080
    feature.x.enabled=true

лӘ…л №нҳ•мңјлЎңлҸ„ л§Ңл“Ө мҲҳ мһҲлӢӨ.

kubectl create configmap app-config \
  --from-literal=LOG_LEVEL=info \
  --from-file=app.properties

kubectl get configmap app-config -o yaml

dataлҠ” UTF-8 н…ҚмҠӨнҠёлҘј, л°”мқҙл„ҲлҰ¬лҠ” binaryData(base64)лҘј м“ҙлӢӨ. м „л°ҳм Ғмқё к°ңл…җмқҖ Configuration л¬ём„ңмҷҖ ConfigMap л¬ём„ңм—җ м •лҰ¬лҸј мһҲлӢӨ.

2. ConfigMap мЈјмһ… вҖ” env, envFrom, ліјлҘЁ л§ҲмҡҙнҠё

ConfigMapмқ„ м»Ён…Ңмқҙл„Ҳм—җ л„ЈлҠ” л°©лІ•мқҖ м„ё к°Җм§ҖлӢӨ.

л°©лІ• 1: envлЎң к°ңлі„ нӮӨ мЈјмһ…. нҠ№м • нӮӨ н•ҳлӮҳлҘј нҷҳкІҪліҖмҲҳлЎң л§Өн•‘н•ңлӢӨ.

spec:
  containers:
  - name: app
    image: myapp:1.0
    env:
    - name: LOG_LEVEL
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: LOG_LEVEL

л°©лІ• 2: envFromмңјлЎң м „мІҙ мЈјмһ…. ConfigMapмқҳ лӘЁл“  нӮӨлҘј н•ңкәјлІҲм—җ нҷҳкІҪліҖмҲҳлЎң л§Ңл“ лӢӨ.

spec:
  containers:
  - name: app
    image: myapp:1.0
    envFrom:
    - configMapRef:
        name: app-config
      prefix: APP_      # м„ нғқ: лӘЁл“  нӮӨ м•һм—җ м ‘л‘җмӮ¬

л°©лІ• 3: ліјлҘЁмңјлЎң л§ҲмҡҙнҠё. ConfigMapмқ„ л””л үн„°лҰ¬лЎң л§ҲмҡҙнҠён•ҳл©ҙ к°Ғ нӮӨк°Җ нҢҢмқјлЎң лӮҳнғҖлӮңлӢӨ.

spec:
  containers:
  - name: app
    image: myapp:1.0
    volumeMounts:
    - name: config-vol
      mountPath: /etc/config
      readOnly: true
  volumes:
  - name: config-vol
    configMap:
      name: app-config
      items:                    # м„ нғқ: нҠ№м • нӮӨл§Ң, лӢӨлҘё мқҙлҰ„мңјлЎң
      - key: app.properties
        path: application.properties

мқҙлҹ¬л©ҙ /etc/config/app.properties к°ҷмқҖ нҢҢмқјмқҙ л§Ңл“Өм–ҙм§„лӢӨ. м„Өм • нҢҢмқјмқ„ нҶөм§ёлЎң л„ҳкёё л•Ң мһҗм—°мҠӨлҹ¬мҡҙ л°©мӢқмқҙлӢӨ.

    cm["ConfigMap app-config"]
    cm -- "configMapKeyRef" --> e1["нҷҳкІҪліҖмҲҳ<br>LOG_LEVEL"]
    cm -- "configMapRef (envFrom)" --> e2["лӘЁл“  нӮӨ<br>вҶ’ нҷҳкІҪліҖмҲҳ"]
    cm -- "configMap volume" --> f["/etc/config/*<br>нҢҢмқјл“Ө"]
    e1 & e2 & f --> pod["м»Ён…Ңмқҙл„Ҳ"]

3. Secretмқҳ мў…лҘҳмҷҖ base64мқҳ н•ңкі„

SecretмқҖ л№„л°ҖлІҲнҳё, нҶ нҒ°, нӮӨ, мқёмҰқм„ң к°ҷмқҖ лҜјк°җ лҚ°мқҙн„°лҘј лӢҙлҠ” мҳӨлёҢм қнҠёлӢӨ. кө¬мЎ°лҠ” ConfigMapкіј л№„мҠ·н•ҳм§Җл§Ң лӘҮ к°Җм§Җ м°Ёмқҙк°Җ мһҲлӢӨ. к°’мқҖ base64лЎң мқёмҪ”л”©лҗҳкі , л…ёл“ңм—җм„ңлҠ” к°ҖлҠҘн•ҳл©ҙ л©”лӘЁлҰ¬(tmpfs)м—җ м ҖмһҘлҗҳл©°, лі„лҸ„мқҳ RBACлЎң м ‘к·јмқ„ лҚ” м—„кІ©нһҲ нҶөм ңн•  мҲҳ мһҲлӢӨ.

Secretм—җлҠ” typeмқҙ мһҲкі , нғҖмһ…л§ҲлӢӨ кё°лҢҖн•ҳлҠ” нӮӨ кө¬мЎ°к°Җ м •н•ҙм ё мһҲлӢӨ.

typeмҡ©лҸ„
Opaque(кё°ліё) мһ„мқҳмқҳ мӮ¬мҡ©мһҗ м •мқҳ лҚ°мқҙн„°
kubernetes.io/dockerconfigjsonн”„лқјмқҙл№— л Ҳм§ҖмҠӨнҠёлҰ¬ мқёмҰқ(imagePullSecrets)
kubernetes.io/tlsTLS мқёмҰқм„ңВ·нӮӨ(tls.crt, tls.key)
kubernetes.io/basic-authusername/password
kubernetes.io/service-account-tokenServiceAccount нҶ нҒ°
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque
data:
  username: YWRtaW4=          # "admin"мқҳ base64
  password: czNjcjN0UHdk      # "s3cr3tPwd"мқҳ base64

stringDataлҘј м“°л©ҙ нҸүл¬ёмңјлЎң мһ‘м„ұн•ҳкі  мҝ лІ„л„ӨнӢ°мҠӨк°Җ мһҗлҸҷ мқёмҪ”л”©н•ңлӢӨ.

stringData:
  username: admin
  password: s3cr3tPwd

base64лҠ” м•”нҳёнҷ”к°Җ м•„лӢҲлӢӨ

base64лҠ” мқёмҪ”л”©мқј лҝҗ лҲ„кө¬лӮҳ л””мҪ”л”©н•  мҲҳ мһҲлӢӨ. echo czNjcjN0UHdk | base64 -dл©ҙ нҸүл¬ёмқҙ лӮҳмҳЁлӢӨ. Secretмқҙ ConfigMapліҙлӢӨ м•Ҳм „н•ң мқҙмң лҠ” base64 л•Ңл¬ёмқҙ м•„лӢҲлқј RBACВ·tmpfsВ·etcd м•”нҳёнҷ” к°ҷмқҖ л¶Җк°Җ мһҘм№ҳ лҚ•л¶„мқҙлӢӨ.

кё°ліё м„Өм •м—җм„ңлҠ” Secretмқҙ etcdм—җ нҸүл¬ё(base64)мңјлЎң м ҖмһҘлҗңлӢӨ. мӢӨм ң ліҙнҳёлҘј мң„н•ҙм„ңлҠ” лӢӨмқҢмқҙ н•„мҡ”н•ҳлӢӨ.

  • etcd м ҖмһҘ лҚ°мқҙн„° м•”нҳёнҷ”(EncryptionConfiguration)
  • Secret м ‘к·јм—җ лҢҖн•ң мөңмҶҢ к¶Ңн•ң RBAC
  • мҡҙмҳҒ нҷҳкІҪм—җм„ңлҠ” мҷёл¶Җ л№„л°Җ кҙҖлҰ¬ м—°лҸҷ(External Secrets Operator, Vault, нҒҙлқјмҡ°л“ң KMS)

Secret кҙҖлҰ¬лҠ” Secret л¬ём„ңлҘј м°ёкі н•ңлӢӨ. Secret мЈјмһ… л°©мӢқмқҖ ConfigMapкіј лҸҷмқјн•ҳлӢӨ(secretKeyRef, envFrom.secretRef, secret ліјлҘЁ).

    env:
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password

4. нҷҳкІҪліҖмҲҳ vs ліјлҘЁ л§ҲмҡҙнҠё

к°ҷмқҖ лҚ°мқҙн„°лҘј нҷҳкІҪліҖмҲҳлЎң мӨ„м§Җ нҢҢмқјлЎң мӨ„м§ҖлҠ” лҸҷмһ‘ м°Ёмқҙк°Җ л¶„лӘ…н•ҳлҜҖлЎң мқҳмӢқм ҒмңјлЎң м„ нғқн•ҙм•ј н•ңлӢӨ.

кё°мӨҖнҷҳкІҪліҖмҲҳ(env/envFrom)ліјлҘЁ л§ҲмҡҙнҠё
ліҖкІҪ мһҗлҸҷ л°ҳмҳҒм•Ҳ лҗЁ(Pod мһ¬мӢңмһ‘ н•„мҡ”)лҗЁ(мһ мӢң л’Ө нҢҢмқј к°ұмӢ )
л…ём¶ң мң„н—ҳ/proc/<pid>/environ, лЎңк·ёВ·нҒ¬лһҳмӢң лҚӨн”„м—җ л…ём¶ң к°ҖлҠҘнҢҢмқј к¶Ңн•ңмңјлЎң м ңн•ң к°ҖлҠҘ
м Ғн•©н•ң лҚ°мқҙн„°м§§мқҖ мҠӨм№јлқј к°’м„Өм • нҢҢмқј, мқёмҰқм„ң, нҒ° к°’
нӮӨ мқҙлҰ„ м ңм•ҪнҷҳкІҪліҖмҲҳ лӘ…лӘ… к·ңм№ҷмқ„ л”°лқјм•ј н•Ёмһҗмң лЎңмҡҙ нҢҢмқјлӘ…

л№„л°Җ м •ліҙ, нҠ№нһҲ мқёмҰқм„ңлӮҳ нҒ° нӮӨлҠ” ліјлҘЁ л§ҲмҡҙнҠёк°Җ к¶ҢмһҘлҗңлӢӨ. нҷҳкІҪліҖмҲҳлҠ” мһҗмӢқ н”„лЎңм„ёмҠӨлЎң мғҒмҶҚлҗҳкі  м—¬лҹ¬ кіім—җ нқҳлҹ¬лӮҳк°Җкё° мүҪлӢӨ. л°ҳлҢҖлЎң лӢЁмҲңн•ң н”Ңлһҳк·ё н•ңл‘җ к°ңлҠ” нҷҳкІҪліҖмҲҳк°Җ к°„нҺён•ҳлӢӨ.

5. м—…лҚ°мқҙнҠё л°ҳмҳҒкіј immutable ConfigMap

ConfigMapВ·Secretмқ„ мҲҳм •н–Ҳмқ„ л•Ң Podм—җ м–ҙл–»кІҢ л°ҳмҳҒлҗҳлҠ”м§Җк°Җ мӢӨл¬ҙм—җм„ң мһҗмЈј н—·к°ҲлҰ°лӢӨ.

  • ліјлҘЁ л§ҲмҡҙнҠё: kubeletмқҙ мЈјкё°м ҒмңјлЎң лҸҷкё°нҷ”н•ҳлҜҖлЎң мһ мӢң(ліҙнҶө 1л¶„ лӮҙ) л’Ө л§ҲмҡҙнҠёлҗң нҢҢмқјмқҙ к°ұмӢ лҗңлӢӨ. лӢЁ, subPathлЎң л§ҲмҡҙнҠён•ң нҢҢмқјмқҖ к°ұмӢ лҗҳм§Җ м•ҠлҠ”лӢӨ. лҳҗ м• н”ҢлҰ¬мјҖмқҙм…ҳмқҙ нҢҢмқј ліҖкІҪмқ„ к°җм§Җн•ҙ лҰ¬лЎңл“ңн•ҳм§Җ м•Ҡмңјл©ҙ к°’л§Ң л°”лҖҢкі  лҸҷмһ‘мқҖ к·ёлҢҖлЎңлӢӨ.
  • нҷҳкІҪліҖмҲҳ(env/envFrom): м»Ён…Ңмқҙл„Ҳ мӢңмһ‘ мӢңм җм—җ н•ң лІҲ мЈјмһ…лҗҳлҜҖлЎң ConfigMapмқ„ л°”кҝ”лҸ„ л°ҳмҳҒлҗҳм§Җ м•ҠлҠ”лӢӨ. PodлҘј мһ¬мӢңмһ‘н•ҙм•ј н•ңлӢӨ.

env кё°л°ҳ м„Өм •мқ„ м•Ҳм „н•ҳкІҢ к°ұмӢ н•ҳл Өл©ҙ, ConfigMap лӮҙмҡ© н•ҙмӢңлҘј Pod н…ңн”ҢлҰҝ м–ҙл…ён…Ңмқҙм…ҳм—җ л„Јм–ҙ ліҖкІҪ мӢң Deployment лЎӨм•„мӣғмқҙ мһҗлҸҷмңјлЎң мқјм–ҙлӮҳкІҢ н•ҳлҠ” нҢЁн„ҙмқ„ м“ҙлӢӨ(мҳҲ: Helmмқҳ checksum/config м–ҙл…ён…Ңмқҙм…ҳ).
