DNSмҷҖ м„ңл№„мҠӨ л””мҠӨм»ӨлІ„лҰ¬

н•ҷмҠө лӘ©н‘ң

  • CoreDNSк°Җ нҒҙлҹ¬мҠӨн„° DNSлЎңм„ң м–ҙл–Ө м—ӯн• мқ„ н•ҳлҠ”м§Җ мқҙн•ҙн•ңлӢӨ.
  • ServiceмҷҖ Podмқҳ DNS л ҲмҪ”л“ң лӘ…лӘ… к·ңм№ҷмқ„ м •нҷ•нһҲ м•ҲлӢӨ.
  • dnsPolicyмҷҖ search лҸ„л©”мқёмқҙ м§§мқҖ мқҙлҰ„ мЎ°нҡҢлҘј м–ҙл–»кІҢ к°ҖлҠҘн•ҳкІҢ н•ҳлҠ”м§Җ нҢҢм•…н•ңлӢӨ.
  • м„ңл№„мҠӨ л””мҠӨм»ӨлІ„лҰ¬ нҢЁн„ҙ(нҷҳкІҪліҖмҲҳ vs DNS, headless)мқ„ л№„көҗн•ңлӢӨ.

1. CoreDNSмқҳ м—ӯн• 

мҝ лІ„л„ӨнӢ°мҠӨм—җм„ң ServiceлҘј IP лҢҖмӢ  мқҙлҰ„мңјлЎң л¶ҖлҘј мҲҳ мһҲлҠ” кІғмқҖ нҒҙлҹ¬мҠӨн„° лӮҙл¶Җ DNS лҚ•л¶„мқҙлӢӨ. нҳ„мһ¬ кё°ліё кө¬нҳ„мқҖ CoreDNSлЎң, kube-system л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ DeploymentлЎң л–  мһҲкі  kube-dnsлқјлҠ” ServiceлЎң л…ём¶ңлҗңлӢӨ. лӘЁл“  PodлҠ” мқҙ DNS м„ңлІ„лҘј к°ҖлҰ¬нӮӨлҸ„лЎқ мһҗлҸҷ м„Өм •лҗңлӢӨ.

CoreDNSлҠ” API м„ңлІ„лҘј к°җмӢңн•ҳл©° ServiceВ·м—”л“ңнҸ¬мқёнҠё м •ліҙлҘј л°ӣм•„, к·ём—җ лҢҖмқ‘н•ҳлҠ” DNS л ҲмҪ”л“ңлҘј лҸҷм ҒмңјлЎң м ңкіөн•ңлӢӨ. Podк°Җ backendлҘј мЎ°нҡҢн•ҳл©ҙ CoreDNSк°Җ н•ҙлӢ№ Serviceмқҳ ClusterIPлҘј мқ‘лӢөн•ңлӢӨ.

# CoreDNS нҷ•мқё
kubectl get pods -n kube-system -l k8s-app=kube-dns
kubectl get svc -n kube-system kube-dns
# NAME       TYPE        CLUSTER-IP   PORT(S)
# kube-dns   ClusterIP   10.96.0.10   53/UDP,53/TCP

# Podмқҳ resolv.confлҠ” мқҙ DNSлҘј к°ҖлҰ¬нӮЁлӢӨ
kubectl exec -it some-pod -- cat /etc/resolv.conf

CoreDNSмқҳ лҸҷмһ‘мқҖ CorefileмқҙлқјлҠ” м„Өм •(ConfigMap)мңјлЎң м •мқҳлҗңлӢӨ. kubernetes н”Ңлҹ¬к·ёмқёмқҙ нҒҙлҹ¬мҠӨн„° лҸ„л©”мқёмқ„ мІҳлҰ¬н•ҳкі , к·ё мҷё мҷёл¶Җ лҸ„л©”мқёмқҖ м—…мҠӨнҠёлҰј DNSлЎң нҸ¬мӣҢл”©н•ңлӢӨ.

м „мІҙ лҸҷмһ‘мқҖ DNS for Services and Pods л¬ём„ңм—җ м •лҰ¬лҸј мһҲлӢӨ.

2. Serviceмқҳ DNS л ҲмҪ”л“ң к·ңм№ҷ

Serviceмқҳ DNS мқҙлҰ„мқҖ м •н•ҙм§„ к·ңм№ҷмқ„ л”°лҘёлӢӨ. н‘ңмӨҖ нҳ•мӢқмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

<service-name>.<namespace>.svc.<cluster-domain>

кё°ліё нҒҙлҹ¬мҠӨн„° лҸ„л©”мқёмқҖ cluster.localмқҙлҜҖлЎң, default л„Өмһ„мҠӨнҺҳмқҙмҠӨмқҳ backend ServiceлҠ” лӢӨмқҢ FQDNмқ„ к°Җм§„лӢӨ.

backend.default.svc.cluster.local

н•өмӢ¬мқҖ мқҙлҰ„мқҙ л„Өмһ„мҠӨнҺҳмқҙмҠӨлҘј нҸ¬н•Ён•ңлӢӨлҠ” м җмқҙлӢӨ. к°ҷмқҖ л„Өмһ„мҠӨнҺҳмқҙмҠӨ м•Ҳм—җм„ңлҠ” м§§кІҢ backendл§Ң мҚЁлҸ„ лҗҳм§Җл§Ң, лӢӨлҘё л„Өмһ„мҠӨнҺҳмқҙмҠӨмқҳ ServiceлҘј л¶ҖлҘҙл Өл©ҙ л„Өмһ„мҠӨнҺҳмқҙмҠӨлҘј лӘ…мӢңн•ҙм•ј н•ңлӢӨ.

# к°ҷмқҖ л„Өмһ„мҠӨнҺҳмқҙмҠӨ: м§§мқҖ мқҙлҰ„
curl http://backend:8080

# лӢӨлҘё л„Өмһ„мҠӨнҺҳмқҙмҠӨ(payment)мқҳ Service
curl http://gateway.payment:8080
curl http://gateway.payment.svc.cluster.local:8080   # FQDN
лҢҖмғҒDNS мқҙлҰ„л°ҳнҷҳ
мқјл°ҳ Servicesvc.ns.svc.cluster.localA л ҲмҪ”л“ң = ClusterIP
headless Servicesvc.ns.svc.cluster.localA л ҲмҪ”л“ң = к°Ғ Pod IPл“Ө
лӘ…лӘ…лҗң нҸ¬нҠё_http._tcp.svc.ns.svc.cluster.localSRV л ҲмҪ”л“ң

3. Podмқҳ DNS л ҲмҪ”л“ң

Podм—җлҸ„ DNS л ҲмҪ”л“ңлҘј л§Ңл“Ө мҲҳ мһҲлӢӨ. мқјл°ҳ Podмқҳ кё°ліё A л ҲмҪ”л“ң нҳ•мӢқмқҖ IPмқҳ м җмқ„ лҢҖмӢңлЎң л°”кҫј нҳ•нғңлӢӨ.

<pod-ip-dashes>.<namespace>.pod.cluster.local
# мҳҲ: 10.244.1.7 вҶ’ 10-244-1-7.default.pod.cluster.local

мқҙ нҳ•нғңлҠ” мһҳ м“°мқҙм§Җ м•ҠлҠ”лӢӨ. мӢӨл¬ҙм—җм„ң мқҳлҜё мһҲлҠ” кІғмқҖ headless Serviceм—җ мҶҚн•ң Podмқҳ м•Ҳм •м Ғмқё мқҙлҰ„мқҙлӢӨ. StatefulSetкіј headless ServiceлҘј н•Ёк»ҳ м“°л©ҙ к°Ғ Podк°Җ лӢӨмқҢ мқҙлҰ„мқ„ к°Җм§„лӢӨ.

<pod-hostname>.<headless-svc>.<namespace>.svc.cluster.local
# мҳҲ: db-0.db-headless.default.svc.cluster.local

мқҙ м•Ҳм •м Ғмқё мқҙлҰ„ лҚ•л¶„м—җ лҚ°мқҙн„°лІ мқҙмҠӨ нҒҙлҹ¬мҠӨн„° к°ҷмқҖ мҠӨн…ҢмқҙнҠён’Җ мӣҢнҒ¬лЎңл“ңм—җм„ң к°Ғ л©ӨлІ„лҘј кі м •лҗң мЈјмҶҢлЎң м§Җм№ӯн•  мҲҳ мһҲлӢӨ.

4. dnsPolicyмҷҖ search лҸ„л©”мқё

backendмІҳлҹј м§§мқҖ мқҙлҰ„мқҙ м–ҙл–»кІҢ FQDNмңјлЎң н•ҙм„қлҗҳлҠ”м§ҖлҠ” Podмқҳ /etc/resolv.confм—җ мһҲлҠ” search лҸ„л©”мқё лҚ•л¶„мқҙлӢӨ.

kubectl exec -it some-pod -- cat /etc/resolv.conf
# nameserver 10.96.0.10
# search default.svc.cluster.local svc.cluster.local cluster.local
# options ndots:5

backendлҘј мЎ°нҡҢн•ҳл©ҙ лҰ¬мЎёлІ„к°Җ search лҸ„л©”мқёмқ„ мҲңм„ңлҢҖлЎң л¶ҷм—¬ backend.default.svc.cluster.localмқ„ лЁјм Җ мӢңлҸ„н•ҳкі , кұ°кё°м„ң мқ‘лӢөмқ„ л°ӣлҠ”лӢӨ. ndots:5лҠ” м җмқҙ 5к°ң лҜёл§Ңмқё мқҙлҰ„мқҖ мҡ°м„  search лҸ„л©”мқёмқ„ л¶ҷм—¬ мӢңлҸ„н•ҳлқјлҠ” мқҳлҜёлӢӨ.

Podмқҳ DNS лҸҷмһ‘мқҖ dnsPolicyлЎң м ңм–ҙн•ңлӢӨ.

dnsPolicyлҸҷмһ‘
ClusterFirst(кё°ліё) нҒҙлҹ¬мҠӨн„° DNSлҘј лЁјм Җ, мҷёл¶ҖлҠ” м—…мҠӨнҠёлҰјмңјлЎң нҸ¬мӣҢл”©
Defaultл…ёл“ңмқҳ resolv.confлҘј к·ёлҢҖлЎң мғҒмҶҚ(нҒҙлҹ¬мҠӨн„° DNS лҜёмӮ¬мҡ©)
ClusterFirstWithHostNethostNetwork Podм—җм„ң нҒҙлҹ¬мҠӨн„° DNSлҘј м“°л Өл©ҙ мқҙкұёлЎң
NoneлӘЁл“  DNS м„Өм •мқ„ dnsConfigлЎң м§Ғм ‘ м§Җм •

Noneмқ„ м“°л©ҙ searchВ·nameserverВ·optionsлҘј мҷ„м „нһҲ м»ӨмҠӨн„°л§ҲмқҙмҰҲн•  мҲҳ мһҲлӢӨ.

spec:
  dnsPolicy: "None"
  dnsConfig:
    nameservers:
    - 10.96.0.10
    searches:
    - custom.svc.cluster.local
    options:
    - name: ndots
      value: "2"

ndots:5мҷҖ мҷёл¶Җ лҸ„л©”мқё мЎ°нҡҢ

ndots:5 л•Ңл¬ём—җ api.github.com(м җ 2к°ң)мІҳлҹј мҷёл¶Җ лҸ„л©”мқёмқ„ мЎ°нҡҢн•ҳл©ҙ, лҰ¬мЎёлІ„к°Җ search лҸ„л©”мқёмқ„ лЁјм Җ л¶ҷм—¬ м—¬лҹ¬ лІҲ н—ӣмЎ°нҡҢн•ң л’Өм—җм•ј ліёлһҳ мқҙлҰ„мқ„ мӢңлҸ„н•ңлӢӨ. мҷёл¶Җ нҳём¶ңмқҙ л§ҺмқҖ мӣҢнҒ¬лЎңл“ңм—җм„ң DNS м§Җм—°мқҙ мқҳмӢ¬лҗҳл©ҙ FQDN(лҒқм—җ м җ, api.github.com.)мқ„ м“°кұ°лӮҳ ndotsлҘј лӮ®м¶”лҠ” кІғмқ„ кі л Өн•ңлӢӨ.

5. м„ңл№„мҠӨ л””мҠӨм»ӨлІ„лҰ¬ нҢЁн„ҙ

мҝ лІ„л„ӨнӢ°мҠӨм—җм„ң лӢӨлҘё м„ңл№„мҠӨлҘј м°ҫлҠ” л°©лІ•мқҖ нҒ¬кІҢ л‘җ к°Җм§ҖлӢӨ.

нҷҳкІҪліҖмҲҳ л°©мӢқ. Podк°Җ мғқм„ұлҗ  л•Ң, мқҙлҜё мЎҙмһ¬н•ҳлҠ” Serviceл“Өмқҳ м •ліҙк°Җ нҷҳкІҪліҖмҲҳлЎң мЈјмһ…лҗңлӢӨ(мҳҲ: BACKEND_SERVICE_HOST, BACKEND_SERVICE_PORT). лӢЁ, Pod мғқм„ұ мӢңм җм—җ мқҙлҜё мһҲлҚҳ Serviceл§Ң л°ҳмҳҒлҗңлӢӨлҠ” кІ°м •м Ғ н•ңкі„к°Җ мһҲлӢӨ. лӮҳмӨ‘м—җ л§Ңл“Өм–ҙм§„ ServiceлҠ” ліҙмқҙм§Җ м•ҠлҠ”лӢӨ.

kubectl exec -it some-pod -- env | grep SERVICE
