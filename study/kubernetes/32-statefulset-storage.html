StatefulSet мҠӨнҶ лҰ¬м§Җ нҢЁн„ҙ

н•ҷмҠө лӘ©н‘ң

  • volumeClaimTemplatesк°Җ Podл§ҲлӢӨ м „мҡ© PVCлҘј м–ҙл–»кІҢ мһҗлҸҷ л°ңкёүн•ҳлҠ”м§Җ мқҙн•ҙн•ңлӢӨ.
  • Podк°Җ мһ¬мӢңмһ‘В·мһ¬мҠӨмјҖмӨ„лҸјлҸ„ к°ҷмқҖ л””мҠӨнҒ¬м—җ лӢӨмӢң л¶ҷлҠ” м•Ҳм •м Ғ мҠӨнҶ лҰ¬м§Җ мӢқлі„мһҗмқҳ мӣҗлҰ¬лҘј мқөнһҢлӢӨ.
  • лҚ°мқҙн„°лІ мқҙмҠӨВ·л©”мӢңм§Җ нҒҗ к°ҷмқҖ мҠӨн…ҢмқҙнҠён’Җ м•ұмқ„ StatefulSetмңјлЎң л°°нҸ¬н•ҳлҠ” нҢЁн„ҙмқ„ лӢӨлЈ¬лӢӨ.
  • мҠӨмјҖмқјлӢӨмҡҙ мӢң PVCк°Җ ліҙмЎҙлҗҳлҠ” лҸҷмһ‘кіј к·ём—җ л”°лҘё л°ұм—…В·м •лҰ¬ мҡҙмҳҒмқ„ м•ҲлӢӨ.

1. DeploymentлЎңлҠ” м•Ҳ лҗҳлҠ” мқҙмң 

Deploymentмқҳ Podл“ӨмқҖ м„ңлЎң кө¬л¶„лҗҳм§Җ м•ҠлҠ” мқјнҡҢмҡ© ліөм ңліёмқҙлӢӨ. мқҙлҰ„мқҙ лһңлҚӨ н•ҙмӢңлЎң лҒқлӮҳкі (web-7d8f9-abcde), мЈҪмңјл©ҙ м „нҳҖ лӢӨлҘё мқҙлҰ„мқҳ мғҲ PodлЎң лҢҖмІҙлҗҳл©°, лӘЁл‘җ к°ҷмқҖ PVCлҘј кіөмң н•ҳкұ°лӮҳ м•„мҳҲ мҠӨнҶ лҰ¬м§Җк°Җ м—ҶлӢӨ. л¬ҙмғҒнғң мӣ№ м„ңлІ„м—җлҠ” мҷ„лІҪн•ҳм§Җл§Ң, лҚ°мқҙн„°лІ мқҙмҠӨм—җлҠ” м№ҳлӘ…м ҒмқҙлӢӨ.

лҚ°мқҙн„°лІ мқҙмҠӨ ліөм ңліё 3к°ңлҘј мғқк°Ғн•ҙліҙмһҗ. к°Ғ PodлҠ” мһҗкё°л§Ңмқҳ лҚ°мқҙн„° л””л үн„°лҰ¬лҘј к°Җм ём•ј н•ҳкі , мһ¬мӢңмһ‘ нӣ„м—җлҸ„ л°”лЎң к·ё л””л үн„°лҰ¬м—җ лӢӨмӢң л¶ҷм–ҙм•ј н•ҳл©°, нҒҙлҹ¬мҠӨн„° л©ӨлІ„лЎңм„ң м•Ҳм •м Ғмқё л„ӨнҠёмӣҢнҒ¬ мӢ мӣҗмқҙ н•„мҡ”н•ҳлӢӨ. StatefulSetмқҖ мқҙ м„ё к°Җм§Җ вҖ” м•Ҳм •м Ғ мқҙлҰ„, м•Ҳм •м Ғ л„ӨнҠёмӣҢнҒ¬, м•Ҳм •м Ғ мҠӨнҶ лҰ¬м§Җ вҖ” лҘј ліҙмһҘн•ңлӢӨ.

StatefulSetмқҳ PodлҠ” db-0, db-1, db-2мІҳлҹј мҲңм°Ём Ғмқҙкі  м•Ҳм •м Ғмқё мқҙлҰ„(ordinal)мқ„ л°ӣлҠ”лӢӨ. мқҙ мқҙлҰ„мқҙ мҠӨнҶ лҰ¬м§ҖмҷҖ л„ӨнҠёмӣҢнҒ¬ мӢ мӣҗмқҳ лӢ»мқҙ лҗңлӢӨ. м „мІҙ к°ңл…җмқҖ StatefulSet л¬ём„ңм—җ мһҲлӢӨ.

2. volumeClaimTemplates вҖ” Podл§ҲлӢӨ м „мҡ© PVC

н•өмӢ¬ л©”м»ӨлӢҲмҰҳмқҖ volumeClaimTemplatesлӢӨ. StatefulSet мҠӨнҺҷм—җ PVC н…ңн”ҢлҰҝмқ„ л„Јмңјл©ҙ, м»ЁнҠёлЎӨлҹ¬к°Җ к°Ғ Podл§ҲлӢӨ к·ё н…ңн”ҢлҰҝмңјлЎң лі„лҸ„мқҳ PVCлҘј мһҗлҸҷ мғқм„ұн•ңлӢӨ. Pod н•ҳлӮҳм—җ л””мҠӨнҒ¬ н•ҳлӮҳм”© 1:1лЎң л°°м •лҗҳлҠ” кІғмқҙлӢӨ.

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: db
spec:
  serviceName: db-headless
  replicas: 3
  selector:
    matchLabels:
      app: db
  template:
    metadata:
      labels:
        app: db
    spec:
      containers:
      - name: postgres
        image: postgres:16
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: data
          mountPath: /var/lib/postgresql/data
  volumeClaimTemplates:
  - metadata:
      name: data
    spec:
      accessModes:
      - ReadWriteOnce
      storageClassName: fast-ssd
      resources:
        requests:
          storage: 50Gi

мқҙ StatefulSetмқ„ л§Ңл“Өл©ҙ PVCк°Җ мһҗлҸҷмңјлЎң мғқм„ұлҗңлӢӨ.

$ kubectl get pvc
NAME          STATUS   VOLUME     CAPACITY   ACCESS MODES   STORAGECLASS
data-db-0     Bound    pvc-a1..   50Gi       RWO            fast-ssd
data-db-1     Bound    pvc-b2..   50Gi       RWO            fast-ssd
data-db-2     Bound    pvc-c3..   50Gi       RWO            fast-ssd

PVC мқҙлҰ„мқҖ <н…ңн”ҢлҰҝмқҙлҰ„>-<StatefulSetмқҙлҰ„>-<ordinal> к·ңм№ҷмңјлЎң кІ°м •лЎ м ҒмңјлЎң л§Ңл“Өм–ҙм§„лӢӨ(data-db-0). мқҙ кІ°м •лЎ м Ғ мқҙлҰ„мқҙ лӢӨмқҢ м Ҳмқҳ м•Ҳм •м„ұмқ„ к°ҖлҠҘн•ҳкІҢ н•ҳлҠ” м—ҙмҮ лӢӨ.

3. м•Ҳм •м Ғ мҠӨнҶ лҰ¬м§Җ мӢқлі„мһҗ

StatefulSetмқҳ к°ҖмһҘ мӨ‘мҡ”н•ң ліҙмһҘмқҖ, Podк°Җ мЈҪкі  лӢӨмӢң л§Ңл“Өм–ҙм ёлҸ„ к°ҷмқҖ ordinalмқҳ PodлҠ” н•ӯмғҒ к°ҷмқҖ PVCм—җ лӢӨмӢң л¶ҷлҠ”лӢӨлҠ” кІғмқҙлӢӨ. db-1мқҙ мЈҪмңјл©ҙ м»ЁнҠёлЎӨлҹ¬лҠ” мғҲ db-1мқ„ л§Ңл“Өкі , мғҲ PVCлҘј л§Ңл“ңлҠ” кІҢ м•„лӢҲлқј кё°мЎҙ data-db-1мқ„ к·ёлҢҖлЎң лӢӨмӢң л§ҲмҡҙнҠён•ңлӢӨ.

    participant API as kube-apiserver
    participant N1 as Node A
    participant N2 as Node B

    Note over N2: к°ҷмқҖ лҚ°мқҙн„°лЎң ліөкө¬

мқҙ лҚ•л¶„м—җ лҚ°мқҙн„°лІ мқҙмҠӨ ліөм ңліёмқҙ л…ёл“ң мһҘм• лЎң лӢӨлҘё л…ёл“ңм—җ мһ¬мҠӨмјҖмӨ„лҸјлҸ„ мһҗкё° лҚ°мқҙн„°лҘј мһғм§Җ м•ҠлҠ”лӢӨ. PVCк°Җ Podк°Җ м•„лӢҲлқј StatefulSetм—җ л¬¶м—¬ лҸ…лҰҪм ҒмңјлЎң мЎҙмһ¬н•ҳкё° л•Ңл¬ёмқҙлӢӨ(м•һ мұ•н„°м—җм„ң ліё PV лқјмқҙн”„мӮ¬мқҙнҒҙмқҙ м—¬кё°м„ң м§„к°ҖлҘј л°ңнңҳн•ңлӢӨ).

м—¬кё°м—җ лҚ”н•ҙ StatefulSetмқҖ м•Ҳм •м Ғ л„ӨнҠёмӣҢнҒ¬ мӢ мӣҗлҸ„ мӨҖлӢӨ. serviceNameмңјлЎң м§Җм •н•ң headless ServiceмҷҖ кІ°н•©н•ҳл©ҙ к°Ғ Podк°Җ db-0.db-headless.<л„Өмһ„мҠӨнҺҳмқҙмҠӨ>.svc.cluster.local к°ҷмқҖ кі м • DNS мқҙлҰ„мқ„ к°–лҠ”лӢӨ. лҚ°мқҙн„°лІ мқҙмҠӨ нҒҙлҹ¬мҠӨн„° л©ӨлІ„лҒјлҰ¬ м„ңлЎңлҘј мқҙ м•Ҳм •м Ғ мЈјмҶҢлЎң м°ҫлҠ”лӢӨ.

apiVersion: v1
kind: Service
metadata:
  name: db-headless
spec:
  clusterIP: None        # headless
  selector:
    app: db
  ports:
  - port: 5432

4. мҠӨн…ҢмқҙнҠён’Җ м•ұ л°°нҸ¬ нҢЁн„ҙ

мҠӨн…ҢмқҙнҠён’Җ м•ұмқ„ StatefulSetмңјлЎң мҳ¬лҰҙ л•Ң мһҗмЈј м“°лҠ” нҢЁн„ҙл“Өмқҙ мһҲлӢӨ.

мҲңм°Ё кё°лҸҷ(OrderedReady). кё°ліё лҸҷмһ‘мқҖ PodлҘј db-0 вҶ’ db-1 вҶ’ db-2 мҲңм„ңлЎң н•ҳлӮҳм”©, м•һ Podк°Җ Readyк°Җ лҗң л’Өм—җм•ј лӢӨмқҢмқ„ лқ„мҡҙлӢӨ. н”„лқјмқҙлЁёлҰ¬к°Җ лЁјм Җ л– м•ј л Ҳн”ҢлҰ¬м№ҙк°Җ л¶ҷмқ„ мҲҳ мһҲлҠ” DBм—җ м Ғн•©н•ҳлӢӨ. мҲңм„ңк°Җ мӨ‘мҡ”м№ҳ м•Ҡмңјл©ҙ podManagementPolicy: ParallelлЎң лҸҷмӢңм—җ лқ„мӣҢ кё°лҸҷмқ„ л№ лҘҙкІҢ н•ңлӢӨ.

мҙҲкё°нҷ”лҠ” init container лҳҗлҠ” ordinal л¶„кё°. db-0мқ„ н”„лқјмқҙлЁёлҰ¬лЎң, лӮҳлЁём§ҖлҘј л Ҳн”ҢлҰ¬м№ҙлЎң л¶ҖнҢ…н•ҳлҠ” мӢқмқҳ м—ӯн•  л¶„кё°лҠ” Pod мқҙлҰ„мқҳ ordinal(hostnameмқҙ db-0мқём§Җ)лЎң нҢҗлӢЁн•ҳлҠ” нҢЁн„ҙмқ„ нқ”нһҲ м“ҙлӢӨ.

м—…лҚ°мқҙнҠёлҠ” м—ӯмҲң лЎӨл§Ғ. updateStrategy: RollingUpdateмқҙл©ҙ к°ҖмһҘ нҒ° ordinalл¶Җн„°(db-2 вҶ’ db-1 вҶ’ db-0) н•ҳлӮҳм”© көҗмІҙн•ңлӢӨ. partitionмқ„ м§Җм •н•ҳл©ҙ к·ё лІҲнҳё мқҙмғҒл§Ң м—…лҚ°мқҙнҠён•ҙ м№ҙлӮҳлҰ¬ л°°нҸ¬мІҳлҹј лӢЁкі„м ҒмңјлЎң көҙлҰҙ мҲҳ мһҲлӢӨ.

spec:
  podManagementPolicy: OrderedReady
  updateStrategy:
    type: RollingUpdate
    rollingUpdate:
      partition: 2        # db-2л§Ң лЁјм Җ м—…лҚ°мқҙнҠё

мӢӨм ңлЎңлҠ” м§Ғм ‘ StatefulSetмқ„ мһ‘м„ұн•ҳкё°ліҙлӢӨ кІҖмҰқлҗң Operator(мҳҲ: CloudNativePG, Strimzi)лҘј м“°лҠ” кІҪмҡ°к°Җ л§Һм§Җл§Ң, к·ё Operatorл“ӨлҸ„ лӮҙл¶Җм ҒмңјлЎң StatefulSetкіј volumeClaimTemplatesлҘј мқҙл ҮкІҢ нҷңмҡ©н•ңлӢӨ.

5. мҠӨмјҖмқјлӢӨмҡҙкіј PVC ліҙмЎҙ

мҡҙмҳҒм—җм„ң к°ҖмһҘ л§Һмқҙ лҶҖлқјлҠ” м§Җм җмқҙ мҠӨмјҖмқјлӢӨмҡҙмқҙлӢӨ. StatefulSetмқҳ replicasлҘј 3м—җм„ң 1лЎң мӨ„мқҙл©ҙ db-2, db-1 PodлҠ” мӮӯм ңлҗҳм§Җл§Ң, к·ёл“Өмқҳ PVC(data-db-2, data-db-1)лҠ” мһҗлҸҷмңјлЎң мӮӯм ңлҗҳм§Җ м•Ҡкі  к·ёлҢҖлЎң лӮЁлҠ”лӢӨ.

stateDiagram-v2
    [*] --> replicas3: replicas=3
    replicas3 --> replicas1: мҠӨмјҖмқјлӢӨмҡҙ (replicas=1)
    replicas1 --> PodмӮӯм ң: db-2, db-1 Pod м ңкұ°
    PodмӮӯм ң --> PVCліҙмЎҙ: data-db-2, data-db-1\nPVCлҠ” к·ёлҢҖлЎң лӮЁмқҢ
    PVCліҙмЎҙ --> мһ¬нҷ•мһҘ: replicas=3мңјлЎң ліөк·Җ
    мһ¬нҷ•мһҘ --> лҚ°мқҙн„°ліөкө¬: к°ҷмқҖ PVC мһ¬мӮ¬мҡ©\nвҶ’ мҳӣ лҚ°мқҙн„° ліөмӣҗ
    PVCліҙмЎҙ --> мҲҳлҸҷмӮӯм ң: м •л§җ лІ„лҰҙ л•Ң\nkubectl delete pvc
    мҲҳлҸҷмӮӯм ң --> [*]

мқҙкІғмқҖ лІ„к·ёк°Җ м•„лӢҲлқј м•Ҳм „мқ„ мң„н•ң мқҳлҸ„лҗң лҸҷмһ‘мқҙлӢӨ. мӢӨмҲҳлЎң мҠӨмјҖмқјмқ„ мӨ„мҳҖлӢӨк°Җ лӢӨмӢң лҠҳлҰ¬л©ҙ к°ҷмқҖ PVCм—җ лӢӨмӢң л¶ҷм–ҙ лҚ°мқҙн„°к°Җ ліөкө¬лҗңлӢӨ. лҢҖмӢ  л¶Җмһ‘мҡ©мқҙ мһҲлӢӨ вҖ” м •л§җлЎң л””мҠӨнҒ¬к°Җ н•„мҡ” м—Ҷм–ҙмЎҢлӢӨл©ҙ к·ё PVCлҘј мҲҳлҸҷмңјлЎң м§ҖмӣҢм•ј нҒҙлқјмҡ°л“ң л№„мҡ©мқҙ л©Ҳм¶ҳлӢӨ.

# мҠӨмјҖмқјлӢӨмҡҙ нӣ„ м •л§җ лІ„лҰҙ PVC м •лҰ¬
kubectl delete pvc data-db-2 data-db-1

мҝ лІ„л„ӨнӢ°мҠӨ мөңмӢ  лІ„м „м—җм„ңлҠ” мқҙ м •лҰ¬лҘј мһҗлҸҷнҷ”н•ҳлҠ” persistentVolumeClaimRetentionPolicyлҘј StatefulSetм—җ м§Ғм ‘ м§Җм •н•  мҲҳ мһҲлӢӨ.

spec:
  persistentVolumeClaimRetentionPolicy:
    whenScaled: Delete       # мҠӨмјҖмқјлӢӨмҡҙ мӢң PVC мӮӯм ң
    whenDeleted: Retain      # StatefulSet мӮӯм ң мӢңм—” ліҙмЎҙ

whenScaledлҠ” мҠӨмјҖмқјлӢӨмҡҙ мӢң, whenDeletedлҠ” StatefulSet мһҗмІҙлҘј м§Җмҡё л•Ңмқҳ PVC мІҳлҰ¬лҘј к°Ғк°Ғ Retain(кё°ліё, ліҙмЎҙ) лҳҗлҠ” Delete(мӮӯм ң)лЎң м •н•ңлӢӨ.

6. л°ұм—… кі л ӨмӮ¬н•ӯ

мҠӨнҶ лҰ¬м§Җк°Җ Podм—җ м•Ҳм •м ҒмңјлЎң л¶ҷлҠ”лӢӨкі  н•ҙм„ң л°ұм—…мқҙ м Җм ҲлЎң лҗҳлҠ” кұҙ м•„лӢҲлӢӨ. PVCВ·л””мҠӨнҒ¬лҠ” л…ёл“ңВ·zone мһҘм• м—җлҠ” лІ„н…ЁлҸ„, л…јлҰ¬м Ғ мҶҗмғҒ(мһҳлӘ»лҗң DELETE, м•ұ лІ„к·ё, лһңм„¬мӣЁм–ҙ)м—җлҠ” л¬ҙл Ҙн•ҳлӢӨ. лі„лҸ„мқҳ л°ұм—… м „лһөмқҙ л°ҳл“ңмӢң н•„мҡ”н•ҳлӢӨ.

  • ліјлҘЁ мҠӨлғ…мғ· кё°л°ҳ. м•һ мұ•н„°мқҳ VolumeSnapshotмңјлЎң к°Ғ PVCлҘј мЈјкё°м ҒмңјлЎң л– л‘”лӢӨ. лӢЁ, DBлҠ” мҠӨлғ…мғ· м§Ғм „м—җ м• н”ҢлҰ¬мјҖмқҙм…ҳ л ҲлІЁ м •н•©м„ұ(мҳҲ: pg_backup_start, fsync, лҳҗлҠ” мһ к№җ read-only)мқ„ нҷ•ліҙн•ҙм•ј ліөмӣҗліёмқҙ к№Ём§Җм§Җ м•ҠлҠ”лӢӨ. лё”лЎқ мҠӨлғ…мғ·л§ҢмңјлЎңлҠ” мқјкҙҖм„ұмқҙ ліҙмһҘлҗҳм§Җ м•ҠлҠ”лӢӨ.
  • м• н”ҢлҰ¬мјҖмқҙм…ҳ л ҲлІЁ л°ұм—…. pg_dump, mysqldump, лҳҗлҠ” DB мһҗмІҙмқҳ PITR(Point-In-Time Recovery)мқ„ CronJobмңјлЎң лҸҢл Ө мҳӨлёҢм қнҠё мҠӨнҶ лҰ¬м§Җм—җ м Ғмһ¬н•ңлӢӨ. мҠӨлғ…мғ·ліҙлӢӨ лҠҗлҰ¬м§Җл§Ң л…јлҰ¬м Ғ мқјкҙҖм„ұкіј л¶Җл¶„ ліөмӣҗм—җ к°•н•ҳлӢӨ.
  • нҒҙлҹ¬мҠӨн„° м „мІҙ л°ұм—… лҸ„кө¬. Velero к°ҷмқҖ лҸ„кө¬лЎң StatefulSet л§ӨлӢҲнҺҳмҠӨнҠёмҷҖ PVC мҠӨлғ…мғ·мқ„ н•Ёк»ҳ л°ұм—…н•ҙ нҒҙлҹ¬мҠӨн„° лӢЁмң„ ліөкө¬лҘј мӨҖл№„н•ңлӢӨ.

ліөмӣҗ м Ҳм°ЁлҸ„ лҜёлҰ¬ кІҖмҰқн•ҙм•ј н•ңлӢӨ. мҠӨлғ…мғ·м—җм„ң мғҲ PVCлҘј л§Ңл“Ө л•Ң StatefulSetмқҳ кІ°м •лЎ м Ғ PVC мқҙлҰ„ к·ңм№ҷ(data-db-0)м—җ л§һм¶° лҜёлҰ¬ PVCлҘј к№”м•„л‘җкі  StatefulSetмқ„ л¶ҷмқҙл©ҙ, м»ЁнҠёлЎӨлҹ¬к°Җ мғҲ л””мҠӨнҒ¬лҘј л§Ңл“Өм§Җ м•Ҡкі  ліөмӣҗлҗң PVCлҘј к·ёлҢҖлЎң мұ„нғқн•ҳкІҢ н•  мҲҳ мһҲлӢӨ. л°ұм—…мқҖ л– л‘җлҠ” кІғліҙлӢӨ ліөмӣҗмқҙ лҗҳлҠ”м§Җ м •кё°м ҒмңјлЎң нҷ•мқён•ҳлҠ” кІҢ лҚ” мӨ‘мҡ”н•ҳлӢӨ.

н•өмӢ¬ м •лҰ¬

  • DeploymentлҠ” л¬ҙмғҒнғң ліөм ңліёмҡ©мқҙкі , StatefulSetмқҖ Podл§ҲлӢӨ м•Ҳм •м Ғ мқҙлҰ„В·л„ӨнҠёмӣҢнҒ¬В·мҠӨнҶ лҰ¬м§ҖлҘј ліҙмһҘн•ҙ мҠӨн…ҢмқҙнҠён’Җ м•ұм—җ м“ҙлӢӨ.
  • volumeClaimTemplatesлҠ” Podл§ҲлӢӨ <н…ңн”ҢлҰҝ>-<м…ӢмқҙлҰ„>-<ordinal> к·ңм№ҷмқҳ м „мҡ© PVCлҘј кІ°м •лЎ м ҒмңјлЎң мһҗлҸҷ л°ңкёүн•ңлӢӨ.
  • к°ҷмқҖ ordinalмқҳ PodлҠ” мһ¬мӢңмһ‘В·мһ¬мҠӨмјҖмӨ„лҸјлҸ„ н•ӯмғҒ к°ҷмқҖ PVCм—җ лӢӨмӢң л¶ҷм–ҙ лҚ°мқҙн„°лҘј мһғм§Җ м•Ҡмңјл©°, headless ServiceлЎң м•Ҳм •м Ғ DNS мӢ мӣҗк№Ңм§Җ м–»лҠ”лӢӨ.
  • мҠӨмјҖмқјлӢӨмҡҙ мӢң PVCлҠ” кё°ліём ҒмңјлЎң ліҙмЎҙлҗҳл©°(м•Ҳм „мһҘм№ҳ), persistentVolumeClaimRetentionPolicyлЎң мһҗлҸҷ м •лҰ¬лҘј м ңм–ҙн•  мҲҳ мһҲлӢӨ.
  • м•Ҳм •м Ғ мҠӨнҶ лҰ¬м§Җк°Җ л°ұм—…мқ„ лҢҖмӢ н•ҳм§ҖлҠ” м•ҠлҠ”лӢӨ вҖ” м •н•©м„ұмқ„ нҷ•ліҙн•ң мҠӨлғ…мғ·В·м•ұ л ҲлІЁ л°ұм—…кіј ліөмӣҗ кІҖмҰқмқҙ лі„лҸ„лЎң н•„мҡ”н•ҳлӢӨ.

лӢӨмқҢ мұ•н„°

м§ҖкёҲк№Ңм§Җ мҠӨнҶ лҰ¬м§Җ мҳҒм—ӯ вҖ” Volume, PV/PVC, CSI, StatefulSet мҠӨнҶ лҰ¬м§Җ вҖ” лҘј к№ҠкІҢ лӢӨлӨҳлӢӨ. лҚ°мқҙн„°к°Җ м•Ҳм „н•ҳкІҢ ліҙмЎҙлҗҳлҠ” кё°л°ҳмқ„ м„ёмӣ мңјлӢҲ, мқҙм ң к·ё лҚ°мқҙн„°мҷҖ нҒҙлҹ¬мҠӨн„° мһҗмӣҗм—җ лҲ„к°Җ м ‘к·јн•  мҲҳ мһҲлҠ”м§ҖлҘј нҶөм ңн•  м°ЁлЎҖлӢӨ. лӢӨмқҢ мұ•н„° мқёмҰқВ·мқёк°ҖмҷҖ RBACм—җм„ңлҠ” нҒҙлҹ¬мҠӨн„°мқҳ мқёмҰқ нқҗлҰ„кіј RBACлЎң к¶Ңн•ңмқ„ лӢӨмҠӨлҰ¬лҠ” л°©лІ•мқ„ лӢӨлЈ¬лӢӨ.
