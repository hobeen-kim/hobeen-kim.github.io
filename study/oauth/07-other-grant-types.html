1. Implicit Flow вҖ” н”„лһҳк·ёлЁјнҠёлЎң нҶ нҒ°мқ„ л°”лЎң л°ӣлҠ”лӢӨ

Implicit FlowлҠ” OAuth 2.0 мҙҲкё°м—җ SPAВ·Native м•ұмқ„ мң„н•ҙ кі м•ҲлҗҗлӢӨ. Authorization Code Flowмқҳ Code көҗнҷҳ лӢЁкі„лҘј мғқлһөн•ҳкі , /authorize мқ‘лӢө мһҗмІҙм—җ Access Tokenмқ„ мӢЈлҠ”лӢӨ.

н•өмӢ¬мқҖ URL н”„лһҳк·ёлЁјнҠё(# мқҙнӣ„)м—җ нҶ нҒ°мқ„ мӢЈлҠ”лӢӨлҠ” м җмқҙлӢӨ. н”„лһҳк·ёлЁјнҠёлҠ” лёҢлқјмҡ°м Җк°Җ м„ңлІ„лЎң м „мҶЎн•ҳм§Җ м•ҠмңјлҜҖлЎң, AS мқ‘лӢө м§Ғнӣ„ нҒҙлқјмқҙм–ёнҠё JSл§Ң мқҙ л¶Җл¶„мқ„ мқҪлҠ”лӢӨлҠ” м•„мқҙл””м–ҙмҳҖлӢӨ.

л“ұмһҘ л°°кІҪ

2012л…„ лӢ№мӢң SPAВ·Native нҷҳкІҪм—җм„ңлҠ” лӢӨмқҢ л¬ём ңк°Җ мһҲм—ҲлӢӨ.

  • CORSк°Җ м•„м§Ғ ліҙнҺёнҷ”лҗҳм§Җ м•Ҡм•„ SPAк°Җ м§Ғм ‘ AS /tokenмқ„ нҳём¶ңн•ҳкё° м–ҙл Өмӣ лӢӨ.
  • лӘЁл°”мқј OSмқҳ WebViewк°Җ лӢӨм–‘н–Ҳкі , лёҢлқјмҡ°м Җ м„ём…ҳкіј м•ұ мғҒнғң м—°кІ°мқҙ ліөмһЎн–ҲлӢӨ.
  • Public Clientм—җ Client Secretмқ„ мӨ„ мҲҳ м—ҶмңјлҜҖлЎң Authorization Code + Secret мЎ°н•©мқҙ л¶Ҳк°ҖлҠҘн–ҲлӢӨ.

мқҙлҹ° мғҒнҷ©м—җм„ң "Code көҗнҷҳ лӢЁкі„лҘј мғқлһөн•ҳмһҗ"лҠ” нғҖнҳ‘м•Ҳмқҙ Implicit FlowмҳҖлӢӨ.

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” URL л…ём¶ң

Implicit Flowмқҳ к°ҖмһҘ к·јліём Ғмқё м•Ҫм җмқҖ нҶ нҒ°мқҙ URLм—җ м°ҚнһҢлӢӨлҠ” кІғмқҙлӢӨ.

  • лёҢлқјмҡ°м Җ нҷ•мһҘ н”„лЎңк·ёлһЁмқҙ window.location.hrefлҘј мқҪмқҢ
  • м—җлҹ¬ лҰ¬нҸ¬нҠё м„ңл№„мҠӨ(Sentry л“ұ)к°Җ URLмқ„ нҸ¬н•ЁмӢңнӮҙ
  • лёҢлқјмҡ°м Җ нһҲмҠӨнҶ лҰ¬ лҸҷкё°нҷ” кё°лҠҘ
  • м•…м„ұ CSSлӮҳ мқҙлҜём§Җ лЎңлҚ”к°Җ URL м°ёмЎ°

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” Token Injection

кіөкІ©мһҗк°Җ н”јн•ҙмһҗм—җкІҢ мһҗмӢ мқҙ мқҙлҜё л°ӣмқҖ Access Tokenмқҙ лӢҙкёҙ URLмқ„ нҒҙлҰӯн•ҳкІҢ л§Ңл“ лӢӨ.

https://app.example.com/callback#access_token=<кіөкІ©мһҗ нҶ нҒ°>&token_type=Bearer

н”јн•ҙмһҗ SPAмқҳ JSлҠ” мқҙ нҶ нҒ°мқ„ к·ёлҢҖлЎң localStorageм—җ м ҖмһҘн•ҳкі  "лЎңк·ёмқёлҗЁ"мңјлЎң мІҳлҰ¬н•ңлӢӨ. н”јн•ҙмһҗк°Җ мқҙнӣ„ мҲҳн–үн•ҳлҠ” лӘЁл“  мһ‘м—…мқҙ кіөкІ©мһҗ кі„м •мңјлЎң кё°лЎқлҗңлӢӨ. "Login CSRF"мқҳ OAuth нҢҗмқҙлӢӨ.

Authorization Code FlowлҠ” state кІҖмҰқкіј Code көҗнҷҳмңјлЎң мқҙ кіөкІ©мқ„ л§үм§Җл§Ң, ImplicitмқҖ к·ё мһҘм№ҳк°Җ л¶ҖмЎұн•ҳлӢӨ.

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” Refresh Token м—ҶмқҢ

ImplicitмқҖ м„Өкі„мғҒ Refresh Tokenмқ„ л°ңкёүн•ҳм§Җ м•ҠлҠ”лӢӨ. м§§мқҖ Access Token л§ҢлЈҢ мӢң, мӮ¬мҡ©мһҗлҠ” л§ӨлІҲ мһ¬мқёмҰқн•ҙм•ј н•ңлӢӨ. мқҙлҘј н”јн•ҳл Өкі  Implicit кө¬нҳ„мІҙл“ӨмқҖ "мЎ°мҡ©н•ң мһ¬мқёмҰқ(silent authentication)"мқ„ мң„н•ҙ мҲЁкІЁм§„ iframeм—җм„ң /authorizeлҘј лӢӨмӢң нҳём¶ңн•ҳлҠ” кјјмҲҳлҘј мҚјлӢӨ. мқҙлҠ” лӢӨмӢң Cross-Site нҠёлһҳнӮ№кіј м—°лҸҷлҸј мғҲ м·Ём•Ҫм җмқ„ л§Ңл“Өм—ҲлӢӨ.

нҳ„мһ¬ мғҒнғң

  • OAuth 2.0 Security Best Current Practice (draft-ietf-oauth-security-topics)к°Җ Implicit Flow мӮ¬мҡ©мқ„ кёҲм§Җн•ңлӢӨ.
  • OAuth 2.1мқҖ Implicitмқ„ кіөмӢқ м ңкұ° мҳҲм •мқҙлӢӨ.
  • мЈјмҡ” IdP(Google, Auth0 л“ұ)лҸ„ мӢ к·ң нҒҙлқјмқҙм–ёнҠём—җм„ң Implicitмқ„ кё°ліё л№„нҷңм„ұнҷ”н–ҲлӢӨ.

лҢҖмІҙм•ҲмқҖ Authorization Code + PKCEлӢӨ. CORSк°Җ ліҙнҺёнҷ”лҗң нҳ„мһ¬ SPAлҸ„ /tokenмқ„ м§Ғм ‘ нҳём¶ңн•  мҲҳ мһҲмңјлҜҖлЎң, Implicitмқҳ мЎҙмһ¬ мқҙмң к°Җ мӮ¬лқјмЎҢлӢӨ.

2. ROPC вҖ” Resource Owner Password Credentials

ROPCлҠ” OAuth 2.0мқҳ л„Ө к°Җм§Җ кё°ліё Grant Type мӨ‘ н•ҳлӮҳмқҙм§Җл§Ң, мқҙлҰ„л§Ң л“Өм–ҙлҸ„ мҷң мң„н—ҳн•ңм§Җ м§җмһ‘мқҙ к°„лӢӨ. нҒҙлқјмқҙм–ёнҠёк°Җ мӮ¬мҡ©мһҗмқҳ IDмҷҖ л№„л°ҖлІҲнҳёлҘј м§Ғм ‘ л°ӣм•„м„ң ASм—җ м „лӢ¬н•ҳлҠ” л°©мӢқмқҙлӢӨ.

кё°ліё лҸҷмһ‘

sequenceDiagram
    participant U as мӮ¬мҡ©мһҗ
    participant C as нҒҙлқјмқҙм–ёнҠё
    participant AS as AS
    U->>C: ID + л№„л°ҖлІҲнҳё мһ…л Ҙ
    Note over C: л№„л°ҖлІҲнҳёк°Җ нҒҙлқјмқҙм–ёнҠёлҘј нҶөкіј
    C->>AS: POST /token<br>grant_type=password<br>username=u&password=p<br>client_id=..&client_secret=..
    AS->>AS: мӮ¬мҡ©мһҗ мқёмҰқ
    AS-->>C: access_token + refresh_token

мҡ”мІӯ мҳҲмӢң.

POST /token HTTP/1.1
Host: auth.example.com
Authorization: Basic <client credentials>
Content-Type: application/x-www-form-urlencoded

grant_type=password
&username=alice@example.com
&password=TopSecret
&scope=openid profile

мҷң мЎҙмһ¬н–ҲлҠ”к°Җ

RFC 6749к°Җ ROPCлҘј нҸ¬н•ЁмӢңнӮЁ мқҙмң лҠ” м ңн•ңм Ғ мғҒнҷ©мқ„ мң„н•ң кІғмқҙм—ҲлӢӨ.

  • л Ҳкұ°мӢң м•ұмқҙ л№„л°ҖлІҲнҳё мһ…л Ҙ UIм—җ мқҳмЎҙн•ҙм„ң мҰүмӢң лёҢлқјмҡ°м Җ лҰ¬лӢӨмқҙл үнҠёлЎң м „нҷҳн•  мҲҳ м—Ҷмқ„ л•Ң
  • 1м°Ё мӢ лў° кҙҖкі„мқё м•ұ(к°ҷмқҖ нҡҢмӮ¬ мһҗмІҙ лӘЁл°”мқј м•ұ)мқҙ кі мң  лЎңк·ёмқё UXлҘј мң м§Җн•  л•Ң

мқҙ мЎ°кұҙмқҖ мІҳмқҢл¶Җн„° мўҒм•ҳкі , мӢңк°„мқҙ м§ҖлӮҳл©ҙм„ң кұ°мқҳ мӮ¬лқјмЎҢлӢӨ.

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” л№„л°ҖлІҲнҳё л…ём¶ң

CH1м—җм„ң лӢӨлЈ¬ "л№„л°ҖлІҲнҳё кіөмң мқҳ 4лҢҖ л¬ём ң"к°Җ к·ёлҢҖлЎң мһ¬нҳ„лҗңлӢӨ. нҒҙлқјмқҙм–ёнҠёк°Җ мӮ¬мҡ©мһҗ л№„л°ҖлІҲнҳёлҘј нҸүл¬ёмңјлЎң ліҙкі  м ҖмһҘк№Ңм§Җ н•  мҲҳ мһҲлӢӨ. OAuthмқҳ н•өмӢ¬ лӘ©н‘ң(л№„л°ҖлІҲнҳё кіөмң  м ңкұ°)лҘј м •л©ҙмңјлЎң мң„л°°н•ңлӢӨ.

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” MFA мҡ°нҡҢ л¶Ҳк°Җ

MFA, мҶҢм…ң лЎңк·ёмқё, мң„н—ҳ кё°л°ҳ мқёмҰқ(Risk-based Authentication) к°ҷмқҖ ASмқҳ лӘЁл“  ліҙм•Ҳ кё°лҠҘмқҙ мҡ°нҡҢлҗңлӢӨ. ASлҠ” username+passwordл§Ң л°ӣмңјлҜҖлЎң, MFA мұҢлҰ°м§ҖлҘј лқ„мҡё мҲҳ м—ҶлӢӨ. кё°м—… SSOВ·OTPВ·мғқмІҙ мқёмҰқ л“ұ нҳ„лҢҖ мқёмҰқ н”ҢлЎңмҡ°мҷҖ нҳёнҷҳлҗҳм§Җ м•ҠлҠ”лӢӨ.

кіөкІ© мӢңлӮҳлҰ¬мҳӨ вҖ” н”јмӢұ н‘ңл©ҙ нҷ•лҢҖ

мӮ¬мҡ©мһҗк°Җ "мқҙ м•ұм—җ лӮҙ IdP л№„л°ҖлІҲнҳёлҘј мһ…л Ҙн•ҳлҠ” кІҢ м •мғҒ"мқҙлқјлҠ” мҠөкҙҖмқ„ к°–кІҢ лҗңлӢӨ. мқҙлҠ” н”јмӢұ м•ұмқҙ м •мғҒ м•ұкіј кө¬л¶„лҗҳм§Җ м•ҠкІҢ л§Ңл“ңлҠ” мөңм•…мқҳ кҙҖн–үмқҙлӢӨ. OAuthк°Җ мІҳмқҢл¶Җн„° м—Ҷм• л Ө н–ҲлҚҳ м•ҲнӢ°нҢЁн„ҙмқҙлӢӨ.

3. Client Credentials вҖ” м„ңлІ„ к°„ мқёмҰқ

Client Credentials GrantлҠ” OAuth 2.0мқҳ л„Ө к°Җм§Җ Grant мӨ‘ мң мқјн•ҳкІҢ мӮ¬мҡ©мһҗк°Җ л“ұмһҘн•ҳм§Җ м•ҠлҠ” н”ҢлЎңмҡ°лӢӨ. M2M(Machine-to-Machine) нҶөмӢ мҡ©мқҙлӢӨ.

кё°ліё лҸҷмһ‘

sequenceDiagram
    participant C as нҒҙлқјмқҙм–ёнҠё (м„ңлІ„)
    participant AS as Authorization Server
    participant RS as Resource Server
    C->>AS: POST /token<br>grant_type=client_credentials<br>client_id + client_secret
    AS->>AS: нҒҙлқјмқҙм–ёнҠё кІҖмҰқ
    AS-->>C: access_token

мӮ¬мҡ©мһҗ к°ңмһ… м—Ҷмқҙ, нҒҙлқјмқҙм–ёнҠёк°Җ мһҗмӢ мқ„ м§Ғм ‘ мқёмҰқн•ҙм„ң нҶ нҒ°мқ„ л°ӣлҠ”лӢӨ. нҶ нҒ°мқҳ subлҠ” мӮ¬мҡ©мһҗк°Җ м•„лӢҲлқј нҒҙлқјмқҙм–ёнҠё мһҗмӢ мқҙлӢӨ.

м Ғн•©н•ң мӢңлӮҳлҰ¬мҳӨ

  • м„ңл№„мҠӨ к°„ нҳём¶ң: л§ҲмқҙнҒ¬лЎңм„ңл№„мҠӨ Aк°Җ Bмқҳ APIлҘј нҳём¶ңн•  л•Ң
  • м„ңлІ„ лҢҖ м„ңлІ„ лҚ°мқҙн„° лҸҷкё°нҷ”: ERP вҶ” CRM к°„ м—°лҸҷ
  • кҙҖлҰ¬мһҗ API: мЎ°м§Ғ лӢЁмң„ кҙҖлҰ¬(мӮ¬мҡ©мһҗ мғқм„ұ л“ұ)

л¶Җм Ғн•©н•ң мӢңлӮҳлҰ¬мҳӨ

  • мӮ¬мҡ©мһҗк°Җ м§Ғм ‘ м“°лҠ” м•ұ: SPA, лӘЁл°”мқј, мӣ№ м•ұ
  • мӮ¬мҡ©мһҗлі„ мһҗмӣҗ м ‘к·ј: "aliceмқҳ нҢҢмқј"мқ„ нҳём¶ңн•  л•Ң
  • к°җмӮ¬к°Җ мӮ¬мҡ©мһҗ лӢЁмң„лЎң н•„мҡ”н•ң кІҪмҡ°: к°ңмқёлі„ н–үмң„ лЎңк№…

ScopeмҷҖ к¶Ңн•ң кІҪкі„

Client CredentialsлҠ” нҒҙлқјмқҙм–ёнҠё мһҗмІҙк°Җ к°Җм§„ к¶Ңн•ңл§Ң мҡ”мІӯн•  мҲҳ мһҲлӢӨ. ASлҠ” нҒҙлқјмқҙм–ёнҠё л“ұлЎқ мӢң н—Ҳмҡ©н•ң Scope м§‘н•©л§Ң л°ңкёүн•ңлӢӨ. мҳҲлҘј л“Өм–ҙ "л°°м№ҳ м„ңлІ„" нҒҙлқјмқҙм–ёнҠёлҠ” reports.read, metrics.writeл§Ң мҡ”мІӯ к°ҖлҠҘн•ҳкі , мӮ¬мҡ©мһҗ к°ңмқё нҢҢмқј м ‘к·ј(drive.readonly)мқҖ мҡ”мІӯн•ҙлҸ„ кұ°л¶ҖлҗңлӢӨ.

Confused Deputy мЈјмқҳ

Client CredentialsлЎң л°ӣмқҖ нҶ нҒ°мңјлЎң мӮ¬мҡ©мһҗлі„ мһҗмӣҗмқ„ мЎ°мһ‘н•ҳл©ҙ Confused Deputy кіөкІ©м—җ л…ём¶ңлҗңлӢӨ. мҳҲлҘј л“Өм–ҙ APIк°Җ "мқҙ нҶ нҒ°мқҙ мң нҡЁн•ҳкі  files.write Scopeк°Җ мһҲмңјлӢҲ н—Ҳмҡ©"мқҙлқјкі л§Ң нҢҗлӢЁн•ҳл©ҙ, м•…мқҳм Ғ мҡ”мІӯмқҙ м–ҙл–Ө мӮ¬мҡ©мһҗмқҳ нҢҢмқјмқҙл“  мҲҳм •н•  мҲҳ мһҲлӢӨ.

// м•ҲнӢ°нҢЁн„ҙ
@PostMapping("/files/{userId}")
public void writeFile(@PathVariable String userId, @RequestBody String content) {
    // нҶ нҒ° Scopeл§Ң нҷ•мқён•ҳкі  userId мҶҢмң к¶Ң кІҖмҰқ м•Ҳ н•Ё
    fileService.write(userId, content);
}

мқҙ кІҪмҡ° л№„мҰҲлӢҲмҠӨ л Ҳмқҙм–ҙмқҳ мқёк°Җк°Җ лі„лҸ„лЎң н•„мҡ”н•ҳлӢӨ. OAuth нҶ нҒ°мқҖ "л¬ҙм—Үмқ„ н•  мҲҳ мһҲлҠ”к°Җ"лҘј м•Ңл ӨмЈјм§Җл§Ң, "лҲ„кө¬мқҳ мһҗмӣҗм—җ м ‘к·јн•ҳлҠ”к°Җ"лҠ” м•ұмқҙ м§Ғм ‘ кІҖмҰқн•ҙм•ј н•ңлӢӨ.

5. OAuth 2.1 л°©н–Ҙм„ұ

OAuth 2.1мқҖ нҳ„мһ¬ IETFм—җм„ң л“ңлһҳн”„нҠёлЎң м§„н–ү мӨ‘мқё к·ңкІ©мңјлЎң, 2.0мқҳ лӢӨм–‘н•ң RFCмҷҖ Best Current PracticeлҘј нҶөн•©В·лӢЁмҲңнҷ”н•ңлӢӨ. н•өмӢ¬ ліҖкІҪмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

Implicit кіөмӢқ м ңкұ°

response_type=token мӮ¬мҡ©мқҙ к·ңкІ©м—җм„ң м ңкұ°лҗңлӢӨ. лӘЁл“  мқёк°Җ мҡ”мІӯмқҖ response_type=codeлЎң мҲҳл ҙн•ңлӢӨ.

ROPC кіөмӢқ м ңкұ°

grant_type=password м§Җмӣҗмқҙ к·ңкІ©м—җм„ң м ңкұ°лҗңлӢӨ. кё°мЎҙ нҳёнҷҳм„ұмқ„ мң„н•ҙ ASк°Җ м§Җмӣҗн•  мҲҳлҠ” мһҲмңјлӮҳ л№„н‘ңмӨҖ нҷ•мһҘмңјлЎң л¶„лҘҳлҗңлӢӨ.

PKCE н•„мҲҳнҷ”

лӘЁл“  Authorization Code Flowм—җ PKCEк°Җ н•„мҲҳк°Җ лҗңлӢӨ. Confidential ClientлҸ„ мҳҲмҷёк°Җ м•„лӢҲлӢӨ. Client Secretмқҙ мһҲлҚ”лқјлҸ„ PKCEлҘј лі‘н–үн•ҙм„ң "мқҙ нҶ нҒ° мҡ”мІӯмқҙ к·ё Authorization мҡ”мІӯкіј к°ҷмқҖ м„ём…ҳ"мһ„мқ„ мқҙмӨ‘мңјлЎң мҰқлӘ…н•ңлӢӨ.

Refresh Token к¶ҢмһҘ мӮ¬н•ӯ

  • Public Clientм—җм„ң Refresh Token мӮ¬мҡ© мӢң нҡҢм „(Rotation)мқҙ н•„мҲҳ
  • Refresh Token мһ¬мӮ¬мҡ© нғҗм§Җ(reuse detection) к¶ҢмһҘ
  • Sender-constrained token(DPoP, mTLS) мһҘл Ө

redirect_uri л§Өм№ӯ м—„кІ©нҷ”

  • м •нҷ• мқјм№ҳ(exact match)к°Җ кё°ліё к·ңм№ҷ
  • л¶Җл¶„ л§Өм№ӯВ·мҷҖмқјл“ңм№ҙл“ң кёҲм§Җ
