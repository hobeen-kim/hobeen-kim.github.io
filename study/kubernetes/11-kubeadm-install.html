нҒҙлҹ¬мҠӨн„° м„Өм№ҳ (kubeadm)

н•ҷмҠө лӘ©н‘ң

  • м»Ён…Ңмқҙл„Ҳ лҹ°нғҖмһ„В·мҠӨмҷ‘В·м»Өл„җ лӘЁл“ҲВ·sysctl л“ұ л…ёл“ң мӮ¬м „ мӨҖл№„ н•ӯлӘ©мқ„ л№ м§җм—Ҷмқҙ м җкІҖн•ңлӢӨ.
  • kubeadm initмқҙ control planeмқ„ м–ҙл–Ө лӢЁкі„лЎң л¶ҖнҠёмҠӨнҠёлһ©н•ҳлҠ”м§Җ phase лӢЁмң„лЎң мқҙн•ҙн•ңлӢӨ.
  • CNI н”Ңлҹ¬к·ёмқёмқ„ м„Өм№ҳн•ҙ нҢҢл“ң л„ӨнҠёмӣҢнҒ¬лҘј нҷңм„ұнҷ”н•ҳкі  л…ёл“ңк°Җ Readyк°Җ лҗҳлҠ” кіјм •мқ„ ліёлӢӨ.
  • worker л…ёл“ңлҘј kubeadm joinмңјлЎң н•©лҘҳмӢңнӮӨкі  м„Өм№ҳлҘј кІҖмҰқн•ңлӢӨ.

1. kubeadmмқҙлһҖ л¬ҙм—Үмқёк°Җ

kubeadmмқҖ "мөңмҶҢн•ңмңјлЎң лҸҷмһ‘ к°ҖлҠҘн•ң(minimum viable)" мҝ лІ„л„ӨнӢ°мҠӨ нҒҙлҹ¬мҠӨн„°лҘј л¶ҖнҠёмҠӨнҠёлһ©н•ҳлҠ” кіөмӢқ лҸ„кө¬лӢӨ. нҒҙлҹ¬мҠӨн„°лҘј мІҳмқҢл¶Җн„° мҶҗмңјлЎң кө¬м„ұн•ҳл Өл©ҙ мқёмҰқм„ңлҘј л°ңкёүн•ҳкі , etcdлҘј лқ„мҡ°кі , kube-apiserverВ·controller-managerВ·schedulerлҘј к°Ғк°Ғ м„Өм •н•ҳкі , kubeletмқ„ м—°кІ°н•ҙм•ј н•ңлӢӨ. kubeadmмқҖ мқҙ кіјм •мқ„ init/join л‘җ лӘ…л №мңјлЎң л¬¶м–ҙ н‘ңмӨҖнҷ”н•ңлӢӨ.

kubeadmмқҙ мұ…мһ„м§ҖлҠ” лІ”мң„мҷҖ мұ…мһ„м§Җм§Җ м•ҠлҠ” лІ”мң„лҘј кө¬л¶„н•ҳлҠ” кІҢ мӨ‘мҡ”н•ҳлӢӨ.

kubeadmмқҙ н•ҳлҠ” мқјkubeadmмқҙ н•ҳм§Җ м•ҠлҠ” мқј
PKI/kubeconfig мғқм„ұ, control plane л¶ҖнҠёмҠӨнҠёлһ©мқён”„лқј(VM/л„ӨнҠёмӣҢнҒ¬) н”„лЎңл№„м ҖлӢқ
kube-proxyВ·CoreDNS м• л“ңмҳЁ м„Өм№ҳCNI н”Ңлҹ¬к·ёмқё м„Өм№ҳ (мӮ¬мҡ©мһҗ лӘ«)
л…ёл“ң join, м—…к·ёл Ҳмқҙл“ң м Ҳм°ЁлӘЁлӢҲн„°л§ҒВ·лЎңк№…В·мҠӨнҶ лҰ¬м§Җ кө¬м„ұ

мһҗм„ён•ң лӮҙмҡ©мқҖ kubeadmмңјлЎң нҒҙлҹ¬мҠӨн„° л§Ңл“Өкё° кіөмӢқ л¬ём„ңлҘј л”°лҘёлӢӨ.

2. л…ёл“ң мӮ¬м „ мӨҖл№„

init/joinмқ„ мӢӨн–үн•ҳкё° м „м—җ лӘЁл“  л…ёл“ң(control planeВ·worker)м—җм„ң лҸҷмқјн•ң мӮ¬м „ мӨҖл№„к°Җ н•„мҡ”н•ҳлӢӨ. мқҙ лӢЁкі„лҘј кұҙл„Ҳлӣ°л©ҙ kubeletмқҙ кё°лҸҷлҗҳм§Җ м•Ҡкұ°лӮҳ нҢҢл“ң к°„ нҶөмӢ мқҙ к№Ём§„лӢӨ.

мҠӨмҷ‘ л№„нҷңм„ұнҷ”

kubeletмқҖ кё°ліём ҒмңјлЎң мҠӨмҷ‘мқҙ мјңм ё мһҲмңјл©ҙ кё°лҸҷмқ„ кұ°л¶Җн•ңлӢӨ. л©”лӘЁлҰ¬ м••л°• мӢң мҠӨмҷ‘мқҙ QoSВ·OOM лҸҷмһ‘мқ„ мҳҲмёЎ л¶Ҳк°ҖлҠҘн•ҳкІҢ л§Ңл“Өкё° л•Ңл¬ёмқҙлӢӨ.

# мҰүмӢң л№„нҷңм„ұнҷ”
sudo swapoff -a
# мһ¬л¶ҖнҢ… нӣ„м—җлҸ„ мң м§Җ вҖ” fstabмқҳ swap лқјмқё мЈјм„қ мІҳлҰ¬
sudo sed -i '/ swap / s/^/#/' /etc/fstab

м»Өл„җ лӘЁл“Ҳкіј sysctl

нҢҢл“ң л„ӨнҠёмӣҢнҒ¬ лёҢлҰ¬м§ҖлҘј нҶөкіјн•ҳлҠ” нҠёлһҳн”Ҫмқҙ iptables к·ңм№ҷмқ„ кұ°м№ҳлҸ„лЎқ br_netfilter лӘЁл“Ҳкіј sysctl нҢҢлқјлҜён„°лҘј м„Өм •н•ңлӢӨ.

cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter

cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system
  • overlay: containerdмқҳ overlayfs мҠӨлғ…мғ·н„°м—җ н•„мҡ”н•ҳлӢӨ.
  • br_netfilter + bridge-nf-call-iptables: лёҢлҰ¬м§Җ нҠёлһҳн”Ҫмқҙ iptables(=Service лқјмҡ°нҢ…)лҘј нғҖкІҢ н•ңлӢӨ.
  • ip_forward: л…ёл“ңк°Җ нҢҢл“ң к°„ нҢЁнӮ·мқ„ нҸ¬мӣҢл”©н•ҳлҸ„лЎқ н•ңлӢӨ.

Warning

bridge-nf-call-iptablesк°Җ 0мқҙл©ҙ ClusterIP Serviceк°Җ к°„н—җм ҒмңјлЎң лҸҷмһ‘н•ҳм§Җ м•ҠлҠ” лӮңн•ҙн•ң мһҘм• к°Җ мғқкёҙлӢӨ. м„Өм№ҳ м „м—җ л°ҳл“ңмӢң 1мқём§Җ нҷ•мқён•ңлӢӨ.

м»Ён…Ңмқҙл„Ҳ лҹ°нғҖмһ„ (containerd)

мҝ лІ„л„ӨнӢ°мҠӨлҠ” CRI(Container Runtime Interface)лҘј кө¬нҳ„н•ң лҹ°нғҖмһ„мқ„ мҡ”кө¬н•ңлӢӨ. нҳ„мһ¬ н‘ңмӨҖмқҖ containerdлӢӨ.

# containerd м„Өм№ҳ (л°°нҸ¬нҢҗ нҢЁнӮӨм§Җ лҳҗлҠ” кіөмӢқ л°”мқҙл„ҲлҰ¬)
sudo apt-get update && sudo apt-get install -y containerd

# кё°ліё м„Өм • мғқм„ұ
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml >/dev/null

к°ҖмһҘ нқ”н•ң н•Ём •мқҖ cgroup л“ңлқјмқҙлІ„ л¶Ҳмқјм№ҳлӢӨ. systemd кё°л°ҳ л°°нҸ¬нҢҗм—җм„ңлҠ” kubeletкіј containerd лӘЁл‘җ systemd cgroup л“ңлқјмқҙлІ„лҘј мҚЁм•ј н•ңлӢӨ.

# /etc/containerd/config.toml
[plugins."io.containerd.grpc.v1.cri".containerd.runtimes.runc.options]
  SystemdCgroup = true

sudo systemctl restart containerd
sudo systemctl enable containerd

kubeadmВ·kubeletВ·kubectl м„Өм№ҳ

м„ё нҢЁнӮӨм§ҖлҘј лҸҷмқј л§Ҳмқҙл„Ҳ лІ„м „мңјлЎң м„Өм№ҳн•ңлӢӨ.

# нҢЁнӮӨм§Җ м ҖмһҘмҶҢ м¶”к°Җ (мҳҲ: v1.31)
sudo apt-get update && sudo apt-get install -y apt-transport-https ca-certificates curl gpg

  sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

sudo apt-get install -y kubelet kubeadm kubectl

sudo apt-mark hold kubelet kubeadm kubectl

apt-mark holdлЎң лІ„м „мқ„ кі м •н•ҳлҠ” мқҙмң лҠ” CH12мқҳ лІ„м „ мҠӨнҒҗ м •мұ… л•Ңл¬ёмқҙлӢӨ. м—…к·ёл Ҳмқҙл“ңлҠ” мқҳлҸ„м ҒВ·лӢЁкі„м ҒмңјлЎң н•ҙм•ј н•ңлӢӨ.

3. kubeadm init вҖ” control plane л¶ҖнҠёмҠӨнҠёлһ©

control plane л…ёл“ңм—җм„ң kubeadm initмқ„ мӢӨн–үн•ңлӢӨ. CNIм—җ л”°лқј Pod CIDRмқҙ м •н•ҙм§ҖлҜҖлЎң --pod-network-cidrлҘј лҜёлҰ¬ л§һм¶ҳлӢӨ.

sudo kubeadm init \
  --pod-network-cidr=10.244.0.0/16 \
  --apiserver-advertise-address=10.0.0.10 \
  --control-plane-endpoint=10.0.0.10
  • --pod-network-cidr: нҢҢл“ңм—җ н• лӢ№н•  л„ӨнҠёмӣҢнҒ¬ лҢҖм—ӯ. FlannelмқҖ 10.244.0.0/16мқ„ кё°лҢҖн•ңлӢӨ.
  • --control-plane-endpoint: л©ҖнӢ° control plane(HA)мңјлЎң нҷ•мһҘн•  к°ҖлҠҘм„ұмқҙ мһҲмңјл©ҙ мІҳмқҢл¶Җн„° м§Җм •н•ңлӢӨ. лЎңл“ңл°ёлҹ°м„ң DNSлҘј л„ЈлҠ” кІҢ мқҙмғҒм ҒмқҙлӢӨ.

initмқҙ кұ°м№ҳлҠ” phase

kubeadm initмқҖ лӮҙл¶Җм ҒмңјлЎң м—¬лҹ¬ phaseлҘј мҲңм°Ё мӢӨн–үн•ңлӢӨ. мқҙлҘј мқҙн•ҙн•ҳл©ҙ мӢӨнҢЁ м§Җм җмқ„ м •нҷ•нһҲ м§ҡмқ„ мҲҳ мһҲлӢӨ.

    K->>FS: etcd (м •м Ғ Pod manifest)
    K->>Kubelet: kubelet-start
    Kubelet->>API: м •м Ғ Pod кё°лҸҷ (etcd, apiserver...)
    API-->>K: API мқ‘лӢө лҢҖкё° (healthz)
    K->>API: upload-config / mark-control-plane
    K->>API: bootstrap-token мғқм„ұ
    K->>API: addon (kube-proxy, CoreDNS)

нҠ№м • phaseл§Ң л”°лЎң мӢӨн–үн•  мҲҳлҸ„ мһҲлӢӨ.

# мӮ¬м „ м җкІҖл§Ң мҲҳн–ү
sudo kubeadm init phase preflight
# мқёмҰқм„ңл§Ң лҜёлҰ¬ мғқм„ұ

5. worker л…ёл“ң join

init м¶ңл Ҙ лҒқм—җ join лӘ…л №мқҙ м•ҲлӮҙлҗңлӢӨ. worker л…ёл“ңм—җм„ң к·ёлҢҖлЎң мӢӨн–үн•ңлӢӨ(2мһҘмқҳ мӮ¬м „ мӨҖл№„к°Җ workerм—җлҸ„ лҒқлӮҳ мһҲм–ҙм•ј н•ңлӢӨ).

  --token abcdef.0123456789abcdef \

  • --discovery-token-ca-cert-hash: к°Җм§ң API м„ңлІ„м—җ мҶҚм§Җ м•ҠлҸ„лЎқ CA мқёмҰқм„ңлҘј кІҖмҰқн•ҳлҠ” н•ҙмӢң.
