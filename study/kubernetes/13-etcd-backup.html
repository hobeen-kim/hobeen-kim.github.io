etcd л°ұм—…кіј ліөкө¬

н•ҷмҠө лӘ©н‘ң

  • etcdк°Җ нҒҙлҹ¬мҠӨн„°м—җм„ң л§ЎлҠ” м—ӯн• кіј нӮӨ-к°’ лҚ°мқҙн„° кө¬мЎ°лҘј мқҙн•ҙн•ңлӢӨ.
  • etcdctl snapshot saveлЎң мқјкҙҖлҗң мҠӨлғ…мғ·мқ„ л§Ңл“ңлҠ” л°©лІ•мқ„ мқөнһҢлӢӨ.
  • etcdctl snapshot restoreлЎң мҠӨлғ…мғ·м—җм„ң нҒҙлҹ¬мҠӨн„° мғҒнғңлҘј ліөкө¬н•ҳлҠ” м Ҳм°ЁлҘј мҲҷм§Җн•ңлӢӨ.
  • л°ұм—… мЈјкё°В·ліҙкҙҖВ·кІҖмҰқмқ„ нҸ¬н•Ён•ң мҡҙмҳҒ м „лһөкіј мһ¬н•ҙ ліөкө¬ мӢңлӮҳлҰ¬мҳӨлҘј м„Өкі„н•ңлӢӨ.

1. etcdмқҳ м—ӯн• кіј лҚ°мқҙн„° кө¬мЎ°

etcdлҠ” мҝ лІ„л„ӨнӢ°мҠӨмқҳ лӘЁл“  нҒҙлҹ¬мҠӨн„° мғҒнғңлҘј м ҖмһҘн•ҳлҠ” л¶„мӮ° нӮӨ-к°’ м ҖмһҘмҶҢлӢӨ. PodВ·DeploymentВ·ServiceВ·SecretВ·ConfigMap л“ұ мҡ°лҰ¬к°Җ kubectlлЎң л§Ңл“ңлҠ” лӘЁл“  мҳӨлёҢм қнҠёлҠ” кІ°көӯ etcdм—җ кё°лЎқлҗңлӢӨ. etcdк°Җ нҒҙлҹ¬мҠӨн„°мқҳ мң мқјн•ң м§„мӢӨ кіөкёүмӣҗ(single source of truth)мқҙл©°, etcdлҘј мһғмңјл©ҙ нҒҙлҹ¬мҠӨн„° мғҒнғң м „мІҙлҘј мһғлҠ”лӢӨ.

flowchart LR
    Client["kubectl / м»ЁнҠёлЎӨлҹ¬"] --> API["kube-apiserver"]
    API -->|"мң мқјн•ҳкІҢ etcdмҷҖ м§Ғм ‘ нҶөмӢ "| Etcd[("etcd")]
    subgraph Etcd
        K1["/registry/pods/default/web-xxx"]
        K2["/registry/deployments/default/web"]

н•өмӢ¬ нҠ№м§•мқ„ м •лҰ¬н•ҳл©ҙ мқҙл ҮлӢӨ.

нҠ№м§•м„ӨлӘ…
м ‘к·ј кІҪлЎңмҳӨм§Ғ kube-apiserverл§Ң etcdмҷҖ м§Ғм ‘ нҶөмӢ н•ңлӢӨ. лӢӨлҘё м»ҙнҸ¬л„ҢнҠёлҠ” apiserverлҘј кұ°м№ңлӢӨ.
лҚ°мқҙн„° лӘЁлҚё/registry/<resource>/<namespace>/<name> нҳ•нғңмқҳ нӮӨм—җ м§Ғл ¬нҷ”лҗң мҳӨлёҢм қнҠё м ҖмһҘ
н•©мқҳ м•Ңкі лҰ¬мҰҳRaft. лӢӨмҲҳкІ°(quorum) кё°л°ҳ к°•н•ң мқјкҙҖм„ұ
к¶ҢмһҘ л©ӨлІ„ мҲҳнҷҖмҲҳ(3, 5). quorum = (N/2)+1

etcd м•Ҳмқ„ м§Ғм ‘ л“Өм—¬лӢӨліҙкё°

apiserverлҘј кұ°м№ҳм§Җ м•Ҡкі  etcdм—җ м§Ғм ‘ м§Ҳмқҳн•  мҲҳлҸ„ мһҲлӢӨ(л””лІ„к№…/кІҖмҰқ мҡ©лҸ„).

ETCDCTL_API=3 etcdctl \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key \
  get /registry/ --prefix --keys-only | head

kubeadmмңјлЎң м„Өм№ҳн•ң нҒҙлҹ¬мҠӨн„°м—җм„ң etcdлҠ” control plane л…ёл“ңмқҳ м •м Ғ PodлЎң л–  мһҲкі , мқёмҰқм„ңлҠ” /etc/kubernetes/pki/etcd/м—җ мһҲлӢӨ. мғҒм„ё м„Өм •В·мҡҙмҳҒмқҖ etcd нҒҙлҹ¬мҠӨн„° мҡҙмҳҒ л¬ём„ңлҘј м°ёкі н•ңлӢӨ.

2. etcdctlлЎң л°ұм—… мӨҖл№„

л°ұм—…мқҳ н•өмӢ¬ лҸ„кө¬лҠ” etcdм—җ нҸ¬н•Ёлҗң etcdctlмқҙлӢӨ. API лІ„м „ 3мқ„ лӘ…мӢңн•ҳкі , TLS мқёмҰқм„ңлЎң мқёмҰқн•ңлӢӨ.

мқёмҰқ м •ліҙ нҷҳкІҪліҖмҲҳнҷ”

л§ӨлІҲ кёҙ н”Ңлһҳк·ёлҘј м“°м§Җ м•ҠлҸ„лЎқ нҷҳкІҪліҖмҲҳлЎң л¬¶лҠ”лӢӨ.

export ETCDCTL_API=3
export ETCDCTL_ENDPOINTS=https://127.0.0.1:2379
export ETCDCTL_CACERT=/etc/kubernetes/pki/etcd/ca.crt
export ETCDCTL_CERT=/etc/kubernetes/pki/etcd/server.crt
export ETCDCTL_KEY=/etc/kubernetes/pki/etcd/server.key

л©ӨлІ„В·мғҒнғң нҷ•мқё

л°ұм—… м „м—җ etcdк°Җ кұҙк°•н•ңм§Җ(лҰ¬лҚ”к°Җ мһҲкі  л©ӨлІ„к°Җ м •мғҒмқём§Җ) нҷ•мқён•ңлӢӨ.

etcdctl endpoint health
etcdctl endpoint status --write-out=table
etcdctl member list -w table

endpoint statusмқҳ DB SIZEлЎң лҚ°мқҙн„° м–‘мқ„ к°ҖлҠ н•ҳкі , IS LEADERлЎң лҰ¬лҚ”лҘј нҷ•мқён•ңлӢӨ.

3. etcdctl snapshot save

etcdctl snapshot save /backup/etcd-snapshot-$(date +%F-%H%M).db

мқҙ лӘ…л №мқҖ etcdмқҳ нҳ„мһ¬ мғҒнғңлҘј мқјкҙҖлҗң лӢЁмқј нҢҢмқј(.db)лЎң л– лӮёлӢӨ. etcdлҠ” MVCC кё°л°ҳмқҙлқј мҠӨлғ…мғ·мқ„ лңЁлҠ” лҸҷм•Ҳм—җлҸ„ м“°кё°к°Җ к°ҖлҠҘн•ҳл©°, мҠӨлғ…мғ·мқҖ лӘ…л № мӢӨн–ү мӢңм җмқҳ мқјкҙҖлҗң мҠӨлғ…мҲҸмқҙлӢӨ.

мҠӨлғ…мғ· кІҖмҰқ

мҠӨлғ…мғ·мқҙ мҶҗмғҒлҗҳм§Җ м•Ҡм•ҳлҠ”м§Җ н•ҙмӢңВ·лҰ¬л№„м „мқ„ нҷ•мқён•ңлӢӨ.

4. snapshot restore вҖ” ліөкө¬

ліөкө¬лҠ” "мҠӨлғ…мғ·м—җм„ң мғҲ etcd лҚ°мқҙн„° л””л үнҶ лҰ¬лҘј л§Ңл“  л’Ө, apiserverк°Җ к·ё л””л үнҶ лҰ¬лҘј ліҙкІҢ н•ҳлҠ”" мһ‘м—…мқҙлӢӨ.

sequenceDiagram
    participant Op as мҡҙмҳҒмһҗ
    participant Static as kubelet(м •м Ғ Pod)
    participant Etcd as etcd

    Static->>API: apiserver Pod мӨ‘м§Җ

    Static->>Etcd: мғҲ data-dirлЎң etcd кё°лҸҷ
    Static->>API: apiserver мһ¬кё°лҸҷ
    API->>Etcd: ліөкө¬лҗң мғҒнғң лЎңл“ң
    API-->>Op: нҒҙлҹ¬мҠӨн„° мғҒнғң ліөмӣҗ мҷ„лЈҢ

ліөкө¬ м Ҳм°Ё

# 1) control plane м •м§Җ вҖ” м •м Ғ Pod manifestлҘј мһ мӢң л№јлӮёлӢӨ

sudo mv /etc/kubernetes/manifests/etcd.yaml \
        /etc/kubernetes/manifests/kube-apiserver.yaml \
        /etc/kubernetes/manifests-backup/

# 2) мҠӨлғ…мғ·м—җм„ң мғҲ лҚ°мқҙн„° л””л үнҶ лҰ¬ ліөмӣҗ
sudo ETCDCTL_API=3 etcdctl snapshot restore \
  /backup/etcd-snapshot-2026-06-15-0300.db \
  --data-dir=/var/lib/etcd-restored

# 3) etcd manifestмқҳ hostPath(volume) кІҪлЎңлҘј /var/lib/etcd-restoredлЎң мҲҳм •
#    (etcd.yamlмқҳ etcd-data volume hostPath)

# 4) manifest ліөк·Җ вҖ” kubeletмқҙ к°җм§Җн•ҙ etcd/apiserver мһ¬кё°лҸҷ
sudo mv /etc/kubernetes/manifests-backup/etcd.yaml \
        /etc/kubernetes/manifests-backup/kube-apiserver.yaml \
        /etc/kubernetes/manifests/

мҠӨлғ…мғ· мқҙнӣ„ мғқм„ұлҗң мҳӨлёҢм қнҠёлҠ” мӮ¬лқјм§Җкі , мҠӨлғ…мғ· мӢңм җ мғҒнғңлЎң лҗҳлҸҢм•„к°„лӢӨ. мқҙкІғмқҙ RPO(н—Ҳмҡ© лҚ°мқҙн„° мҶҗмӢӨ)к°Җ л°ұм—… мЈјкё°м—җ м§Ғм ‘ л¬¶мқҙлҠ” мқҙмң лӢӨ.

5. л°ұм—… мҡҙмҳҒ м „лһө

мҠӨлғ…мғ·мқ„ "н•ң лІҲ л– ліё м Ғ мһҲлӢӨ"мҷҖ "мҡҙмҳҒ м „лһөмқҙ мһҲлӢӨ"лҠ” лӢӨлҘҙлӢӨ. лӢӨмқҢ н•ӯлӘ©мқ„ м •мұ…мңјлЎң лӘ» л°•лҠ”лӢӨ.

Warning

etcd мҠӨлғ…мғ·м—җлҠ” лӘЁл“  Secretмқҙ л“Өм–ҙ мһҲлӢӨ. м ҖмһҘ мӢң etcd мһҗмІҙлҘј м•”нҳёнҷ”(EncryptionConfiguration)н•ҳкұ°лӮҳ, мҠӨлғ…мғ· нҢҢмқјмқ„ м•”нҳёнҷ”н•ҙ ліҙкҙҖн•ңлӢӨ. мһҗм„ён•ң лӮҙмҡ©мқҖ CH36. кіөкёүл§қВ·лҹ°нғҖмһ„ ліҙм•Ҳм—җм„ң лӢӨлЈ¬лӢӨ.

ліөкө¬ нӣҲл Ё

"ліөкө¬н•ҙ ліё м Ғ м—ҶлҠ” л°ұм—…"мқҖ л°ұм—…мқҙ м•„лӢҲлӢӨ.

  • мЈјкё°м ҒмңјлЎң кІ©лҰ¬ нҷҳкІҪм—җ мҠӨлғ…мғ·мқ„ ліөкө¬н•ҙ ліёлӢӨ.
  • ліөкө¬ мҶҢмҡ” мӢңк°„(RTO)мқ„ мӢӨмёЎн•ҙ лӘ©н‘ңм№ҳмҷҖ л№„көҗн•ңлӢӨ.
  • м Ҳм°Ём„ңлҘј к·ёлҢҖлЎң л”°лқј н–Ҳмқ„ л•Ң л§үнһҲлҠ” м§Җм җмқ„ л¬ём„ңм—җ л°ҳмҳҒн•ңлӢӨ.

6. мһ¬н•ҙ ліөкө¬ мӢңлӮҳлҰ¬мҳӨ

мғҒнҷ©лі„лЎң л¬ҙм—Үмқ„ ліөкө¬ мҲҳлӢЁмңјлЎң м“°лҠ”м§Җ м •лҰ¬н•ңлӢӨ.

мӮ¬кұҙліөкө¬ л°©лІ•мҳҲмғҒ RTOмҳҲмғҒ RPO
мӢӨмҲҳлЎң лҰ¬мҶҢмҠӨ лҢҖлҹү мӮӯм ңмөңк·ј мҠӨлғ…мғ· restore10~30л¶„л§Ҳм§Җл§ү мҠӨлғ…мғ· мӢңм җ
etcd лҚ°мқҙн„° мҶҗмғҒмҠӨлғ…мғ· restore30л¶„~мҲҳ мӢңк°„л§Ҳм§Җл§ү мҠӨлғ…мғ· мӢңм җ
control plane л…ёл“ң мҶҢмӢӨ(лӢЁмқј)л…ёл“ң мһ¬кө¬м„ұ + мҠӨлғ…мғ· restoreмҲҳ мӢңк°„л§Ҳм§Җл§ү мҠӨлғ…мғ· мӢңм җ
etcd л©ӨлІ„ 1к°ң мһҘм• (3л…ёл“ң)л©ӨлІ„ көҗмІҙ(ліөкө¬ л¶Ҳн•„мҡ”)мҲҳ л¶„0(quorum мң м§Җ)
м „мІҙ etcd нҒҙлҹ¬мҠӨн„° мҶҢмӢӨмҠӨлғ…мғ·м—җм„ң мғҲ нҒҙлҹ¬мҠӨн„° мһ¬кө¬м„ұмҲҳ мӢңк°„л§Ҳм§Җл§ү мҠӨлғ…мғ· мӢңм җ

л©ҖнӢ° л©ӨлІ„ etcdмқҳ нҡҢліөл Ҙ

3л…ёл“ң etcdлҠ” 1л…ёл“ңк°Җ мЈҪм–ҙлҸ„ quorum(2/3)мқҙ мң м§Җлҗҳм–ҙ нҒҙлҹ¬мҠӨн„°к°Җ кі„мҶҚ лҸҷмһ‘н•ңлӢӨ. мқҙ кІҪмҡ°лҠ” ліөкө¬к°Җ м•„лӢҲлқј л©ӨлІ„ көҗмІҙлЎң мІҳлҰ¬н•ңлӢӨ.

# мЈҪмқҖ л©ӨлІ„ м ңкұ°
etcdctl member remove <member-id>
# мғҲ л©ӨлІ„ м¶”к°Җ
etcdctl member add etcd-new --peer-urls=https://10.0.0.13:2380

мҠӨлғ…мғ· ліөкө¬лҠ” quorumмқҙ к№ЁмЎҢкұ°лӮҳ(кіјл°ҳ л©ӨлІ„ лҸҷмӢң мҶҢмӢӨ) лҚ°мқҙн„° мһҗмІҙк°Җ мҶҗмғҒлҗҗмқ„ л•Ңмқҳ мөңнӣ„ мҲҳлӢЁмқҙлӢӨ. к·ёлһҳм„ң мҡҙмҳҒ к¶Ңкі лҠ” л‘җ кІ№мқҙлӢӨ вҖ” нҸүмҶҢм—” нҷҖмҲҳ л©ӨлІ„лЎң нҡҢліөл Ҙмқ„ нҷ•ліҙн•ҳкі , к·ёлһҳлҸ„ л¬ҙл„ҲмЎҢмқ„ л•ҢлҘј мң„н•ҙ мҠӨлғ…мғ·мқ„ л‘”лӢӨ.

ліөкө¬ м Ҳм°Ём„ң мҳҲмӢң

## мӮ¬м „ м •ліҙ
- мҠӨлғ…мғ· мң„м№ҳ: s3://backups/etcd/ (мӢңк°„лі„)
- etcd мқёмҰқм„ң: /etc/kubernetes/pki/etcd/

## RTO/RPO лӘ©н‘ң
- RTO: 1мӢңк°„
- RPO: 1мӢңк°„ (л§ӨмӢң мҠӨлғ…мғ·)

## м Ҳм°Ё
1. мөңмӢ  мҠӨлғ…мғ· лӢӨмҡҙлЎңл“ң + snapshot statusлЎң л¬ҙкІ°м„ұ нҷ•мқё
2. control plane manifest мқҙлҸҷ(etcd/apiserver м •м§Җ)
3. etcdctl snapshot restore --data-dir
4. etcd manifest hostPath к°ұмӢ  нӣ„ manifest ліөк·Җ
5. kubectl get nodes / get pods -A лЎң кІҖмҰқ

## кІҖмҰқ нӣҲл Ё
- л¶„кё° 1нҡҢ кІ©лҰ¬ нҷҳкІҪ ліөкө¬, RTO мӢӨмёЎ кё°лЎқ

н•өмӢ¬ м •лҰ¬

  • etcdлҠ” нҒҙлҹ¬мҠӨн„°мқҳ мң мқјн•ң м§„мӢӨ кіөкёүмӣҗмқҙл©°, мҳӨм§Ғ kube-apiserverл§Ң м§Ғм ‘ нҶөмӢ н•ңлӢӨ. etcdлҘј мһғмңјл©ҙ нҒҙлҹ¬мҠӨн„° мғҒнғң м „мІҙлҘј мһғлҠ”лӢӨ.
  • л°ұм—…мқҖ л°ҳл“ңмӢң etcdctl snapshot saveлЎң мқјкҙҖлҗң мҠӨлғ…мғ·мқ„ л§Ңл“ лӢӨ. нҢҢмқј ліөмӮ¬лҠ” мқјкҙҖм„ұмқ„ ліҙмһҘн•ҳм§Җ лӘ»н•ңлӢӨ.
  • ліөкө¬лҠ” control plane м •м§Җ вҶ’ snapshot restoreлЎң мғҲ data-dir мғқм„ұ вҶ’ etcd manifest кІҪлЎң к°ұмӢ  вҶ’ мһ¬кё°лҸҷ мҲңм„ңлӢӨ.
  • мҠӨлғ…мғ·м—җлҠ” лӘЁл“  Secretмқҙ л“Өм–ҙ мһҲмңјлҜҖлЎң м•”нҳёнҷ”В·мҳӨн”„мӮ¬мқҙнҠё ліҙкҙҖВ·ліөкө¬ нӣҲл Ёмқ„ мҡҙмҳҒ м •мұ…мңјлЎң лӘ» л°•лҠ”лӢӨ.
  • нҷҖмҲҳ л©ӨлІ„(3/5)лЎң нҡҢліөл Ҙмқ„ нҷ•ліҙн•ҳл©ҙ 1л…ёл“ң мһҘм• лҠ” л©ӨлІ„ көҗмІҙлЎң мІҳлҰ¬лҗҳкі , мҠӨлғ…мғ· ліөкө¬лҠ” quorumмқҙ к№Ём§„ мөңнӣ„мқҳ мҲҳлӢЁмңјлЎң лӮЁлҠ”лӢӨ.

лӢӨмқҢ мұ•н„°

etcd л°ұм—…мңјлЎң лҚ°мқҙн„° мёЎл©ҙмқҳ м•Ҳм „л§қмқ„ к°–м·„лӢӨ. к·ёлҹ°лҚ° etcdВ·apiserverВ·kubeletмқҖ лӘЁл‘җ мқёмҰқм„ңлЎң м„ңлЎңлҘј мӢ лў°н•ңлӢӨ. мқҙ мқёмҰқм„ңк°Җ м–ҙл–»кІҢ кө¬м„ұлҗҳкі  л§ҢлЈҢлҗҳл©° к°ұмӢ лҗҳлҠ”м§Җ лӘЁлҘҙл©ҙ м–ҙлҠҗ лӮ  нҒҙлҹ¬мҠӨн„°к°Җ нҶөм§ёлЎң л©Ҳм¶ң мҲҳ мһҲлӢӨ. CH14. мқёмҰқм„ңВ·PKIВ·kubeconfigм—җм„ң нҒҙлҹ¬мҠӨн„° PKI кө¬мЎ°, мқёмҰқм„ң к°ұмӢ , kubeconfig кө¬мЎ°лҘј лӢӨлЈ¬лӢӨ.

  • лӢӨмқҢ: CH14. мқёмҰқм„ңВ·PKIВ·kubeconfig
