н•ҷмҠө лӘ©н‘ң

  • Protocol Mapperмқҳ лӮҙмһҘ нғҖмһ…лі„ лҸҷмһ‘мқ„ мқҙн•ҙн•ҳкі  н•„мҡ”н•ң нҒҙл Ҳмһ„мқ„ м„ нғқн•  мҲҳ мһҲлӢӨ.
  • User AttributeлҘј нҶ нҒ°м—җ мӢЈлҠ” м»ӨмҠӨн…Җ нҒҙл Ҳмһ„ м¶”к°Җ кіјм •мқ„ кө¬м„ұн•  мҲҳ мһҲлӢӨ.

1. Client Scope вҖ” Default Scopes vs Optional Scopes

Keycloakмқҳ Client ScopeлҠ” Clientм—җ мһ¬мӮ¬мҡ© к°ҖлҠҘн•ң Protocol MapperмҷҖ Role Scope Mapping л¬¶мқҢмқҙлӢӨ. Clientл§ҲлӢӨ лҸҷмқјн•ң м„Өм •мқ„ л°ҳліөн•ҙм„ң л§Ңл“Өм§Җ м•ҠлҸ„лЎқ, кіөнҶө нҶ нҒ° м„Өкі„ лӢЁмң„лҘј лЁјм Җ м •мқҳн•ҳкі  м—¬лҹ¬ Clientм—җ л¶ҷмқҙлҠ” л°©мӢқмқҙлӢӨ.

Scopeмқҳ л‘җ м¶•

н•ҳлӮҳмқҳ Clientм—җлҠ” Client ScopeлҘј л‘җ к°Җм§Җ л°©мӢқмңјлЎң м—°кІ°н•ңлӢӨ.

кө¬л¶„нҶ нҒ° л°ңкёү мЎ°кұҙмӮ¬мҡ©мһҗ лҸҷмқҳлҢҖн‘ң мҡ©лҸ„
Default Client Scopesн•ӯмғҒ нҸ¬н•Ёлі„лҸ„ лҸҷмқҳ м—ҶмқҢлӮҙл¶Җ кё°ліё нҒҙл Ҳмһ„(roles, web-origins)
Optional Client Scopesscope нҢҢлқјлҜён„°лЎң мҡ”мІӯлҗ  л•Ңл§ҢConsent Required мӢң н‘ңмӢңprofile, email, address, phone

scope=openid profile emailмІҳлҹј мҡ”мІӯлҗң к°’мқҙ Optional Scopeм—җ л§Өм№ӯлҗҳл©ҙ н•ҙлӢ№ Scopeмқҳ Mapperл“Өмқҙ нҶ нҒ°м—җ м Ғмҡ©лҗңлӢӨ. мҡ”мІӯлҗҳм§Җ м•ҠмқҖ Optional ScopeлҠ” нҸүк°Җ лҢҖмғҒм—җм„ң м ңмҷёлҗңлӢӨ.

Realm-level vs Client-level

Client ScopeлҠ” л‘җ кі„мёөмңјлЎң кҙҖлҰ¬н•ңлӢӨ.

  • Realm-level Client Scope: Realm м „мІҙм—җм„ң м—¬лҹ¬ Clientк°Җ кіөмң н•ҳлҠ” ScopeлӢӨ. openid/profile/email к°ҷмқҖ н‘ңмӨҖ ScopeлҠ” Realm мҲҳмӨҖмңјлЎң мқҙлҜё м ңкіөлҗңлӢӨ.
  • Client-level (Dedicated) Scope: к°ңлі„ Client м „мҡ© ScopeлӢӨ. Admin Consoleмқҳ Client мғҒм„ё нҷ”л©ҙм—җм„ң Dedicated scopeм—җ MapperлҘј м§Ғм ‘ м¶”к°Җн•  мҲҳ мһҲлӢӨ.

кіөнҶө нҒҙл Ҳмһ„мқҖ Realm-level ScopeлЎң лӘЁл“Ҳнҷ”н•ҳкі , нҠ№м • Clientм—җл§Ң н•„мҡ”н•ң к·ңм№ҷмқҖ Dedicated ScopeлЎң кІ©лҰ¬н•ҳлҠ” кІғмқҙ мҡҙмҳҒмғҒ к№”лҒ”н•ҳлӢӨ.

2. Protocol Mapper лӮҙмһҘ нғҖмһ…

Protocol MapperлҠ” мӮ¬мҡ©мһҗ лӘЁлҚёмқҳ к°’мқ„ нҶ нҒ° нҒҙл Ҳмһ„мңјлЎң ліҖнҷҳн•ҳлҠ” к·ңм№ҷмқҙлӢӨ. KeycloakмқҖ OIDCмҷҖ SAML к°Ғк°Ғм—җ лҢҖн•ҙ м—¬лҹ¬ лӮҙмһҘ Mapper нғҖмһ…мқ„ м ңкіөн•ңлӢӨ. м—¬кё°м„ңлҠ” OIDC Mapper мӨ‘ мһҗмЈј м“°мқҙлҠ” нғҖмһ…мқ„ м •лҰ¬н•ңлӢӨ.

Script Mapperмқҳ нҳ„мӢӨ

Script MapperлҠ” мң м—°н•ҳм§Җл§Ң кё°ліём ҒмңјлЎң л№„нҷңм„ұнҷ”лҗң Preview кё°лҠҘмқҙлӢӨ. нҷңм„ұнҷ”н•ҳл Өл©ҙ м„ңлІ„ мӢңмһ‘ мӢң --features=scripts н”Ңлһҳк·ёлҘј мјңм•ј н•ҳкі , ліҙм•ҲмғҒ JS мӢӨн–ү нҷҳкІҪмқҖ лІҲл“Өлҗң Nashorn кё°л°ҳмқҙлқј ECMAScript лІ„м „кіј м„ұлҠҘ м ңм•Ҫмқҙ мһҲлӢӨ. мҡҙмҳҒ нҷҳкІҪм—җм„ңлҠ” CH15 SPIм—җм„ң лӢӨлЈ° м»ӨмҠӨн…Җ Mapper ProviderлЎң лҢҖмІҙн•ҳлҠ” кІғмқҙ мқјл°ҳм ҒмқҙлӢӨ.

Audience Mapperмқҳ мӨ‘мҡ”м„ұ

Access Tokenмқҳ aud нҒҙл Ҳмһ„мқҖ "мқҙ нҶ нҒ°мқҙ м–ҙлҠҗ Resource ServerлҘј лҢҖмғҒмңјлЎң н•ҳлҠ”к°Җ"лҘј лң»н•ңлӢӨ. Resource Serverк°Җ JWTлҘј кІҖмҰқн•  л•Ң мһҗмӢ мқҙ кё°лҢҖн•ҳлҠ” audмҷҖ мқјм№ҳн•ҳм§Җ м•Ҡмңјл©ҙ кұ°л¶Җн•ҙм•ј н•ңлӢӨ. н•ҳлӮҳмқҳ Clientк°Җ м—¬лҹ¬ APIлҘј нҳём¶ңн•ңлӢӨл©ҙ Audience MapperлЎң audм—җ н•ҙлӢ№ APIмқҳ Client IDлҘј нҸ¬н•ЁмӢңнӮЁлӢӨ. мһҗм„ён•ң л°°кІҪмқҖ OAuth CH10. ID Tokenкіј JWTлҘј м°ёмЎ°н•ңлӢӨ.

4. ConsentмҷҖ Scope

мӮ¬мҡ©мһҗк°Җ нҠ№м • Clientм—җ мІҳмқҢ лЎңк·ёмқён•  л•Ң, "мқҙ м•ұмқҙ лӢ№мӢ мқҳ мқҙл©”мқјВ·н”„лЎңн•„м—җ м ‘к·јн•©лӢҲлӢӨ. лҸҷмқҳн•ҳмӢңкІ мҠөлӢҲк№Ң?" нҷ”л©ҙмқ„ ліё м Ғмқҙ мһҲмқ„ кІғмқҙлӢӨ. мқҙ лҸҷмқҳ(Consent) нҷ”л©ҙмқҳ лӮҙмҡ©мқҖ к·ёлҢҖлЎң Client Scopeк°Җ нҶөм ңн•ңлӢӨ.

Consent Required нҶ кёҖ

Client м„Өм •мқҳ Consent RequiredлҘј OnмңјлЎң л°”кҫёл©ҙ мӮ¬мҡ©мһҗлҠ” мөңмҙҲ лЎңк·ёмқё мӢң Scope лӘ©лЎқмқ„ нҷ•мқён•ҳкі  мҠ№мқён•ҙм•ј н•ңлӢӨ. к°Ғ Scopeмқҳ Display on consent screenкіј Consent screen textк°Җ мқҙ нҷ”л©ҙм—җ н‘ңмӢңлҗҳлҠ” л Ҳмқҙлё”мқҙлӢӨ.

м–ём ң Consentк°Җ н•„мҡ”н•ңк°Җ

лӮҙл¶Җ м „мҡ© Client(мӮ¬лӮҙ мҡҙмҳҒ мҪҳмҶ”, м„ңл№„мҠӨ к°„ нҳём¶ң)лҠ” Consent RequiredлҘј кәјл‘җлҠ” кІғмқҙ н‘ңмӨҖмқҙлӢӨ. л°ҳлҢҖлЎң м„ңл“ңнҢҢнӢ° м•ұм—җкІҢ APIлҘј к°ңл°©н•ҳлҠ” кІҪмҡ°м—җлҠ” л°ҳл“ңмӢң мјңм•ј н•ҳл©°, мқҙлҠ” OAuth ліёлһҳмқҳ "мӮ¬мҡ©мһҗ лҸҷмқҳ мң„мһ„" мӣҗм№ҷмқ„ м§ҖнӮӨлҠ” кІғмқҙкё°лҸ„ н•ҳлӢӨ(OAuth CH1. нҢЁмҠӨмӣҢл“ң кіөмң  мӢңлҢҖ м°ёмЎ°).

Consent мқҙл Ҙкіј мһ¬лҸҷмқҳ

н•ң лІҲ лҸҷмқҳн•ң ClientВ·Scope мЎ°н•©мқҖ мӮ¬мҡ©мһҗ кі„м •м—җ кё°лЎқлҗңлӢӨ. мӮ¬мҡ©мһҗлҠ” Account Consoleмқҳ Linked applicationsм—җм„ң лҸҷмқҳлҘј мІ нҡҢн•  мҲҳ мһҲкі , кҙҖлҰ¬мһҗлҠ” Admin REST APIлЎң мқјкҙ„ мІ нҡҢн•  мҲҳ мһҲлӢӨ. мғҲ Scopeк°Җ м¶”к°Җлҗҳл©ҙ лӢӨмқҢ лЎңк·ёмқё мӢң "м¶”к°Җ лҸҷмқҳ"л§Ң л”°лЎң л¬»лҠ”лӢӨ.

Offline Access Scope

offline_access ScopeлҠ” Consent нҷ”л©ҙм—җм„ңлҸ„ нҠ№лі„н•ҳкІҢ м·ЁкёүлҗңлӢӨ. мқҙ Scopeк°Җ мҠ№мқёлҗҳл©ҙ Refresh Tokenмқҙ м„ём…ҳ л§ҢлЈҢ мқҙнӣ„м—җлҸ„ мң нҡЁн•ң Offline TokenмңјлЎң л°ңкёүлҗңлӢӨ. мһҘкё° л°ұк·ёлқјмҡҙл“ң мһ‘м—…(нҒ¬лЎ , л°°м№ҳ)м—җ м“°мқҙл©°, ліҙм•ҲмғҒ Consent лӘ…мӢң лҸҷмқҳлҘј к°•м ңн•ҳлҠ” кІғмқҙ мӣҗм№ҷмқҙлӢӨ. Refresh Token м „лһөмқҖ OAuth CH13. нҶ нҒ° м „лһөм—җм„ң лӢӨлЈ¬лӢӨ.

5. ID Token vs Access Token vs UserInfo л¶„кё°

Protocol Mapperл§ҲлӢӨ "м–ҙлҠҗ нҶ нҒ°/мқ‘лӢөм—җ нҸ¬н•Ён• м§Җ"лҘј к°ңлі„ нҶ кёҖлЎң м ңм–ҙн•ңлӢӨ. лӢЁмҲңн•ҙ ліҙмқҙм§Җл§Ң мқҙ л¶„кё°лҠ” OIDC н‘ңмӨҖмқ„ мҳ¬л°”лҘҙкІҢ л”°лҘҙлҠ” н•өмӢ¬мқҙлӢӨ.

м„ё мұ„л„җмқҳ мҡ©лҸ„

мұ„л„җмҶҢл№„мһҗлӢҙкё°м—җ м Ғм Ҳн•ң кІғ
ID TokenClient(н”„лЎ нҠём—”л“ң)мӮ¬мҡ©мһҗ мӢқлі„(sub), мқёмҰқ мӮ¬мӢӨ(iss, aud, auth_time)
Access TokenResource Serverк¶Ңн•ң нҢҗлӢЁм—җ н•„мҡ”н•ң мөңмҶҢ м •ліҙ(sub, scope, aud, Role)

л¶„кё° мӣҗм№ҷ

  • ID TokenмқҖ Clientк°Җ лЎңк·ёмқё м„ұкіө м—¬л¶ҖлҘј нҷ•мқён•ҳл ӨлҠ” мҡ©лҸ„лӢӨ. мөңмҶҢн•ңмқҳ мӮ¬мҡ©мһҗ мӢқлі„ нҒҙл Ҳмһ„л§Ң лӢҙлҠ”лӢӨ.
  • Access TokenмқҖ API нҳём¶ң мӢңл§ҲлӢӨ л„ӨнҠёмӣҢнҒ¬лҘј мҳӨк°„лӢӨ. л„Ҳл¬ҙ л§ҺмқҖ нҒҙл Ҳмһ„мқ„ лӢҙмңјл©ҙ н—ӨлҚ” нҒ¬кё° л¬ём ң(нҠ№нһҲ мҝ нӮӨ кё°л°ҳ BFF нҢЁн„ҙм—җм„ң)мҷҖ нҶ нҒ° лЎңк·ё лҲ„м¶ң мң„н—ҳмқҙ м»Өм§„лӢӨ.
  • UserInfoлҠ” Access TokenмңјлЎң мЎ°нҡҢн•ҳлҠ” лі„лҸ„ м—”л“ңнҸ¬мқёнҠёлӢӨ. мһҗмЈј ліҖн•ҳкұ°лӮҳ лҜјк°җн•ң н”„лЎңн•„мқҖ м—¬кё°м„ңл§Ң л…ём¶ңн•ҳкі  Access Tokenм—җм„ңлҠ” л№јлҠ” м „лһөмқҙ л§ҺлӢӨ.

  • Access Tokenм—җ email лӢҙкё°: мқҙл©”мқјмқҖ к¶Ңн•ң нҢҗлӢЁм—җ н•„мҡ”н•ң м •ліҙк°Җ м•„лӢҲлӢӨ. APIк°Җ мӢқлі„мқҙ н•„мҡ”н•ҳл©ҙ subлЎң м¶©л¶„н•ҳкі , н‘ңмӢңмҡ© мқҙлҰ„мқҖ UserInfoм—җм„ң к°Җм ёмҳЁлӢӨ.
  • ID Tokenм—җ Role лӢҙкё°: ID TokenмқҖ "лЎңк·ёмқё мӮ¬мӢӨ мҰқлӘ…"мқҙм§Җ "к¶Ңн•ң мЈјмһҘ"мқҙ м•„лӢҲлӢӨ. RoleмқҖ Access Tokenм—җ лӢҙлҠ” кІғмқҙ л§һлӢӨ.

н‘ңмӨҖмқ„ м§Җмјңм•ј н•ҳлҠ” мқҙмң 

  • мғҒнҳё мҡҙмҡ©м„ұ: Spring Security, Auth.js, oidc-client-ts к°ҷмқҖ лқјмқҙлёҢлҹ¬лҰ¬лҠ” н‘ңмӨҖ Scope/нҒҙл Ҳмһ„ мқҙлҰ„мқ„ м „м ңлЎң нҢҢмӢұн•ңлӢӨ. emailмқ„ user_emailлЎң л°”кҫёлҠ” мҲңк°„ нҳёнҷҳмқҙ к№Ём§„лӢӨ.
  • Discovery мқјкҙҖм„ұ: /.well-known/openid-configurationмқҳ scopes_supportedмҷҖ claims_supportedлҠ” мӢӨм ң Client Scope кө¬м„ұкіј мқјм№ҳн•ҙм•ј н•ңлӢӨ.
  • к°җмӮ¬В·лЎңк·ё к·ңм•Ҫ: SIEM нҲҙкіј к°җмӮ¬ лЎңк·ё нҢҢмқҙн”„лқјмқёмқҙ н‘ңмӨҖ нҒҙл Ҳмһ„ мқҙлҰ„мқ„ м „м ңлЎң н•„л“ңлҘј л§Өн•‘н•ңлӢӨ.

м»ӨмҠӨн…Җ нҒҙл Ҳмһ„ л„Өмқҙл°Қ к·ңм№ҷ

н‘ңмӨҖ нҒҙл Ҳмһ„кіј м¶©лҸҢмқ„ н”јн•ҳл Өл©ҙ м»ӨмҠӨн…Җ нҒҙл Ҳмһ„мқҖ л„Өмһ„мҠӨнҺҳмқҙмҠӨлҘј м“°лҠ” кІғмқҙ м•Ҳм „н•ҳлӢӨ. мҳҲ: corp.department, https://corp.io/employee_number. URI нҳ•нғңлҠ” кёём§Җл§Ң м¶©лҸҢ к°ҖлҠҘм„ұмқҙ 0м—җ к°Җк№ҢмӣҢ кё°м—… мӢңмҠӨн…ңм—җм„ң мһҗмЈј м“ҙлӢӨ.

н‘ңмӨҖ vs м»ӨмҠӨн…Җ м„Өкі„ м§Җм№Ё

  • н‘ңмӨҖ ScopeлЎң н‘ңнҳ„лҗҳлҠ” кІғмқҖ м ҲлҢҖ лҰ¬л„Өмқҙл°Қн•ҳм§Җ л§җ кІғ вҖ” emailмқҖ emailмқҙм–ҙм•ј н•ңлӢӨ.
  • мӮ¬лӮҙ м „мҡ© нҒҙл Ҳмһ„л§Ң м»ӨмҠӨн…Җ Scope(corp-profile, corp-hr)лЎң л¬¶кі , л„Өмһ„мҠӨнҺҳмқҙмҠӨлҘј л¶Җм—¬н•ңлӢӨ.
  • н•ҳлӮҳмқҳ Scopeм—җ л„Ҳл¬ҙ л§ҺмқҖ MapperлҘј л¶ҷмқҙм§Җ л§җкі  м—ӯн• лі„лЎң мӘјк° лӢӨ(corp-hr, corp-billing).

  • Mapper мӮӯм ңлҠ” мҰүмӢң лӘЁл“  мӢ к·ң нҶ нҒ°м—җ л°ҳмҳҒлҗҳм§Җл§Ң, мқҙлҜё л°ңкёүлҗң нҶ нҒ°мқҖ л§ҢлЈҢк№Ңм§Җ мң нҡЁ вҖ” лҜјк°җ лҚ°мқҙн„° м ңкұ° мӢң нҶ нҒ° нҸҗкё°(Backchannel Logout л“ұ) лі‘н–ү.

  • к°Ғ MapperлҠ” ID TokenВ·Access TokenВ·UserInfoм—җ лҢҖн•ң нҸ¬н•Ё м—¬л¶ҖлҘј к°ңлі„ нҶ кёҖлЎң м ңм–ҙн•ҳлҜҖлЎң, мҡ©лҸ„лі„лЎң лӢӨлҘё нҒҙл Ҳмһ„ м„ёнҠёлҘј кө¬м„ұн•ҙ нҶ нҒ° нҒ¬кё°мҷҖ л…ём¶ң лІ”мң„лҘј мөңмҶҢнҷ”н•ҙм•ј н•ңлӢӨ.

  • openid/profile/email/address/phone/offline_accessлҠ” OIDC н‘ңмӨҖ ScopeлЎң лқјмқҙлёҢлҹ¬лҰ¬ нҳёнҷҳм„ұмқ„ мң„н•ҙ лҰ¬л„Өмқҙл°Қн•ҳм§Җ л§җкі , мӮ¬лӮҙ м „мҡ© нҒҙл Ҳмһ„мқҖ л„Өмһ„мҠӨнҺҳмқҙмҠӨ нҳ•нғңмқҳ м»ӨмҠӨн…Җ ScopeлЎң л¶„лҰ¬н•ңлӢӨ.
