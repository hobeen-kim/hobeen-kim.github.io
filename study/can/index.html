CAN 통신 심화

ISOBUS 스터디의 CH1~CH7이 CAN 프로토콜의 입문이었다면, 이 스터디는 그 아래 전기적 특성부터 위로는 차량 진단·보안까지 파고든다. 차량·농기계·산업 임베디드 개발자가 회로·타이밍·드라이버·진단·보안을 한 번에 갖추는 것을 목표로 한다.

선행 지식

  • ISOBUS 스터디 CH1~CH7 — CAN 입문: 통신 기초, 프레임, 중재, 에러, CAN FD 개요
  • 이 스터디는 위 입문의 모든 장을 전제로 전기·타이밍·구현·진단·보안 심화로 진행한다

목차

전기·물리 심화

  1. 차동 신호와 버스 전기 — CANH/CANL, 공통 모드, 종단 저항, 스터브
  2. 파형·계측 — 오실로스코프·로직 분석기, Eye Diagram
  3. 물리 variant — HS-CAN, LS-CAN, single-wire, 광 절연

비트 타이밍

  1. Bit Time 구조 — Sync/Prop/Phase1/Phase2, SJW, TQ
  2. 동기화 — Hard Sync, Resync, 샘플 포인트, 오실레이터 편차

프레임 심화

  1. Classical CAN 심화 — Stuff/CRC 계산·Remote·Overload·Error 프레임·중재 엣지케이스
  2. CAN FD 심화 — BRS/ESI, 향상된 CRC, 샘플 포인트 분리
  3. CAN XL 개요 — SDT, PWME, 차량 이더넷·TSN 경계

신뢰성

  1. 오류 감지·격리 — 5종 오류, TEC/REC, 상태 전이
  2. 버스 부하·응답시간 — Worst-case response, bus load 계산

구현

  1. 컨트롤러·트랜시버 — MCP2515, STM32 bxCAN/FDCAN, TJA1043
  2. MCU 드라이버 — ISR·FIFO·에러 처리 (C)
  3. SocketCAN 기초 — can-utils, can0, python-can

고급 구현과 툴

  1. SocketCAN 고급 — vcan, cangw, isotp 커널 모듈, FD
  2. 하드웨어 툴 — Kvaser/Vector/PEAK, CANalyzer
  3. DBC와 시그널 인코딩 — cantools, DBC → C

상위 프로토콜

  1. Network Management — OSEK NM, AUTOSAR NM, Partial Networking
  2. XCP / CCP — 캘리브레이션, DAQ, STIM
  3. ISO-TP (ISO 15765-2) — SF/FF/CF, Flow Control

진단

  1. UDS (ISO 14229) — SID, DTC, Security Access, Session
  2. OBD-II (ISO 15031) — Mode 1~9, PID, DTC, ELM327

보안

  1. 공격 벡터 — Bus-off, Spoofing, Replay, 차량 해킹 사례
  2. 방어 (SecOC/IDS) — SecOC, CAN-IDS, MAC

트러블슈팅

  1. 실전 트러블슈팅 — 종단 저항·주파수 오차·error passive·DTC 분석

  • ISO 11898-1/2 — CAN data link·physical layer
  • ISO 14229 — UDS, ISO 15765-2 — ISO-TP, ISO 15031 — OBD
